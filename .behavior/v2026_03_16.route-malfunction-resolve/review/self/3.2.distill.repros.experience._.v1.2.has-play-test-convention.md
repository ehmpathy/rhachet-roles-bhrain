# self-review: has-play-test-convention

## journey test convention review

### what the guide asks

journey tests should use `.play.test.ts` suffix to distinguish from:
- `.test.ts` — unit tests
- `.integration.test.ts` — integration tests
- `.acceptance.test.ts` — acceptance tests

### what this repo uses

this repo follows the acceptance test convention:
- `blackbox/*.acceptance.test.ts` — blackbox acceptance tests

the extant pattern is:
- `driver.route.drive.acceptance.test.ts`
- `driver.route.bind.acceptance.test.ts`
- etc.

### decision

**use `.acceptance.test.ts` suffix** — not `.play.test.ts`

**why it holds:**
1. **consistency** — all route tests in this repo use `.acceptance.test.ts`
2. **runner compatibility** — `npm run test:acceptance` runs `*.acceptance.test.ts`
3. **convention established** — a change would require migration of all tests

### planned test file

```
blackbox/driver.route.malfunction.acceptance.test.ts
```

this follows the extant convention for route acceptance tests in this repo.

### conclusion

the play test convention (`.play.test.ts`) is not used in this repo. the acceptance test convention (`.acceptance.test.ts`) is already established and should be followed for consistency.

no issues — we use the repo-appropriate convention.

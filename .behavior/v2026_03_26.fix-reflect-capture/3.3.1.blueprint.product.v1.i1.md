# blueprint: fix reflect.snapshot capture ENOBUFS

## summary

refactor `setSavepoint.ts` to bypass node buffer limits. diff content never enters node:
- apply mode: shell redirect to file, then hash/size from file
- plan mode: shell pipes for hash and size (no files written)

## implementation detail

### apply mode (the fix)

```typescript
// ensure directory exists FIRST (before shell redirect)
fs.mkdirSync(savepointsDir, { recursive: true });

// write staged diff directly to file via shell
execSync(`git diff --staged > "${stagedPatchPath}"`, {
  cwd: input.scope.gitRepoRoot,
});

// write unstaged diff directly to file via shell
execSync(`git diff > "${unstagedPatchPath}"`, {
  cwd: input.scope.gitRepoRoot,
});

// compute hash from files via shell (portable: linux sha256sum, macos shasum)
const combinedHash = execSync(
  `cat "${stagedPatchPath}" "${unstagedPatchPath}" | (sha256sum 2>/dev/null || shasum -a 256) | cut -d' ' -f1`,
  { cwd: input.scope.gitRepoRoot, encoding: 'utf-8' },
).trim();
const hash = combinedHash.slice(0, 7);

// get sizes from filesystem
const stagedBytes = fs.statSync(stagedPatchPath).size;
const unstagedBytes = fs.statSync(unstagedPatchPath).size;
```

### plan mode (shell pipes, no files)

```typescript
// hash via pipe — diff content never enters node
const combinedHash = execSync(
  `(git diff --staged; git diff) | (sha256sum 2>/dev/null || shasum -a 256) | cut -d' ' -f1`,
  { cwd: input.scope.gitRepoRoot, encoding: 'utf-8' },
).trim();
const hash = combinedHash.slice(0, 7);

// sizes via pipe — diff content never enters node
const stagedBytes = parseInt(
  execSync(`git diff --staged | wc -c`, { cwd: input.scope.gitRepoRoot, encoding: 'utf-8' }).trim(),
  10,
);
const unstagedBytes = parseInt(
  execSync(`git diff | wc -c`, { cwd: input.scope.gitRepoRoot, encoding: 'utf-8' }).trim(),
  10,
);
```

## why this approach

| approach | pros | cons |
|----------|------|------|
| maxBuffer increase only | simple | still has a cap |
| shell pipes everywhere | no cap, no buffer | runs git diff multiple times in plan mode |
| stream via spawn | no cap | complex, async refactor |

chosen: **shell pipes everywhere**
- diff content never enters node (per vision)
- no buffer limit in either mode
- plan mode runs `git diff` 3x (hash, staged size, unstaged size) — acceptable tradeoff
- minimal code change, interface unchanged

# blueprint: self-review hashless after $hashbar attempts

## summary

implement hashless promises for self-reviews after `$hashbar` challenge attempts. this prevents infinite loops while review rigor remains.

## codepaths

### domain.objects

```
RouteStoneGuardReviewSelf
├─ [○] slug: string
├─ [○] say: string
└─ [+] hashbar?: number                # optional, default 3
```

### domain.operations/route/promise

```
getSelfReviewTriggeredCount                              # [+] create
├─ input: { stone, slug, route }
├─ glob: $stone.guard.selfreview.$slug.*.triggered
├─ returns: { count: number; newestMtime: Date | null }
└─ .why = count distinct hashes attempted for this slug

getSelfReviewChallengeDecision                           # [~] update
├─ [○] input: { stone, slug, hash, route }
├─ [+] input: { ..., hashbar?: number }                  # add hashbar param (default: 3)
│
├─ [○] lookup triggered report for this hash
│  └─ if absent → create, return challenged
│
├─ [+] check hashbar threshold
│  ├─ call getSelfReviewTriggeredCount({ stone, slug, route })
│  ├─ if count > hashbar AND newestMtime > 90s
│  │  └─ return { decision: 'allowed', hashless: true }
│  └─ else
│     └─ continue to time check for this hash
│
├─ [○] enforce 90s minimum for this hash
│  └─ if elapsed < 90s → return challenged
│
└─ [○] return allowed (hash-bound)

getStonePromises                                         # [~] update
├─ [○] glob: $stone.guard.promise.$slug.$hash.md         # hash-bound
├─ [+] glob: $stone.guard.promise.$slug.md               # hashless (no hash in filename)
├─ [+] if hashless found → include in results (takes precedence)
└─ .why = hashless promises are firm checkpoints

setStoneAsPromised                                       # [~] update
├─ [○] input: { stone, slug, hash, route }
├─ [+] input: { ..., hashless?: boolean }                # new param
│
├─ [+] if hashless = true
│  └─ write to: $stone.guard.promise.$slug.md            # no hash in filename
├─ [○] else
│  └─ write to: $stone.guard.promise.$slug.$hash.md
└─ .why = hashless promises don't invalidate on hash change
```

### caller flow (stepRouteStoneSet)

```
stepRouteStoneSet
├─ [○] parse guard → extract reviews.self
├─ [~] for each self-review
│  ├─ [○] extract slug, say
│  ├─ [+] extract hashbar (default: 3)
│  └─ [~] call getSelfReviewChallengeDecision({ ..., hashbar })
│
├─ [~] if decision.allowed AND decision.hashless
│  └─ [+] call setStoneAsPromised({ ..., hashless: true })
├─ [○] else if decision.allowed
│  └─ [○] call setStoneAsPromised({ ..., hashless: false })
└─ [○] else challenged → block passage
```

## contracts

### input: guard file with hashbar

```yaml
reviews:
  self:
    - slug: all-done
      hashbar: 3           # optional, default: 3
      say: |
        did you complete all?

    - slug: tests-pass
      hashbar: 5           # more iterations for this one
      say: |
        do all tests pass?
```

### output: triggered watermark files (unchanged)

```
.route/$stone.guard.selfreview.$slug.$hash.triggered
```

glob pattern: `$stone.guard.selfreview.$slug.*.triggered`

### output: promise artifacts

```
.route/$stone.guard.promise.$slug.$hash.md    # hash-bound (current)
.route/$stone.guard.promise.$slug.md          # hashless (new: firm checkpoint)
```

## acceptance tests

add to `driver.route.self-review.acceptance.test.ts`:

```
given '[hashbar] stone with reviews.self (hashbar: 3)'
  when '[t0] clone iterates 3 times (hash changes 3x)'
    then 'passage blocked on first 3 attempts'
    then 'passage allowed on 4th attempt (hashbar exceeded)'
    then 'hashless promise created'

  when '[t1] later hash changes occur'
    then 'hashless promise still valid'
    then 'firm checkpoint preserved'

given '[hashbar] stone with configurable hashbar per slug'
  when '[t0] hashbar: 2 on slug all-done'
    then 'allowed after 3 attempts'
  when '[t1] hashbar: 5 on slug tests-pass'
    then 'allowed after 6 attempts'

given '[hashbar] default hashbar when not specified'
  when '[t0] no hashbar in guard'
    then 'default hashbar: 3 applies'
```

## invariants

1. **glob count = attempt count** — no new state file needed
2. **hashless wins** — if hashless promise found, ignore hash-bound promises
3. **time enforcement always** — even after hashbar exceeded, newest mtime must be > 90s
4. **invisible mechanism** — clone sees "blocked" or "progressed", never "invalidated"
5. **backwards compatible** — default hashbar: 3 preserves reasonable behavior

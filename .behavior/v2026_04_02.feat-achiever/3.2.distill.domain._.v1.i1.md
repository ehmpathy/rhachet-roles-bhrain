# domain distillation: achiever role

## overview

this document defines the domain objects, operations, and their relationships for the achiever role.

---

## domain objects

### Goal (DomainLiteral)

the primary domain object. represents a discrete goal with full articulation.

```typescript
interface Goal {
  // identity
  slug: string;                    // unique identifier within scope

  // why (motivation)
  why: {
    ask: string;                   // what was said (verbatim or summarized)
    purpose: string;               // the motivation behind the ask
    benefit: string;               // what success enables
  };

  // what (vision)
  what: {
    outcome: string;               // the desired end state
  };

  // how (path)
  how: {
    task: string;                  // actionable approach
    gate: string;                  // verification criteria
  };

  // status
  status: {
    choice: GoalStatusChoice;      // blocked | enqueued | inflight | fulfilled
    reason: string;                // why this status
  };

  // optional dependency
  when?: PickOne<{
    goal: string;                  // blocked until goal slug is fulfilled
    event: string;                 // blocked on external event
  }>;

  // origin
  source: GoalSource;              // peer:human | peer:robot | self

  // timestamps
  createdAt: string;               // ISO date
  updatedAt: string;               // ISO date
}

class Goal extends DomainLiteral<Goal> implements Goal {
  public static nested = {
    why: DomainLiteral,
    what: DomainLiteral,
    how: DomainLiteral,
    status: DomainLiteral,
    when: DomainLiteral,
  };
}
```

**why nested?** forces articulation. each field is a distinct thought. shallow inference becomes impossible.

### GoalStatusChoice (enum)

```typescript
type GoalStatusChoice = 'blocked' | 'enqueued' | 'inflight' | 'fulfilled';
```

| choice | what it means |
|--------|---------------|
| blocked | cannot proceed, dependency or blocker |
| enqueued | ready to start, not yet active |
| inflight | work in progress |
| fulfilled | complete with evidence |

### GoalSource (enum)

```typescript
type GoalSource = 'peer:human' | 'peer:robot' | 'self';
```

| source | origin |
|--------|--------|
| peer:human | human sent the ask |
| peer:robot | another robot sent the ask |
| self | brain generated from self-observation |

### Ask (DomainLiteral)

represents a single peer input accumulated by hook.onTalk.

```typescript
interface Ask {
  hash: string;                    // deterministic content hash
  content: string;                 // the ask content
  receivedAt: string;              // ISO timestamp
}

class Ask extends DomainLiteral<Ask> implements Ask {}
```

**persistence:** appended as JSONL line to `asks.inventory.jsonl`.

### Coverage (DomainLiteral)

represents a link from ask to goal.

```typescript
interface Coverage {
  hash: string;                    // ask content hash
  goalSlug: string;                // goal that covers this ask
  coveredAt: string;               // ISO timestamp
}

class Coverage extends DomainLiteral<Coverage> implements Coverage {}
```

**persistence:** appended as JSONL line to `asks.coverage.jsonl`.

## domain operations

### goal.memory.set

creates or updates a goal.

```typescript
/**
 * .what = persist a goal to .goals/
 * .why = enables goal memory across context compression
 */
export const setGoal = async (
  input: {
    goal: Goal;                              // full schema for new, partial for update
    covers?: string[];                       // ask hashes this goal covers
    scope: 'route' | 'repo';                 // where to persist
  },
  context: {
    route?: string;                          // route path if scope=route
    branch: string;                          // current branch
    log: LogMethods;
  },
): Promise<{
  path: string;                              // where goal was persisted
  covered: string[];                         // hashes recorded as covered
}> => { ... };
```

**behavior:**
- new goal: creates `$offset.$slug.goal.yaml` and `$offset.$slug.status=$choice.flag`
- update goal: updates `.goal.yaml` and renames `.flag` if status changed
- covers: appends entries to `asks.coverage.jsonl`
- validates: full schema required for new, status change requires reason

### goal.memory.get

reads extant goals.

```typescript
/**
 * .what = read goals from .goals/
 * .why = enables goal enumeration and status check
 */
export const getGoals = async (
  input: {
    scope: 'route' | 'repo';                 // where to read
    filter?: {
      status?: GoalStatusChoice;             // filter by status
      slug?: string;                         // get specific goal
    };
  },
  context: {
    route?: string;                          // route path if scope=route
    branch: string;                          // current branch
    log: LogMethods;
  },
): Promise<{
  goals: Goal[];                             // may be empty
}> => { ... };
```

**behavior:**
- reads all `.goal.yaml` files in scope
- reads `.flag` files for status.choice
- filters by status if specified
- returns empty array if no goals

### goal.infer.triage

reads uncovered asks and extant goals for triage.

```typescript
/**
 * .what = enumerate uncovered asks and extant goals
 * .why = enables brain to triage asks into goals
 */
export const getTriageState = async (
  input: {
    scope: 'route' | 'repo';                 // where to read
  },
  context: {
    route?: string;
    branch: string;
    log: LogMethods;
  },
): Promise<{
  asks: Ask[];                               // uncovered asks
  goals: Goal[];                             // extant goals
  coverage: Coverage[];                      // extant coverage entries
}> => { ... };
```

**behavior:**
- reads `asks.inventory.jsonl` for all asks
- reads `asks.coverage.jsonl` for covered hashes
- filters asks to return only uncovered
- reads all goals for reference

### ask.inventory.append

appends an ask to the inventory (used by hook.onTalk).

```typescript
/**
 * .what = append ask to inventory
 * .why = accumulates peer input for triage
 */
export const appendAskToInventory = async (
  input: {
    content: string;                         // the ask content
    scope: 'route' | 'repo';
  },
  context: {
    route?: string;
    branch: string;
    log: LogMethods;
  },
): Promise<{
  ask: Ask;                                  // the appended ask with hash
}> => { ... };
```

**behavior:**
- computes deterministic content hash
- appends JSONL line to `asks.inventory.jsonl`
- returns the Ask with computed hash

### ask.coverage.append

appends coverage entries (used internally by goal.memory.set).

```typescript
/**
 * .what = record that asks are covered by a goal
 * .why = tracks which asks have been triaged
 */
export const appendCoverage = async (
  input: {
    hashes: string[];                        // ask hashes
    goalSlug: string;                        // goal that covers them
    scope: 'route' | 'repo';
  },
  context: {
    route?: string;
    branch: string;
    log: LogMethods;
  },
): Promise<{
  coverage: Coverage[];                      // the appended entries
}> => { ... };
```

**behavior:**
- creates Coverage entry for each hash
- appends JSONL lines to `asks.coverage.jsonl`

## persistence structure

### route-scoped

```
$route/.goals/
├── asks.inventory.jsonl           # all peer input
├── asks.coverage.jsonl            # hash → goalSlug map
├── 0000000.define-shape.goal.yaml
├── 0000000.define-shape.status=fulfilled.flag
├── 0003600.write-tests.goal.yaml
└── 0003600.write-tests.status=blocked.flag
```

### repo-scoped

```
.goals/$branch/
├── .gitignore                     # auto-created
├── asks.inventory.jsonl
├── asks.coverage.jsonl
├── 0000000.fix-auth-test.goal.yaml
├── 0000000.fix-auth-test.status=inflight.flag
├── 0086400.update-readme.goal.yaml
└── 0086400.update-readme.status=enqueued.flag
```

**file name format:**
- `$offset.$slug.goal.yaml` — goal content (YAML)
- `$offset.$slug.status=$choice.flag` — status marker (empty file)
- offset = seconds from parent dir mtime, 7-digit leftpad

**why flag files?** status.choice visible from filename. `glob('**/*.status=inflight.flag')` finds active goals with zero file reads.

## relationships

### object relationships

```
Ask (1) --covers--> (1) Coverage --references--> (1) Goal
```

- one Ask may have one Coverage entry
- one Coverage references one Goal by slug
- one Goal may be referenced by many Coverage entries

### treestruct

```
achiever
├── domain.objects
│   ├── Goal
│   ├── Ask
│   └── Coverage
└── domain.operations
    ├── goal.memory.set (writes Goal, Coverage)
    ├── goal.memory.get (reads Goal)
    ├── goal.infer.triage (reads Ask, Coverage, Goal)
    ├── ask.inventory.append (writes Ask)
    └── ask.coverage.append (writes Coverage)
```

### dependencies

| operation | reads | writes |
|-----------|-------|--------|
| goal.memory.set | Goal (if update) | Goal, Coverage |
| goal.memory.get | Goal | — |
| goal.infer.triage | Ask, Coverage, Goal | — |
| ask.inventory.append | — | Ask |
| ask.coverage.append | — | Coverage |

## composition for wish

### usecase: multi-part request triage

```
hook.onTalk fires
  → appendAskToInventory (accumulate)

hook.onStop fires
  → getTriageState (check uncovered)
  → if uncovered > 0, halt and prompt triage

brain runs triage
  → for each uncovered ask:
    → setGoal with --covers (persists goal + coverage)
  → getTriageState (verify zero uncovered)
  → session may end
```

### usecase: goal lifecycle

```
brain starts work
  → setGoal with status=inflight

brain completes work
  → setGoal with status=fulfilled, reason=evidence

brain is blocked
  → setGoal with status=blocked, when.goal=$slug
```

### usecase: goal persistence

```
context compresses
  → goals remain in .goals/ (filesystem)

new session starts
  → getGoals (read extant)
  → resume work
```

## validation rules

### Goal schema validation

| field | required | validation |
|-------|----------|------------|
| slug | yes | non-empty string |
| why.ask | yes | non-empty string |
| why.purpose | yes | non-empty string |
| why.benefit | yes | non-empty string |
| what.outcome | yes | non-empty string |
| how.task | yes | non-empty string |
| how.gate | yes | non-empty string |
| status.choice | yes | valid enum value |
| status.reason | yes | non-empty string |
| source | yes | valid enum value |
| createdAt | yes | ISO date string |
| updatedAt | yes | ISO date string |

### scope validation

| scope | requirement |
|-------|-------------|
| route | context.route must be set |
| repo | context.branch must not be 'main' |

## summary

| object | purpose | persistence |
|--------|---------|-------------|
| Goal | discrete goal with full articulation | `.goal.yaml` + `.flag` |
| Ask | peer input for triage | `asks.inventory.jsonl` |
| Coverage | ask → goal link | `asks.coverage.jsonl` |

| operation | purpose |
|-----------|---------|
| goal.memory.set | create/update goal |
| goal.memory.get | read goals |
| goal.infer.triage | enumerate uncovered asks |
| ask.inventory.append | accumulate peer input |
| ask.coverage.append | record coverage |

the domain supports the wish: persist goals, track asks, ensure coverage.

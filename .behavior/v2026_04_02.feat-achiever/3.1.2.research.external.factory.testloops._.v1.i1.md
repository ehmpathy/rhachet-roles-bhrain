## category 2: test-fns BDD framework

### [3] test-fns given/when/then

**source:** npmjs.com/package/test-fns

**type:** [official] — npm package

**quote:** "BDD-style test utilities for jest with given/when/then semantics."

**pattern:**
```typescript
import { given, when, then, useBeforeAll, useThen } from 'test-fns';

given('[case1] scenario description', () => {
  const scene = useBeforeAll(async () => {
    return { entity: await createEntity() };
  });

  when('[t0] action occurs', () => {
    const result = useThen('operation succeeds', async () =>
      performOperation(scene.entity),
    );

    then('expected outcome', () => {
      expect(result.status).toEqual('success');
    });
  });
});
```

**relevance:** all achiever acceptance tests use this pattern.

## category 7: skill invocation testing

### [14] invokeSkill test utility

**source:** `blackbox/.test/invokeRouteSkill.ts`

**type:** [internal] — bhrain pattern

**pattern:**
```typescript
export const invokeSkill = async (input: {
  skill: string;
  args: Record<string, string>;
  cwd: string;
  stdin?: string;
}): Promise<{ code: number; stdout: string; stderr: string }> => {
  const argsStr = Object.entries(input.args)
    .map(([k, v]) => `--${k} "${v}"`)
    .join(' ');

  const cmd = input.stdin
    ? `echo '${input.stdin}' | npx rhachet run --skill ${input.skill} ${argsStr}`
    : `npx rhachet run --skill ${input.skill} ${argsStr}`;

  return execAsync(cmd, { cwd: input.cwd });
};
```

**relevance:** achiever tests invoke skills via this pattern.

## category 9: content hash testing

### [18] deterministic hash verification

**source:** node.js crypto

**type:** [official] — node.js built-in

**pattern:**
```typescript
import crypto from 'crypto';

const computeHash = (content: string): string =>
  crypto.createHash('sha256').update(content).digest('hex').slice(0, 8);

then('hash is deterministic', () => {
  const hash1 = computeHash('fix the test');
  const hash2 = computeHash('fix the test');
  expect(hash1).toEqual(hash2);
});

then('different content produces different hash', () => {
  const hash1 = computeHash('fix the test');
  const hash2 = computeHash('update the readme');
  expect(hash1).not.toEqual(hash2);
});
```

**relevance:** achiever uses content hash for ask identification.

## convergence signals

### BDD structure is standard

**sources [3, 4, 5] agree:** given/when/then with useBeforeAll and useThen is the bhrain pattern.

### temp directories with symlinks

**sources [6, 7] agree:** genTempDir with node_modules symlinks enables isolated skill testing.

### JSONL append verification

**sources [8, 9] agree:** read file, split lines, parse JSON, verify content and order.

### hook simulation via stdin

**sources [12, 13] agree:** pipe JSON to hook, verify exit code and stderr.

---

## anti-patterns from sources

### anti-pattern 1: mocking file system

**sources [6, 7] warn:** don't mock fs. use real temp directories for integration tests.

### anti-pattern 2: testing implementation

**achiever decision:** test behavior (files created, content correct) not implementation (function calls).

### anti-pattern 3: redundant expensive operations

**sources [4] warn:** use useThen to share results across then blocks.

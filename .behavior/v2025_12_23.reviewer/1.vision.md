


for example

---

to run all of the mechanic rules against the latest diffs since main
```sh
npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.[feedback].v1.[given].by_robot.md
```

---

to run only the evolvability rules of the mechanic specifically
```sh
npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/evolvable.*/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.per_evolvable.[feedback].v1.[given].by_robot.md
```

(note that the output is handcrafted by caller, not something that this skill generates)

---

to run rules from _this_ repo against all diffs since main

```sh
npx rhachet run --repo bhrain --skill review \
  --rules .agent/repo=.this/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.per_repo.[feedback].v1.[given].by_robot.md
```

(note that the output is handcrafted by caller, not something that this skill generates)


---

then, other roles can leverage this skill to make their own narrower skills

e.g., the mechanic could craft a skill which itself runs each surdomain of rules in parallel

```sh
npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/evolvable.*/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.per_evolvable.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/consistent.*/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.per_consistent.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/consistent.*/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.per_readable.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*mechanic/**/lang.*/rule.*.md \
  --diffs since-main \
  --output .behavior/v2025_12_23.say-hello/7.review.mechanics.per_lang.[feedback].v1.[given].by_robot.md
```


and then the builders can invoke that via

```sh
npx rhachet run --role mechanic --skill review
```

which will do the right thing optimally for them


---

same with the usecase of bhuild

e.g.,


```sh

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/criteria.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/2.criteria.md \
  --output .behavior/v2025_12_23.say-hello/2.criteria.md.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/blueprint.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/3.3.blueprint.md \
  --output .behavior/v2025_12_23.say-hello/3.3.blueprint.md.[feedback].v1.[given].by_robot.md

npx rhachet run --repo bhrain --skill review \
  --rules .agent/**/*behaver/**/roadmap.*/rule.*.md \
  --paths .behavior/v2025_12_23.say-hello/3.3.roadmap.md \
  --output .behavior/v2025_12_23.say-hello/3.3.roadmap.md.[feedback].v1.[given].by_robot.md
```

and builders can invoke via

```sh
npx rhachet run --repo bhuild --skill review.behavior --of say-hello
```

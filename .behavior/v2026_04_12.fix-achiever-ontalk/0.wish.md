1. fire when peer sends message

4. NOT halt the brain

what was implemented:

## what needs to happen

### 1. add `hook.onTalk` mode to `goalInferTriage`

currently `parseArgsForTriage` only recognizes `'triage' | 'hook.onStop'`

need to:
- accept `'hook.onTalk'` as a valid mode
- when mode is `hook.onTalk`:
  - read the user's message from stdin (claude code pipes it)
  - call `setAsk({ content, scopeDir })` to append to inventory
  - emit a short reminder to stderr (not full triage)
  - exit 0 (never halt)

### 2. the reminder output

per the original vision (lines 399-414):

```
🦉 to forget an ask is to break a promise. remember.

🔮 goal.triage.infer --from peer --when hook.onTalk
   ├─ from = peer:human
   ├─ ask
   │  ├─
   │  │
   │  │  {the user's message}
   │  │
   │  └─
   │
   └─ consider: does this impact your goals?
      ├─ if yes, triage before you proceed
      └─ run `rhx goal.triage.infer`
```

### 3. stdin handling

the `UserPromptSubmit` hook receives the user's prompt via stdin. the skill needs to:
- read stdin to get the message content
- compute hash via `setAsk`
- append to `asks.inventory.jsonl`

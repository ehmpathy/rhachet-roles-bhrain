# blackbox criteria: peer-review-budgets

## usecase.1 = budget consumption

```
given(reviewer has budget: 3)
  when(review invocation succeeds with blockers)
    then(budget decrements to 2)
      sothat(each review attempt is tracked)
  when(review invocation succeeds with no blockers)
    then(budget decrements to 2)
      sothat(approved reviews still consume budget)
    then(reviewer status is approved)
      sothat(reviewer is terminal)
  when(review invocation malfunctions)
    then(budget stays at 3)
      sothat(system errors are not charged to user)
  when(review is cached from prior run)
    then(budget stays at 3)
      sothat(cached approvals are free)
```

## usecase.2 = exhaustion

```
given(reviewer has budget: 1)
  when(review invocation succeeds)
    then(budget decrements to 0)
    then(reviewer status is exhausted)
      sothat(reviewer cannot run again)
    then(stdout shows exhaustion message)
      sothat(driver knows what happened)

given(reviewer is exhausted)
  when(artifact hash changes)
    then(reviewer stays exhausted)
      sothat(hash changes cannot trigger infinite loops)
  when(stone is rewound via --as rewound)
    then(reviewer budget resets to full)
      sothat(explicit rewind enables fresh review cycle)
```

## usecase.3 = level execution order

```
given(level 1 reviewers: [cheapo, cheap2], level 2 reviewers: [primo])
  when(route.drive begins)
    then(level 1 reviewers run)
    then(level 2 reviewers wait)
      sothat(cheap brains catch basic issues first)

given(level 1 reviewers all approved)
  when(route.drive continues)
    then(level 2 reviewers run)
      sothat(expensive brains only see clean code)

given(level 1 has one approved, one exhausted)
  when(route.drive continues)
    then(level 2 reviewers run)
      sothat(exhaustion is terminal like approved)
      sothat(level progression is not blocked by exhaustion)
```

## usecase.4 = route halts on exhaustion

```
given(all reviewers are terminal)
  when(any reviewer is exhausted)
    then(route halts)
    then(human is prompted for decision)
      sothat(human reviews what exhausted reviewers found)
    then(options show: extend budget OR approve as-is)
      sothat(human has clear paths forward)

given(route is halted due to exhaustion)
  when(human runs rhx route.guard.budget --for review --add 2)
    then(exhausted reviewer budget increases by 2)
    then(reviewer status changes from exhausted to active)
    then(route.drive can continue)
      sothat(human can invest more rounds)
  when(human runs rhx route.stone.set --stone $stone --as approved)
    then(stone is approved)
    then(route.drive continues to next stone)
      sothat(human can accept current state)
```

## usecase.5 = default behaviors

```
given(reviewer has no budget declared)
  when(guard file is parsed)
    then(budget defaults to unlimited)
      sothat(backwards compatibility is preserved)

given(reviewer has no level declared)
  when(guard file is parsed)
    then(level defaults to 1)
      sothat(unleveled reviewers run first by default)
```

## usecase.6 = budget extension granularity

```
given(route has reviewers: [cheapo, primo])
  when(human runs rhx route.guard.budget --for review --add 2)
    then(all reviewers get +2 budget)
      sothat(bulk extension is easy)
  when(human runs rhx route.guard.budget --for review --peer primo --add 2)
    then(only primo gets +2 budget)
      sothat(targeted extension is possible)
```

## usecase.7 = stdout feedback

```
given(reviewer exhausts budget)
  when(review run completes)
    then(stdout shows reviewer slug)
    then(stdout shows budget: 0/N)
    then(stdout shows status: exhausted)
      sothat(driver sees exactly what happened)

given(route.drive shows reviewers)
  when(reviewers have different levels)
    then(stdout shows budget and level for each reviewer)
    then(stdout shows status for each reviewer)
      sothat(driver can track all reviewer states)
```

## usecase.8 = approval unification

```
given(stone has explicit approved? judge)
given(one reviewer is exhausted)
  when(human approves due to exhaustion)
    then(explicit approved? judge is also satisfied)
      sothat(one approval covers all)
  when(human approves for explicit judge)
    then(exhaustion approval is also satisfied)
      sothat(one approval covers all)
```

## usecase.9 = budget per stone scope

```
given(route has stones: [stone.A, stone.B])
given(reviewer primo has budget: 3 declared in both stones)
  when(primo runs on stone.A, consumes 2 rounds)
    then(primo still has budget 3 for stone.B)
      sothat(budget is per stone, not per route)
  when(primo exhausts on stone.A)
    then(primo has fresh budget on stone.B)
      sothat(each stone gets its own budget cycle)
```

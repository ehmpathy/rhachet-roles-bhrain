- src/domain.operations/route/guard/runStoneGuardReviews.ts:510
- src/domain.operations/route/guard/reviewPeerMeter/resetRouteStoneGuardReviewPeerMeters.ts:20

Multiple reads of the append-only meters file occur at startup, followed by writes via setRouteStoneGuardReviewPeerMeter during long-running review execution. No locks, transactions, or atomic file operations protect the shared state. Concurrent execution (multiple stones, retries, or parallel processes on same route) can cause lost updates, duplicate rounds, or inconsistent verdicts.

**snippet**:
```ts

const allMeters = content
  .split('\n')
  .filter(Boolean)
  .map((line) => new RouteStoneGuardReviewPeerMeter(JSON.parse(line)));

// filter to this stone, last entry wins per index
const latestByIndex = new Map<number, RouteStoneGuardReviewPeerMeter>();
for (const meter of allMeters) {
  if (meter.stone === input.stone) {
    latestByIndex.set(meter.index, meter);
  }
}
```

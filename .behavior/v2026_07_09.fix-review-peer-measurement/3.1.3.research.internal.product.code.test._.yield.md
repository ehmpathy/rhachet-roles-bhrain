> import { getReviewCountsFromContent } from './getReviewCountsFromContent';

  preserved; only the assertion shape changes (`if (counts.detected) expect(...)`).
- **[FACT]** this is a **pure unit test** — no brain, no i/o. it stays fast and deterministic. it
  is the natural home for the deterministic-leaf boundary cases (empty, last-match, singular).

- **[REUSE]** the `toMatchSnapshot()` + `ReviewerTreeState` fixture pattern is reused verbatim —
  this is a **pure** test (no brain), so it stays fast and locks the exact render per

(referenced at lines 59, 78, 92, 106, 121, 173, 189, 211, 264, 273 — many `runStoneGuardReviews(`
call sites)

- **[EXTEND]** the guard integration test currently exercises `runStoneGuardReviews` with review
  subprocesses. it **[EXTEND]s** to cover the waterfall end-to-end: a reviewer whose stdout is
  prose (no numbers) + exit 0 → the guard obtains counts via fallback and marks the reviewer with

- **[FACT]** because `runStoneGuardReviews` now needs a brain context, this integration test must

- **[KHUE]** flagged: this integration test invokes a real brain on the fallback path, so its
  fallback cases must also use `when.repeatably(REPEATABLY_CONFIG)`; the deterministic cases stay
  plain (no brain call, deterministic).

## pattern 6 — test scope classification (unit vs integration vs acceptance)

**[6]** the repo's test-scope briefs (loaded rules): `rule.forbid.unit.remote-boundaries`,
`rule.require.test-coverage-by-grain`, `rule.forbid.integration.mocks`.

- **[REUSE]** the grain→scope map dictates our test files:
  - `getReviewCountsFromContent` (transformer, pure) → **`.test.ts`** (unit) — pattern 1
  - `getReviewCountsViaBrain` (communicator, brain i/o) → **`.integration.test.ts`** — pattern 3
  - `getReviewCounts` (orchestrator) → **`.integration.test.ts`** — pattern 3/5
  - `formatGuardReviewerTree` (pure render) → **`.test.ts`** snapshot — pattern 4
- **[FACT]** `rule.forbid.integration.mocks` means the sub-brain tests hit the **real**
  fireworks/deepseek model — never a mock. credentials come from keyrack `env: 'test'`
  (pattern 2a); absence must fail loud via `ConstraintError`, not skip
  (`rule.forbid.failhide` in test scope).

### net shape of the test change

- **[REUSE]** 4 patterns (brain maker, repeatably config, caseBrain template, scope rules) —
  every piece of LLM-test infrastructure the sub-brain needs already exists in the repo.
- **[EXTEND]** 3 patterns: union-shaped assertions (unit), new adversarial caseBrain files
  (integration), and two new snapshot cases (render).
- **[REPLACE]** none — no test is deleted; the parser's boundary cases carry forward.

- **[FACT]** the LLM tests depend on a keyrack `env: 'test'` credential for
  `fireworks/deepseek/v4-flash` (`genTestBrainContext.ts:60`). CI already runs the extant

  (3-of-3) (`repeatably.ts:17`) — the built-in guard against LLM flake, reused as-is.

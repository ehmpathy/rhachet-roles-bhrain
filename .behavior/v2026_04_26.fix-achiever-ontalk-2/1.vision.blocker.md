# blocker: Claude Code lacks source metadata in UserPromptSubmit stdin

## the problem

Claude Code's `UserPromptSubmit` hook fires not only for user prompts but also when subagents/Tasks complete (bug #16952). The stdin JSON lacks any field to distinguish the source:

```json
{
  "session_id": "...",
  "transcript_path": "...",
  "cwd": "...",
  "hook_event_name": "UserPromptSubmit",
  "prompt": "..."  // ← no way to know if this came from user or hook output
}
```

## why this blocks us

we cannot reliably filter spurious `onTalk` triggers because:

1. **no source field** — stdin schema has no `source`, `origin`, or `trigger_type`
2. **bug not fixed** — issue #16952 was closed due to inactivity, not resolution
3. **heuristics are fragile** — content-based detection would break on edge cases

## evidence

- **bug report:** https://github.com/anthropics/claude-code/issues/16952
- **schema source:** https://gist.github.com/FrancisBourre/50dca37124ecc43eaf08328cdcccdb34
- **status:** closed (inactivity), locked, not fixed

## impact beyond asks

per comment on #16952: spurious `UserPromptSubmit` hooks break the agentic loop — they create false turn boundaries that terminate multi-step workflows mid-execution.

## options to unblock

| option | effort | reliability | notes |
|--------|--------|-------------|-------|
| upstream feature request | low | high | request `source` field in stdin schema |
| content heuristics | medium | low | detect `🦉` or `goal.triage` in prompt (fragile) |
| internal state track | medium | medium | rhachet tracks "just ran onStop", skips next onTalk |
| wait for upstream fix | none | unknown | bug closed, no timeline |

## recommendation

file upstream feature request for `source` field in stdin schema. this is the clean fix. heuristics are a bandaid.

## to unblock this route

1. file feature request on anthropics/claude-code
2. or decide to proceed with heuristics workaround
3. or decide to wait for upstream fix

# blueprint: route.stone.set --as rewound

## overview

add `--as rewound` support to `route.stone.set` command. when invoked:
1. find the stone by fuzzy pattern match
2. cascade to all subsequent stones in the route
3. delete guard artifacts for each affected stone
4. append `{stone, status: 'rewound'}` to passage.jsonl
5. emit confirmation tree to stdout

approvals are not deleted — they become invalid when route.drive interprets the passage log and finds a rewind entry that appears after them.

## approval invalidation

### interpretation logic (getStonePassageStatus)

when route.drive checks if a stone is approved:

1. scan passage.jsonl chronologically
2. find approval entry for stone N
3. check for any rewind entry where:
   - the rewound stone index <= N
   - the rewind entry appears AFTER the approval in the log
4. if such a rewind found → approval is invalid

```ts
// pseudocode — simplified because cascade writes rewound for EACH affected stone
const isApprovalValid = (
  stone: string,
  entries: PassageEntry[],
): boolean => {
  let approvalPosition = -1;
  let rewindPosition = -1;

  for (let i = 0; i < entries.length; i++) {
    const entry = entries[i];
    if (entry.stone !== stone) continue;

    if (entry.status === 'approved') approvalPosition = i;
    if (entry.status === 'rewound') rewindPosition = i;
  }

  // no approval found
  if (approvalPosition === -1) return false;

  // approval invalidated if rewind appears after it
  if (rewindPosition > approvalPosition) return false;

  return true;
};
```

## edge cases

| case | behavior |
|------|----------|
| stone not found | BadRequestError with helpful message |
| multiple matches | BadRequestError asks for specificity |
| no guard artifacts | no-op for deletion, still appends passage |
| already rewound | idempotent, appends another passage entry |
| first stone | only affects stones from 1 onwards |
| last stone | only affects that stone (no cascade) |

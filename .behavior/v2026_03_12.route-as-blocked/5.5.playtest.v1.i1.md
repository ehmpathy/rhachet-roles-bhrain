# playtest: route --as blocked

## prerequisites

- repo cloned and dependencies installed (`pnpm install`)
- built: `npm run build`
- a test route with stones to work with

## sandbox

- all file operations target `.temp/playtest-blocked/`
- creates a minimal route for testing

---

## setup

```bash
# create sandbox directory
mkdir -p .temp/playtest-blocked

# create a minimal route for testing
mkdir -p .temp/playtest-blocked/.behavior/test-route

# create a simple stone
cat > .temp/playtest-blocked/.behavior/test-route/1.test.stone << 'EOF'
# test stone

complete this test stone
EOF

# bind the route
cd .temp/playtest-blocked
npx rhx route.bind.set --route .behavior/test-route
cd ../..
```

---

## happy paths

### [case1] first attempt shows nudge

**action:**
```bash
cd .temp/playtest-blocked
npx rhx route.stone.set --stone 1.test --as blocked
```

**expected outcome:**
- stdout shows "passage = unchanged (articulation required)"
- stdout shows nudge with "failure is only the opportunity to begin again"
- stdout shows "what blocks you, grows you"
- stdout shows path to articulate: `.behavior/test-route/.route/blocker/1.test.md`
- `.behavior/test-route/.route/1.test.blocked.triggered` file created

---

### [case2] second attempt with articulation succeeds

**action:**
```bash
cd .temp/playtest-blocked

# create articulation file
mkdir -p .behavior/test-route/.route/blocker
cat > .behavior/test-route/.route/blocker/1.test.md << 'EOF'
# blocker: 1.test

## what blocks me
playtest scenario - simulating a block

## what i tried
- attempted to complete the stone
- reviewed relevant code

## what i need
human guidance
EOF

# second attempt
npx rhx route.stone.set --stone 1.test --as blocked
```

**expected outcome:**
- stdout shows "passage = blocked"
- stdout shows "reason:" with path to articulation file
- passage.jsonl contains entry with `status: "blocked"`

---

### [case3] route.drive shows blocked state

**action:**
```bash
cd .temp/playtest-blocked
npx rhx route.drive
```

**expected outcome:**
- stdout shows "halted, stone marked blocked"
- stdout shows path to articulation file under "reason:"

---

### [case4] robot recovers via --as passed

**action:**
```bash
cd .temp/playtest-blocked
npx rhx route.stone.set --stone 1.test --as passed
npx rhx route.drive
```

**expected outcome:**
- first command shows "passage = allowed"
- second command shows "route complete!" (only one stone in test route)

---

## edgey paths

### [edge1] articulation without seeing nudge first

**setup:**
```bash
# create fresh route
rm -rf .temp/playtest-blocked/.behavior/test-route/.route
mkdir -p .temp/playtest-blocked/.behavior/test-route/.route/blocker
cat > .temp/playtest-blocked/.behavior/test-route/.route/blocker/1.test.md << 'EOF'
# blocker: 1.test
## what blocks me
test
EOF
```

**action:**
```bash
cd .temp/playtest-blocked
npx rhx route.stone.set --stone 1.test --as blocked
```

**expected outcome:**
- stdout shows nudge (same as case1)
- passage remains unchanged
- robot must see nudge before block can succeed

---

### [edge2] guard-initiated blocks handled separately

this is verified by extant acceptance tests. guard-initiated blocks (with `blocker` field like `'approval'`) show "halted, human approval required" instead of "halted, stone marked blocked".

---

## pass/fail criteria

- ✓ pass if: all happy paths produce expected output; nudge shown on first attempt; articulation required for block success; route.drive shows blocked state correctly; robot can recover via --as passed
- ✗ fail if: block succeeds without articulation; nudge not shown; route.drive shows wrong state; recovery fails

## extant mechanisms checked

### delStone.ts

**what it does**: deletes the WHOLE stone file (and its guard file). refuses if artifacts exist.

**does delStoneGuardArtifacts duplicate it?** no — delStone removes the stone itself; delStoneGuardArtifacts removes validation state but preserves the stone.

### delStoneGuardBlockerReport.ts

**what it does**: legacy no-op. passage.jsonl is append-only, so "blocked" state is superseded by subsequent entries.

**does delStoneGuardArtifacts duplicate it?** no — this is about passage entries, not guard files.

### getAllStoneGuardArtifactsByHash.ts

**what it does**: retrieves guard artifacts for a SPECIFIC hash. parses reviews and judges into domain objects.

**does delStoneGuardArtifacts duplicate it?** no — delStoneGuardArtifacts needs to delete artifacts across ALL hashes (not filter by hash). also includes promise and triggered files which getAllStoneGuardArtifactsByHash does not handle.

### getAllStoneArtifacts.ts

**what it does**: retrieves content artifacts (work products), not guard artifacts.

**does delStoneGuardArtifacts duplicate it?** no — different scope entirely. content vs validation state.

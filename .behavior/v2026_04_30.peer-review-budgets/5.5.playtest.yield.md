# playtest: peer-review-budgets

## prerequisites

- `npm run build` completes successfully
- `npx rhachet roles link --role driver` completes successfully
- current directory is repo root

## sandbox

all file operations target `@gitroot/.temp/` to avoid repo pollution.

---

## happy paths

### path 1: budget consumption and exhaustion

this path verifies budget decrements on each review and triggers exhaustion.

**coverage:** `blackbox/driver.route.peer-budget.acceptance.test.ts`
- `given('[journey] peer review budget workflow')` → full journey coverage

**setup:**

```bash
# create a temp route with peer review budget
mkdir -p .temp/playtest-budget
cp -r blackbox/.test/assets/route-peer-budget/* .temp/playtest-budget/
chmod +x .temp/playtest-budget/.test/mock-review.sh
cd .temp/playtest-budget
npx rhachet roles link --role driver
```

**test file: `blackbox/driver.route.peer-budget.acceptance.test.ts`**

**step 1: first review attempt (budget 2 -> 1)**

```bash
mkdir -p src
echo 'export const feature = () => "v1";' > src/feature.ts
rhx route.stone.set --stone 1.execute --route . --as passed
```

expected outcome:
- exit code is non-zero (blocked by review)
- output shows review blockers
- budget consumed (2 -> 1)

pass if: output contains "block" or "fail"
fail if: exit code 0 (should be blocked)

**step 2: second review attempt (budget 1 -> 0, exhausted)**

```bash
echo 'export const feature = () => "v2";' > src/feature.ts
rhx route.stone.set --stone 1.execute --route . --as passed
```

expected outcome:
- exit code is non-zero (blocked)
- output mentions exhaustion or budget
- reviewer shows status: exhausted

pass if: output contains "exhaust" or "budget"
fail if: no mention of exhaustion after budget depleted

**step 3: third attempt (already exhausted)**

```bash
echo 'export const feature = () => "v3";' > src/feature.ts
rhx route.stone.set --stone 1.execute --route . --as passed
```

expected outcome:
- exit code is non-zero
- output shows reviewer is still exhausted (not re-run)

pass if: shows exhaustion persists
fail if: reviewer ran again after exhaustion

### path 2: budget extension and recovery

this path verifies that budget extension un-exhausts a reviewer.

**coverage:** `blackbox/driver.route.peer-budget.acceptance.test.ts`
- `when('[t3] budget extended')` → budget extension
- `when('[t4] pass after budget extension')` → recovery after extension

**setup:** continue from path 1

**step 4: extend budget for exhausted reviewer**

```bash
rhx route.guard.budget --for review --add 2 --peer r1
```

expected outcome:
- exit code 0
- output confirms budget was extended
- shows new budget amount

pass if: output contains "budget" and shows extension
fail if: exit code non-zero or no confirmation

**step 5: pass after budget extension with fixed code**

```bash
# make reviews pass via flag file creation
touch .test/review-should-pass
echo 'export const feature = () => "v4-fixed";' > src/feature.ts
rhx route.stone.set --stone 1.execute --route . --as passed
```

expected outcome:
- exit code 0
- output shows "passage = allowed"

pass if: exit code 0 and passage allowed
fail if: still blocked after budget extension and fix

### path 3: level execution order

this path verifies level 1 reviewers run before level 2 (cheaper brains first, then expensive).

**coverage:** `blackbox/driver.route.peer-budget-levels.journey.acceptance.test.ts`
- `given('[journey] peer review budget levels')` → full journey coverage
- `then('cheapo did NOT run yet (level 1 waits)')` → level execution order

**setup (run from repo root):**

```bash
# return to repo root first
cd "$(git rev-parse --show-toplevel)"

# create a route with two levels
mkdir -p .temp/playtest-levels
cat > .temp/playtest-levels/1.test.guard << 'EOF'
artifacts:
  - src/**/*.ts

reviews:
  peer:
    - slug: cheap
      run: bash -c 'echo "# cheap review"; echo "no blockers"; exit 0'
      budget: 5
      level: 1

    - slug: expensive
      run: bash -c 'echo "# expensive review"; echo "no blockers"; exit 0'
      budget: 3
      level: 2
EOF

cat > .temp/playtest-levels/1.test.stone << 'EOF'
test stone for level verification
EOF

cd .temp/playtest-levels
npx rhachet roles link --role driver
```

**step 6: verify level 1 runs first**

```bash
mkdir -p src
echo 'export const test = 1;' > src/test.ts
rhx route.drive
```

expected outcome:
- output shows both reviewers
- level 1 (cheap) shows status that indicates it runs first
- level 2 (expensive) shows awaits level 1 or runs after

pass if: level order is visible in output
fail if: expensive runs before or concurrently with cheap

## edgey paths

### edge 1: cached review does not consume budget

**coverage:** `blackbox/driver.route.peer-budget.acceptance.test.ts`
- `when('[t5] pass re-attempted (cached review, same artifact hash)')`
- `then('review is cached (not re-run)')` → cache hit
- `then('budget unchanged (3/4, not 4/4)')` → cache preserves budget

**given:** a review has already passed (cached)
**when:** pass is attempted again
**then:** budget should NOT decrement

```bash
# after path 2 completed (review passed)
# run pass again
rhx route.stone.set --stone 1.execute --route . --as passed
```

expected outcome:
- exit code 0
- no budget consumed (cached)

pass if: budget stays same (cache hit)
fail if: budget decremented on cache hit

### edge 2: malfunction does not consume budget

**given:** review command fails with error (not review failure)
**when:** pass is attempted
**then:** budget should NOT decrement

manual verification: confirm malfunction vs review-failure handle in code.

### edge 3: rewound stone resets exhaustion

**given:** reviewer is exhausted
**when:** stone is rewound via `--as rewound`
**then:** budget resets to full

```bash
# after exhaustion in path 1
rhx route.stone.set --stone 1.execute --route . --as rewound
rhx route.drive
```

expected outcome:
- reviewer budget shows full again
- reviewer status is no longer exhausted

pass if: budget reset to declared value
fail if: still shows exhausted after rewind

---

## pass/fail criteria

overall pass if:
- budget decrements on each successful review invocation
- exhaustion triggers after budget hits 0
- `route.guard.budget --for review --add N` extends budget and un-exhausts reviewer
- level 1 reviewers run before level 2 reviewers
- cached reviews do not consume budget
- rewound stones reset exhaustion state

overall fail if:
- budget does not decrement on review
- exhaustion does not trigger at budget 0
- budget extension does not restore reviewer
- levels execute out of order
- cached reviews consume budget
- rewind does not reset exhaustion

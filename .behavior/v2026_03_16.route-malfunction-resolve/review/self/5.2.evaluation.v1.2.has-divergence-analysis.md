### section 3: codepath check — malfunction detection

**blueprint codepath:**
```ts
const latestPerStone = getLatestStatusPerStone(passageEntries);
```

```ts
const latestByStone = new Map<...>();
for (const report of passageReports) {
  latestByStone.set(report.stone, report);
```

**divergences identified:**

| aspect | blueprint | implementation |

**settlement:** backup — inline is simpler for single usage

### section 4: codepath check — cache filter

**blueprint codepath:**
```ts
const cached = priorReviews.find(r =>
  r.hash === artifactHash &&
  r.exitClass === 'passed'
);
```

**implementation codepath:**
```ts
const cachedReviews = priorArtifacts.reviews.filter(
  (r) => r.exitClass !== 'malfunction',
);
// then later:
const found = cachedReviews.find((r) => r.artifactHash === thisHash);
```

**divergences identified:**

| aspect | blueprint | implementation |
|--------|-----------|----------------|
| filter criteria | `=== 'passed'` | `!== 'malfunction'` |
| filter placement | in find | pre-filter then find |

**hostile questions:**

1. does `!== 'malfunction'` behave differently than `=== 'passed'`?
   - YES — `!== 'malfunction'` also keeps 'constraint' reviews
   - 'constraint' = blocker found (valid review, not broken)
   - this is actually better than blueprint

2. does pre-filter affect behavior?
   - NO — same semantic result, just different structure

**settlement:** backup — implementation is more correct

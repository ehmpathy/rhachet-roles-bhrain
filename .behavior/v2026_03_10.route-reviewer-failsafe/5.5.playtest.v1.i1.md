# playtest: route reviewer failsafe

## prerequisites

- rhachet installed and linked
- a route with a guarded stone (reviews and/or judges)
- ability to create test fixtures with echo commands

---

## happy paths

### 1. reviewer exit code 0 (passed)

**setup:**
```bash
cd /tmp && mkdir -p test-playtest-1/.route
cat > /tmp/test-playtest-1/1.test.stone << 'EOF'
# test stone
EOF
cat > /tmp/test-playtest-1/1.test.guard << 'EOF'
artifacts:
  - '1.test*.md'
reviews:
  - 'echo "blockers: 0\nreview passed"'
judges: []
EOF
cat > /tmp/test-playtest-1/1.test.v1.md << 'EOF'
# test artifact
EOF
```

**action:**
```bash
npx rhx route.stone.set --stone 1.test --as passed --route /tmp/test-playtest-1
```

**expected outcome:**
- stone passes
- review artifact created at `.route/1.test.guard.review.*.md`
- artifact contains `├─ stdout` tree bucket
- artifact contains `review passed`

**setup:**
```bash
cd /tmp && mkdir -p test-playtest-2/.route

```

**action:**
```bash

```

**expected outcome:**
- stone blocked
- review artifact contains `blocked by constraints`
- review artifact contains `exit code: 2`
- CLI shows review failed with constraint message


**action:**
```bash

```

**expected outcome:**
- stone blocked by second judge
- first judge artifact shows `passed`
- second judge artifact shows `blocked by constraints`

---

### 5. route.drive halts on malfunction

**setup:**
```bash
cd /tmp && mkdir -p test-playtest-5/.route
echo '{"stone":"1.test","status":"malfunction"}' > /tmp/test-playtest-5/.route/passage.jsonl
cat > /tmp/test-playtest-5/1.test.stone << 'EOF'
# test stone
EOF
```

**action:**
```bash
npx rhx route.drive --route /tmp/test-playtest-5 --mode hook
```

**expected outcome:**
- returns exit code 3
- stdout contains "halted, reviewer/judge malfunctioned"
- stdout contains "please tell a human"
- stderr contains escalation message

## edgey paths

### empty stdout/stderr

**setup:** review that exits 1 with no output
```bash
cat > /tmp/test-playtest-edge/1.test.guard << 'EOF'
reviews:
  - 'exit 1'
EOF
```

**expected:** artifact still created with empty tree buckets, exit code visible

### very long output

**setup:** review that outputs many lines
```bash
cat > /tmp/test-playtest-edge/1.test.guard << 'EOF'
reviews:
  - 'for i in $(seq 1 100); do echo "line $i"; done; exit 2'
EOF
```

**expected:** all lines captured in stdout tree bucket

---

## pass/fail criteria

**pass if:**
- exit code 0 → artifact shows "passed", no blocked message
- exit code 2 → artifact shows "blocked by constraints"
- exit code 1 → artifact shows "blocked by malfunction"
- stdout and stderr both appear in separate tree buckets
- route.drive halts with exit 3 when passage.jsonl has malfunction
- escalation message tells human to intervene

**fail if:**
- artifacts show empty "exit code: N" with no stdout/stderr
- malfunction status does not halt route.drive
- constraint and malfunction are not distinguished
- tree bucket format is broken or absent

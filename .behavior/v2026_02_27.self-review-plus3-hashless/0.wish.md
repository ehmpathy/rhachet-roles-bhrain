wish =

today, we often get into an infiloop where the self review 

1. finds gaps
2. updates the artifact
3. gets a new hash, told to restart the review


---

this makes almost an infinite loop.

---

lets explore the following options

1. after 3 attempts, make approval hashless. i.e., if they've been challenged for approval 3 separate times due to updating the artifact 3 separate times, then they must have done a solid review already

2. always make it hashless, but increaes the timeout


---

feels like after 3 attempts hashless would strike a nice balance 

---

also, we want to ensure that if a self-review was promised, it wont get invalidated. its a firm checkpoint

---

otherwise, we have the cases where self-review.2 ends up invalidating the already approved self-review.1 (which itself already went throught the 3+ challenges)

---

the promises should be hashless always, so they dont get invalidated as other self reviews continue to improve the output


---

we want to go with the 'after 3 attempts, make the triggered watermarks hashless' and 'always make the promises hashless'

and we want to cover the boundary cases via acceptance tests

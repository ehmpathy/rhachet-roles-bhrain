# 4.1 roadmap — peer-review conversation

a checklist-style, dependency-ordered execution plan for `3.3.1.blueprint.product.yield.md`. each phase declares: what it builds, what it depends on, which briefs to read first, its behavioral acceptance criteria, and the verification command that proves each criterion.

---

## the one-commit constraint (read first)

the wisher directed that the feature + refactor A (blocker-branch de-dup) + refactor B (guard/ sub-noun cluster) ship in **ONE atomic commit**. the phases below sequence the BUILD, not the delivery: build phase-by-phase, keep the tree green at every phase boundary, then stage everything and commit ONCE at phase 8. the phase gates are the safety the split-into-PRs reviewer wanted — honored as green checkpoints on the single commit, not as separate merges.

**global invariant** — after every phase: `rhx git.repo.test --what types` stays green, and no extant acceptance snapshot changes UNLESS that phase's criteria explicitly say so (zero-diff behavior preservation for the refactors).

**build steps**:
- [ ] `guard/review/peer/getRouteGuardReviewPeerPathTaken.ts` — pure transformer: given-path string → taken-path via `.replace('._.given.by_peer.', '._.taken.by_self.')`; throw on a malformed given (no given infix)
- [ ] `guard/review/peer/enumRouteGuardReviewPeerFiles.ts` — parameterize the `given|taken` suffix (add a `kind` input); add `ignore:['**/*.report.md']` so the `*.md` glob no longer pulls the adjacent `.report.md`

**verification**:

## phase 2 — contemplation readiness computation (transformer + orchestrator)

> the shared readiness core both the single-slug and all-slug gate paths depend on.

**depends on**: phase 1 (the enumerators + path transformer)

**briefs to read first**:
- `3.1.3.research.internal.product.code.prod._.yield.md` — pattern 6 (self-review presence check), pattern 16 (verdict counts)
- blueprint design-notes B2, B7, B8 (one readiness computation; clean + nitpick-only are satisfied)

**build steps**:
- [ ] `route/contemplate/getAllRouteGuardReviewPeersUncontemplated.ts` — pure transformer: of the givens WITH blockers > 0 (clean AND nitpick-only are satisfied), those minus paired current-hash takens → each unpaired slug tagged `absent` (no taken) or `stale` (taken at a prior hash)
- [ ] `route/contemplate/getRouteGuardReviewPeerContemplationStatus.ts` — orchestrator: enum givens+takens → `{ ready, uncontemplated: {slug, tag}[] }`; accept an optional slug scope so single-slug and all-slug paths share ONE implementation; a clean/nitpick-only reviewer in single-slug scope returns the explicit satisfied variant

**acceptance criteria**:
- all blocker-carrying givens paired → `uncontemplated: []`, `ready: true`
- a nitpick-only (0 blockers, N nitpicks) reviewer is NOT uncontemplated
- an unpaired given tags `absent`; a given whose only taken sits at a prior hash tags `stale`
- single-slug scope reports only the named reviewer; all-slug scope reports every reviewer

**verification**:
- `rhx git.repo.test --what unit --scope getAllRouteGuardReviewPeersUncontemplated` → green (incl. absent-vs-stale + nitpick-only cases)
- `rhx git.repo.test --what integration --scope getRouteGuardReviewPeerContemplationStatus --mode apply` → green (all-slug AND single-slug scope)

## code flow

### before (current)

```
1. enumerate prior judge files for this hash
2. for each file:
   a. parse passed status
   b. parse reason
   c. if index valid and not seen: add to priorByIndex map
3. for each judge command:
   a. if priorByIndex has result: use cached (skip execution)
   b. else: execute judge fresh
```

### after (proposed)

```
1. enumerate prior judge files for this hash
2. for each file:
   a. parse passed status
   b. parse reason
   c. if index valid and not seen AND PASSED: add to priorByIndex map
3. for each judge command:
   a. if priorByIndex has result: use cached (skip execution)
   b. else: execute judge fresh
```

the only difference: step 2c adds `AND PASSED` condition.

## edge cases handled

| scenario | behavior |
|----------|----------|
| prior pass found | cache hit, skip execution |
| prior fail found (exit 1) | cache miss, execute fresh |
| prior block found (exit 2) | cache miss, execute fresh |
| no prior found | cache miss, execute fresh |
| multiple priors for same index | first PASSED result used |

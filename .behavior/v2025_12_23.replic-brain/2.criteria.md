# behavioral criteria for replic-brain

this behavior is split into two independent workstreams:

---

## stream 1: briefs

**focus:** knowledge, documentation, understanding

**criteria:** [2.criteria.stream1.briefs.md](./2.criteria.stream1.briefs.md)

**deliverables:**
- `src/roles/architect/readme.md`
- `src/roles/architect/briefs/brain.replic/`

**usecases:**
- usecase.1 = research and document replic brain architecture
- usecase.2 = create and maintain brain architect role
- usecase.3 = document reference blueprint for claude-code replic brain (pseudocode level)

---

## stream 2: skills

**focus:** implementation, testing, execution

**criteria:** [2.criteria.stream2.skills.md](./2.criteria.stream2.skills.md)

**deliverables:**
- `src/roles/architect/skills/brain.replic.struct1/`

**usecases:**
- usecase.1 = define executable blueprint for brain.replic.struct1 (exact spec)
- usecase.2 = test atomic components of brain structure
- usecase.3 = test integrated brain behavior
- usecase.4 = leverage rhachet for implementation

**boundaries:**
- boundary.1 = invalid inputs to brain
- boundary.2 = llm failures
- boundary.3 = tool execution failures
- boundary.4 = llm backend swapping

---

## relationship

stream 1 produces the **reference blueprint** — pseudocode-level understanding of how claude-code works

stream 2 consumes that understanding to produce the **executable blueprint** — exact spec for brain.replic.struct1

the reference blueprint is educational; the executable blueprint is implementable

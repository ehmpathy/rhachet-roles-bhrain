# blueprint: as-passed-remind-nextstep

## summary

add a reminder line to `formatRouteStoneEmit` output when a stone passes successfully (`action === 'passed'` AND `passage === 'allowed'`). the reminder tells the driver to run `rhx route.drive` to continue.

single file change: `src/domain.operations/route/formatRouteStoneEmit.ts`

## code changes

### 1. add reminder constant

```typescript
/**
 * .what = reminder to continue the route after passage
 * .why = guides driver to next step without confusion
 */
const REMINDER_LINES = [
  '   │',
  '   └─ the way continues, run',
  '      └─ rhx route.drive',
];
```

### 2. guarded stone branch (line ~161)

before:
```typescript
      return [header, '', tree].join('\n');
```

after:
```typescript
      if (input.passage === 'allowed') {
        return [header, '', tree, '', ...REMINDER_LINES].join('\n');
      }
      return [header, '', tree].join('\n');
```

### 3. unguarded stone branch (line ~287)

before:
```typescript
      } else {
        lines.push(`   ├─ stone = ${input.stone}`);
        lines.push(`   └─ passage = ${passageValue}`);
      }
```

after:
```typescript
      } else {
        lines.push(`   ├─ stone = ${input.stone}`);
        if (input.passage === 'allowed') {
          lines.push(`   ├─ passage = ${passageValue}`);
          lines.push(...REMINDER_LINES);
        } else {
          lines.push(`   └─ passage = ${passageValue}`);
        }
      }
```

## scope constraint

reminder appears ONLY when:
- `action === 'passed'`
- `passage === 'allowed'`

reminder does NOT appear for:
- `--as approved` (approval is not passage)
- `--as promised` (promise is not passage)
- `--as rewound` (rewind is backward)
- `passage === 'blocked'` (focus on blockers)
- `passage === 'malfunction'` (focus on error)

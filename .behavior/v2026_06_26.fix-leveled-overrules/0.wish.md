wish =

most guards with review peers are leveled

and often, one of the l1 guards - given the smaller brain - will post a perma blocker, even if the rest have already become terminal due to approval

in these cases, we still want to run the next level of reviewers, even if we want to decide that the l1 level of approvers is overruled

so.

we'd like to update the `overruled` and `forced` behavior to respect the leveled considerations

--

specifically

`overruled` should only apply to the current active level.

e.g., if l1 is active and l3 has not yet run, then `overruled` should only apply to l1

then, after the human `overrules` level one, upon the stone set --as passed attempt, l1 can still try and run as normal

but!

if there are any blockers, they will be overruled and they will all be considered terminal immediately

thereby, l3 can begin to finally run.


---

note that upon `--as forced`, if not all levels of reviews are going to be overruled, then it should NOT invoke --as approved, since its illogical to approve everything until all levels have been approved. (e.g., what if l3 dramatically changes the design?)

therefore, --as forced should only dispatch the approval IF we are at the terminal level of peer reviewers.

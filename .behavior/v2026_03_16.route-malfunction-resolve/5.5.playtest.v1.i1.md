# playtest: route malfunction resolve

## prerequisites

- node.js and npm installed
- repo cloned and dependencies installed (`npm ci`)
- built successfully (`npm run build`)

## sandbox

all operations target:
- `@gitroot/.temp/` for temporary files
- test fixtures already set up in `blackbox/.test/assets/route-malfunction/`

---

## behavior 1: malfunction recovery unblocks hook

### what to verify

after a stone malfunctions and is subsequently passed, the `route.drive --mode hook` should NOT halt. the latest status wins.

### happy path

1. **run acceptance tests**
   ```bash
   npm run test:acceptance:locally -- driver.route.malfunction.acceptance.test.ts
   ```

   **expected outcome:**
   - all 14 tests pass
   - output shows: `14 passed`

2. **verify test coverage via test names**
   ```bash
   npx jest --listTests --testPathPattern driver.route.malfunction
   ```

   **expected outcome:**
   - shows `driver.route.malfunction.acceptance.test.ts`

### pass/fail criteria

- ✓ pass if: all 14 tests pass, no failures
- ✗ fail if: any test fails or throws unexpected error

---

## behavior 2: reviewer malfunction cache exclusion

### what to verify

when a reviewer malfunctions, the broken review should NOT be cached. on retry, a fresh review should run.

### happy path

1. **check test case2 assertions in test file**
   ```bash
   grep -A 20 "case2.*reviewer cache exclusion" blackbox/driver.route.malfunction.acceptance.test.ts
   ```

   **expected outcome:**
   - test file contains case2 with assertions for:
     - malfunction is recorded (t0)
     - fresh review runs on retry (t1)
     - new artifact is created

2. **run the specific cache exclusion tests**
   ```bash
   npm run test:acceptance:locally -- driver.route.malfunction.acceptance.test.ts -t "case2"
   ```

   **expected outcome:**
   - all case2 tests pass
   - tests verify cache miss forces fresh review

### pass/fail criteria

- ✓ pass if: case2 tests pass, cache exclusion verified
- ✗ fail if: cached malfunction reused on retry

---

## behavior 3: detection accuracy

### what to verify

malfunction detection checks LATEST status per stone, not ANY malfunction entry.

### happy path

1. **verify test case1 asserts latest-wins semantics**
   ```bash
   grep -B 5 -A 10 "t1.*stone passes after fix" blackbox/driver.route.malfunction.acceptance.test.ts
   ```

   **expected outcome:**
   - test appends `passed` entry after `malfunction` entry
   - test asserts hook allows stop (exit 0)

2. **run malfunction recovery test**
   ```bash
   npm run test:acceptance:locally -- driver.route.malfunction.acceptance.test.ts -t "case1"
   ```

   **expected outcome:**
   - case1 tests pass
   - malfunction → pass sequence unblocks hook

### pass/fail criteria

- ✓ pass if: case1 tests pass, latest status wins
- ✗ fail if: old malfunction entry blocks despite later pass

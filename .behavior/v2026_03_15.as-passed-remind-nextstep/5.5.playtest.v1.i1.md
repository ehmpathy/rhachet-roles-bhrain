# playtest: as-passed-remind-nextstep

## prerequisites

- rhachet-roles-bhrain repo cloned
- npm dependencies installed (`pnpm install`)
- built (`npm run build`)
- api keys loaded (only for full acceptance tests): `source .agent/repo=.this/role=any/skills/use.apikeys.sh`

## sandbox

all test operations use temp directories created by jest. no manual file cleanup required.

---

## happy paths

### path 1: unguarded stone shows reminder

**action:**
```sh
cd .temp  # create temp dir
mkdir -p playtest-unguarded && cd playtest-unguarded
npx rhachet init
npx rhachet roles link --role mechanic
mkdir -p .route
echo '{"stone": "1.vision", "status": "passed"}' >> .route/passage.jsonl

# create stone and artifact
mkdir -p stones
echo '# vision' > 1.vision.md

# run the command (will invoke via skill)
rhx route.stone.set --stone 1.vision --as passed
```

**expected outcome:**
- output contains `passage = allowed (unguarded)`
- output contains `the way continues, run`
- output contains `rhx route.drive`

---

## pass/fail criteria

### pass if

1. unguarded stone with `passage = allowed` shows:
   - `the way continues, run`
   - `rhx route.drive`

2. guarded stone with `passage = allowed` shows:
   - guard tree first
   - then `the way continues, run`
   - then `rhx route.drive`

3. blocked stone does NOT show reminder

4. all 68 acceptance tests pass:
   ```sh
   source .agent/repo=.this/role=any/skills/use.apikeys.sh
   npm run test:acceptance:locally -- driver.route.set
   ```

### fail if

- reminder appears when `passage = blocked`
- reminder appears for `--as approved` or `--as rewound`
- reminder text differs from specification
- any acceptance test fails

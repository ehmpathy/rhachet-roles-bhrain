## root cause (final)

l3 peer-review guards run `$rhx enroll claude`. `$rhx` →
`<repoRoot>/node_modules/.bin/rhx` (`runStoneGuardReviews.ts` `substituteVars`), and the guard
runner prepends `<repoRoot>/node_modules/.bin` to PATH. so the spawned `claude` is the
**repo-pinned** `@anthropic-ai/claude-code`, which shadowed the global one.

bhrain pinned `2.0.76`, which hung when launched `-p` with the generated local settings. every
manual repro used the global `2.1.87` and worked — so the bug was invisible to hand-run repros
and got mis-blamed on a `link:.` self-link symlink cycle. only bhrain runs `rhx enroll claude`
in its guards at all.

## fix

- `package.json:102`: `@anthropic-ai/claude-code` `2.0.76` → `2.1.87` (matches global).
- `pnpm install` to materialize; `node_modules/.bin/claude` now exec's the `2.1.87` cli.js.
- hazard brief so the next traveler suspects a version mismatch first.

arrival of this stone runs the 5.1 guard's l3 `$rhx enroll claude` reviewers against the

# wish

test peer review budgets with multiple levels and approval scenarios

## stones

### 1.execute: exhaustion + levels (extreme case)
- level 2 reviewer (primo): expensive, runs first, budget: 2
- level 1 reviewer (cheapo): cheap, runs after level 2 terminal, budget: 2
- journey: level 2 exhausts → level 1 continues → all exhaust → human approval required

### 2.deploy: within-budget pass (no approval needed)
- single reviewer with budget: 5
- journey: reviewer passes → stone passes automatically (no approval judge)

### 3.release: within-budget + explicit approval judge
- single reviewer with budget: 5
- explicit approved? judge
- journey: reviewer passes → still blocked → human approval required

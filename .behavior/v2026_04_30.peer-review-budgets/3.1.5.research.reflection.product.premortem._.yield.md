# research: reflection — premortem

## premortem: imagine we shipped and it failed

project forward: we shipped this, and it failed badly. why?

### most plausible failure scenarios

1. **budget consumption is counted incorrectly**: budget is consumed on cache hits (should be free), on malfunction (should not count), or skipped on constraint exit (should count). drivers see wrong budget numbers and lose trust.

2. **exhaustion state corrupts across stones or routes**: exhaustion is tracked per-route but stored incorrectly. a reviewer exhausted on stone A blocks stone B even though it should reset. or worse, exhaustion leaks across routes.

3. **level order ignores terminal states**: level 2 waits forever because one level 1 reviewer is stuck (not terminal). or level 2 runs prematurely while level 1 is still active.

4. **approval confusion**: user approves to unblock exhaustion, but doesn't realize they also approved an explicit judge. or vice versa — they approve the explicit judge but the exhaustion gate still blocks.

5. **defaults break extant routes**: routes without budget/level declarations behave differently than before. backwards compatibility is broken.

---

## what obvious signs did we ignore?

- **complexity of state machine**: exhausted | approved | active across multiple reviewers, levels, and stones is a combinatorial explosion. we may have missed edge cases.

- **cache semantics interaction**: the extant cache is by hash+index. budget consumption adds another dimension (per-route per-reviewer). the intersection may have subtle bugs.

- **passage.jsonl schema drift**: we add `blocker: 'review.peer.exhausted'` but extant code reads blocker types. any switch/case that doesn't handle the new type fails silently.

---

## what assumptions could turn out wrong?

| assumption | what if wrong? | likelihood |
|------------|----------------|------------|
| exhaustion persists per route, not per stone | if stored per stone, exhaustion resets on each stone — defeats the purpose | low (design is clear) |
| `--as rewound` resets exhaustion | if rewound doesn't reset, users can't recover from exhaustion without manual state surgery | medium |
| one approval covers both exhaustion and explicit judge | if separate, users get double-prompted or one gate blocks after the other passes | medium (implementation risk) |
| level N+1 runs after all level N terminal | if "terminal" check is wrong, level N+1 runs too early or never | medium |
| inflight PR for peer review structure will land | if blocked or delayed, we can't extend the structure. must wait or work around | low (PR is approved) |
| budget not specified = unlimited | if defaults to 0, all extant routes break | low (easy to test) |
| cache hit = no budget consumption | if cache check runs after budget decrement, consumption is wrong | medium (order matters) |

---

## what is the nightmare scenario?

**the nightmare**: exhaustion state gets corrupted and persists across routes. a reviewer marked exhausted on route A stays exhausted on route B, route C, etc. — even after `--as rewound`. the only fix is manual deletion of passage.jsonl entries. every route in the repo is blocked.

**why this is bad**:
- affects all drivers, not just one
- no obvious error message — just "reviewer exhausted" on routes that never ran
- requires deep system knowledge to diagnose
- manual fix risks further corruption

**what would cause this**: exhaustion stored with wrong scope key (e.g., branch instead of route), or storage path computed incorrectly.

---

## what mitigations to consider?

| risk | mitigation | cost of mitigation |
|------|------------|--------------------|
| budget consumed on cache hit | explicit check: if cached, skip budget decrement. unit test for this path | low — clear logic |
| budget consumed on malfunction | explicit check: only decrement on exitClass === 'passed' or 'constraint'. unit test | low — clear logic |
| exhaustion corrupts across routes | scope key includes route path. integration test with two routes | medium — test setup |
| level order waits forever | terminal state = approved OR exhausted. unit test for each combination | low — clear definition |
| rewound doesn't reset exhaustion | explicit reset in setStoneAsRewound. integration test | low — one code path |
| approval confusion | single approval flag with clear message about what it covers | low — UX decision |
| defaults break extant routes | default budget = undefined (unlimited), default level = 1. acceptance test with no-budget guard | low — simple defaults |
| passage.jsonl blocker type not handled | exhaustive switch/case with default error. type system enforces | low — typescript helps |

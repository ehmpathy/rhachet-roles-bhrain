# research: internal product code patterns (prod)

## pattern 1: getAllStoneArtifacts — artifact enumeration

**[EXTEND]** — needs $route expansion added

### citation 1: current implementation

file: `src/domain.operations/route/stones/getAllStoneArtifacts.ts:8-26`

```ts
export const getAllStoneArtifacts = async (input: {
  stone: RouteStone;
  route: string;
}): Promise<string[]> => {
  // determine glob pattern from guard or default
  const globs =
    input.stone.guard?.artifacts && input.stone.guard.artifacts.length > 0
      ? input.stone.guard.artifacts
      : [`${input.stone.name}*.md`];

  // enumerate all matches across all globs
  const allMatches: string[] = [];
  for (const glob of globs) {
    const matches = await enumFilesFromGlob({ glob, cwd: input.route });
    allMatches.push(...matches);
  }

  return allMatches;
};
```

### relation to wish

this is the function that requires the fix:
1. `input.stone.guard.artifacts` is NOT expanded — `$route` is passed literally
2. `cwd: input.route` is used — violates rule.forbid.cwd-outside-gitroot
3. default pattern `${input.stone.name}*.md` lacks route prefix

## pattern 2: substituteVars in reviews — variable expansion

**[REUSE]** — same pattern for artifact expansion

### citation 2: reviews substituteVars

file: `src/domain.operations/route/guard/runStoneGuardReviews.ts:206-215`

```ts
const substituteVars = (
  cmd: string,
  vars: { stone: string; route: string; hash: string; output: string },
): string => {
  return cmd
    .replace(/\$stone/g, vars.stone)
    .replace(/\$route/g, vars.route)
    .replace(/\$hash/g, vars.hash)
    .replace(/\$output/g, vars.output);
};
```

### relation to wish

this pattern shows HOW to expand `$route` — simple regex replace. artifact globs should use the same approach (only for `$route`, as others are out of scope).

# research: internal product code (prod)

## pattern.1 = stdout/stderr capture on failure

### location
- `src/domain.operations/route/judges/runStoneGuardJudges.ts:66-96`
- `src/domain.operations/route/guard/runStoneGuardReviews.ts:47-76`

### current behavior
```ts
// when command fails, capture stdout/stderr
const { stdout, stderr } = await execa.command(judge.cmd, { reject: false, ... });

// write artifact with ---stderr--- format
const artifactContent = [
  result.stdout || '',
  '\n---stderr---\n',
  result.stderr || '',
  '\n\n---stderr---\n',
  '\n---metadata---\n',
  `exit code: ${result.code}`,
].join('');
```

### relation to wish
the wish wants tree bucket format instead of `---stderr---` format:
```
├─ stdout
│  ├─
│  │
│  │  [content]
│  │
│  └─
├─ stderr
│  ├─
│  │
│  │  [content]
│  │
│  └─
└─ passage blocked
   ├─ blocked by malfunction
   └─ exit code: 1
```

### verdict: [EXTEND]
reuse the capture logic, replace the artifact format

## pattern.4 = guard tree format

### location
- `src/domain.operations/route/guard/formatGuardTree.ts`

### current behavior
```ts
// judges formatted with command and status
lines.push(`      ${sectionIndent} ${judgePrefix} j${judge.index}: ${judge.cmd}`);
if (judge.cached) {
  lines.push(`      ${sectionIndent} ${judgeIndent} └─ · cached`);
} else {
  detailLines.push(`finished ${dur} ${mark}`);
  if (!judge.passed && judge.reason) {
    detailLines.push(`reason: ${judge.reason}`);
  }
}
```

### relation to wish
wish wants separate format for detailed judge/review output AFTER the summary:
- `🔎 review N` with tree buckets for stdout/stderr
- `🪶 judge N` with tree buckets for stdout/stderr
- current `formatGuardTree` shows summary only (cmd, duration, pass/fail)

### verdict: [EXTEND]
add new formatter for detailed output (formatReviewBucket, formatJudgeBucket)

## pattern.5 = route.drive hook behavior

### location
- `src/domain.operations/route/stepRouteDrive.ts:70-136`

### current behavior
```ts
// track block attempts, max 21
const { state } = await setDriveBlockerState({ route, stone: stone.name });

if (state.count > maxBlocks) {
  // exit 3 = tell human about stuck state
  return {
    emit: {
      stdout: formatRouteDriveExhausted(...),
      stderr: { reason: formatRouteDriveEscalate(...), code: 3 },
    },
  };
}

// block stop with exit code 2
return {
  emit: {
    stdout,
    stderr: { reason: stdout, code: 2 },
  },
};
```

### relation to wish
wish specifies different behavior for constraint vs malfunction:
- constraint (exit 2): continue to drive, guide agent
- malfunction (exit != 0, != 2): halt immediately, escalate to human

current code has exhaustion logic (21 retries) but no malfunction-specific path

### verdict: [EXTEND]
add malfunction detection and immediate escalation (no retry)

## pattern.6 = exit code handle

### location
- `src/domain.operations/route/judges/runStoneGuardJudges.ts:73-81`

### current behavior
```ts
// check if passed (exit code 0)
const passed = result.code === 0;
// any non-zero is "failed"
```

### relation to wish
wish specifies three-way exit code contract:
- 0 = passed
- 2 = ConstraintError (controlled failure)
- other = MalfunctionError (reviewer/judge broke)

current code treats all non-zero as "failed" without distinction between constraint vs malfunction

### verdict: [EXTEND]
add exit code classification: 0=passed, 2=constraint, other=malfunction

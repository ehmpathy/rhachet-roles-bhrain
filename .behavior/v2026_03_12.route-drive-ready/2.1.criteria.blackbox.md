# blackbox criteria: `--as arrived` for route.drive

## usecase.1 = clone marks stone as arrived

```
given(clone has completed a stone's artifact)
  when(clone runs `rhx route.stone.set --stone X --as arrived`)
    then(guards validate the stone)
      sothat(clone doesn't claim judgment, just arrival)
    then(if guards pass, stone is marked passed)
    then(if guards block, clone sees blocker report)
      sothat(clone knows what to fix)
```

## usecase.2 = clone sees normal guidance

```
given(clone is on a stone)
  when(route.drive hook fires, hooks without attempt < 7)
    then(output shows "are you here?" with both options)
    then(output shows `--as arrived` for "ready for review?")
    then(output shows `--as passed` for "ready to continue?")
    then(output shows stone content)
      sothat(clone knows what to do and how to proceed)
```

## usecase.3 = clone sees drum nudge after 7 hooks

```
given(clone is on a stone)
  when(route.drive hook fires, hooks without attempt >= 7)
    then(output shows drum nudge with tao te ching quote)
    then(output shows "are you here?" with both options)
    then(output shows `--as arrived` for "ready for review?")
    then(output shows `--as passed` for "ready to continue?")
    then(output shows stone content)
      sothat(clone picks whichever option resonates, both work)
```

## usecase.4 = hook counter resets on attempt

```
given(clone has triggered route.drive 10 times without attempt)
  when(clone runs `rhx route.stone.set --stone X --as arrived`)
    then(hook counter resets to 0)
      sothat(next route.drive shows normal guidance, not nudge)
  when(clone runs `rhx route.stone.set --stone X --as passed`)
    then(hook counter resets to 0)
      sothat(backwards compat resets counter too)
```

## usecase.5 = backwards compatibility

```
given(clone uses old command)
  when(clone runs `rhx route.stone.set --stone X --as passed`)
    then(stone is evaluated and marked as before)
      sothat(extant workflows continue to work)
```

## usecase.6 = approval-gated stone flow

```
given(clone is on an approval-gated stone)
  when(clone runs `--as arrived` before approval)
    then(output shows "halted, human approval required")
      sothat(clone knows to wait for human)
  when(human runs `--as approved`)
    then(approval gate is satisfied)
  when(clone runs `--as arrived` after approval)
    then(stone passes)
```

## boundary conditions

| condition | behavior |
|-----------|----------|
| hooks = 0-6 | normal guidance ("are you here?" with both options) + stone content |
| hooks >= 7 | drum nudge + "are you here?" (both options) + stone content |
| hooks >= 21 | escalate to human (extant behavior) |
| `--as arrived` | equivalent to `--as passed` |
| `--as passed` | still works (backwards compat) |
| any attempt | resets hook counter |

# research: templates — achiever role

## context

this research identifies templates from internal bhrain patterns and external sources that can guide the achiever role implementation. templates provide proven structures that accelerate development.

---

## category 1: goal schema templates

### [1] SMART goal template

**source:** doran, g. t. (1981). management review.

**type:** [academic] — foundational PM framework

**template:**
```yaml
goal:
  specific: what exactly will be accomplished?
  measurable: how will success be measured?
  achievable: is this realistic given constraints?
  relevant: why does this matter?
  time-bound: when will this be done?
```

**relevance:** validates achiever's `what.outcome` (specific), `how.gate` (measurable), `why.purpose` (relevant).

---

### [2] OKR template

**source:** grove, a. (1983). "high output management."

**type:** [book] — industry standard

**template:**
```yaml
objective: qualitative description of what to achieve
key_results:
  - quantitative measure 1
  - quantitative measure 2
  - quantitative measure 3
```

**relevance:** achiever separates `what.outcome` (objective) from `how.gate` (key result). one gate per goal (not multiple key results) keeps achiever simple.

## category 4: skill shell entrypoint templates

### [11] bhrain skill entrypoint template

**source:** `src/domain.roles/driver/skills/route.drive.sh`

**type:** [internal] — bhrain pattern

**template:**
```bash
#!/usr/bin/env bash
set -euo pipefail
exec node -e "import('package-name/cli/subpath').then(m => m.functionName())" -- "$@"
```

**key elements:**
- `#!/usr/bin/env bash` — portable shebang
- `set -euo pipefail` — fail-fast
- `exec` — replaces shell process
- `node -e` — inline javascript
- `import('package/subpath')` — isolated import
- `-- "$@"` — pass args

**relevance:** all achiever skills use this entrypoint pattern.

## convergence signals

### goal schema structure is universal

**sources [1, 2, 3, 4] agree:** goals need what (outcome), why (purpose), and how (verification). achiever's `why/what/how` nesting aligns with all frameworks.

### append-only logs are standard for audit

**sources [5, 6, 7] agree:** jsonl append-only format is standard for event capture and audit trails.

### hook patterns are established

**sources [8, 9, 10] agree:** claude code hooks provide the mechanism for triage triggers.

---

## anti-patterns from sources

### anti-pattern 1: overly complex goal schema

**sources [18, 19] warn:** autogpt's thoughts schema is complex. achiever keeps it simple: why/what/how.

### anti-pattern 2: in-memory only persistence

**sources [18, 19] warn:** early agents lost context. achiever persists to filesystem.

### anti-pattern 3: multiple key results per goal

**sources [2] note:** OKRs have 3-5 key results per objective. achiever uses one gate per goal for simplicity. decompose into multiple goals if needed.

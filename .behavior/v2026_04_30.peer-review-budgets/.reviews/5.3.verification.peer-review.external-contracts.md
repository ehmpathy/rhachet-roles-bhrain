**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.verification/rule.require.external-contract-integration-tests.md

The rule requires that every external contract, such as APIs, SDKs, or services (e.g., 'rhx review' for peer reviews), must have integration tests that call the real service with real credentials and validate responses against expected shapes. Mocks create false confidence and do not prove actual integration works.

In the codebase, peer review mechanisms are treated as external contracts (e.g., 'rhx review' commands or similar services), but the provided tests use mocked implementations instead of real calls. For instance, acceptance tests employ 'mock-review.sh' scripts, and integration tests set up guards with 'echo' commands simulating review outputs, bypassing real service interactions. No tests demonstrate calling actual external peer review services with credentials and validating real responses.

To comply, introduce at least one integration test per external contract that invokes the real service, uses authentic credentials, and asserts response shapes or contents. If exceptions are needed (e.g., due to rate limits), document them in a vision file and still include at least one real test case for the failure scenario.

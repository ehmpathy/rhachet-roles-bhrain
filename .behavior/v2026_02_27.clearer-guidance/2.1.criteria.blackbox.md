# blackbox criteria: clearer guidance on human approval blocks

## usecase.1 = clone attempts to pass a guarded stone that requires approval

```
given('a stone with an approved? guard')
  when('clone runs --as passed before human approval')
    then('judge output shows clear tree-structured guidance')
      sothat('clone knows exactly what to tell the human')
    then('guidance includes the exact --as approved command')
      sothat('human can copy-paste to approve')
    then('guidance includes the exact --as passed command')
      sothat('clone knows what to run after approval')
    then('exit code is 2 (constraint block)')
      sothat('clone knows it was intentionally blocked')
```

## usecase.2 = clone continues after human grants approval

```
given('a stone with an approved? guard')
given('human has run --as approved')
  when('clone runs --as passed')
    then('stone passes')
    then('exit code is 0')
```

## usecase.3 = progress output order

```
given('a stone with guards')
  when('clone runs --as passed')
    then('owl header appears first')
    then('progress appears directly below owl (no blank line)')
    then('blank line appears after progress')
    then('tree structure appears after blank line')
      sothat('output is visually organized and scannable')
```

## usecase.4 = judge output in stderr section

```
given('a guarded stone that fails approval')
  when('clone runs --as passed')
    then('stderr section shows judge artifact content')
    then('judge artifact includes tree-structured next steps')
    then('tree includes approval command for human')
    then('tree includes pass command for clone')
```

## usecase.5 = no guidance needed when stone passes

```
given('a stone with an approved? guard')
given('approval already granted')
  when('clone runs --as passed')
    then('no guidance section appears')
    then('output shows passage = allowed')
```

## usecase.6 = regression protection via snapshots and assertions

```
given('acceptance tests for approval-blocked scenarios')
  when('judge output format changes')
    then('snapshot tests detect the change')
      sothat('regressions to unclear output are caught')
    then('explicit assertions verify tree structure')
      sothat('specific format requirements are enforced')
```

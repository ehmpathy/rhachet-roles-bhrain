# research: remote access for init.research

## summary

the init.research skill requires access to:

1. **local filesystem** — create `.research/` directories and artifacts
2. **github api** — reference init.behavior template from rhachet-roles-bhuild (for implementation guidance)
3. **web search / llm** — probes.emit phase dispatches agents to investigate (runtime, not skill implementation)

---

## lesson.1 = filesystem access

### what

the skill creates directories and files under `.research/v{isodate}.{name}/`

### contract

standard node.js filesystem apis:
- `fs.mkdir()` — create directories
- `fs.writeFile()` — create template files
- `fs.readFile()` — read wish and probe definitions
- `fs.existsSync()` — check if directory exists (for idempotency guard)

### best practices

per repo patterns:
- use `@src/utils/filesystem` abstractions if available
- follow `.behavior/` directory patterns from init.behavior skill [1]

**[1]** from rhachet-roles-bhuild init.behavior implementation:
```
the skill creates a versioned directory with template files
each phase has a .src file with instructions
artifacts are emitted as .md files
```

## lesson.2 = github api access (template reference)

### what

the wish specifies: "use the rhachet-roles-bhuild branch of vlad/driver-bind for your template" [2]

### contract

github api via `gh` cli:
- `gh api repos/{owner}/{repo}/contents/{path}?ref={branch}` — fetch file contents
- response includes `content` (base64 encoded) and format type

### best practices

per wish: "use the gh api to access it" [2]

industry standard:
- authenticate via `gh auth` (uses GITHUB_TOKEN or oauth)
- handle rate limits (5000 requests/hour for authenticated)
- cache responses where appropriate

**[2]** from `.behavior/v2026_02_09.init-research/0.wish.md`:
```
specifically, use the rhachet-roles-bhuild branch of vlad/driver-bind for your template

use the gh api to access it
```

## lesson.3 = web search / llm access (runtime)

### what

probes.emit phase dispatches parallel agents to investigate research questions

### contract

this is runtime behavior, not skill implementation:
- agents use web search tools
- agents use llm for synthesis
- agents write findings to `probes/*.findings.md`

### best practices

per vision: parallel agents fan out, each writes to dedicated artifact [3]

**[3]** from `.behavior/v2026_02_09.init-research/1.vision.md`:
```
2. **probes.emit** — parallel agents fan out, each to investigate a probe. each writes findings to `.research/.../probes/probe-001.findings.md`, etc.
```

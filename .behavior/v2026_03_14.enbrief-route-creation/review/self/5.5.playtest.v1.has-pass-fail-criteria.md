## pass/fail criteria review

### are pass criteria specific?

**yes.** pass criteria include:
- file existence checks
- section presence checks
- property value checks
- test pass count (2/2)
- subjective clarity check (followable by someone new)

### are fail criteria observable?

**yes.** fail criteria include:
- file absent (observable via `ls` or `cat` error)
- section lacks content (observable via `grep` or manual read)
- wrong boot mode (observable via `cat boot.yml`)
- test failure (observable via test output)
- unclear content (observable via human read)

### are criteria binary?

**yes.** each criterion is checkable as yes/no:
- file exists or not
- section present or not
- property present or not
- tests pass or fail
- content clear or unclear

---

## verdict

pass/fail criteria are specific, observable, and binary. foreman can verify each item.

## feedback loops

### question: what's the slowest verification step?

**answer: acceptance tests (skill invocation via shell)**

| test type | estimated time | notes |
|-----------|----------------|-------|
| unit tests | <100ms | pure logic, fast |
| integration tests | ~1s | file I/O, genTempDir |
| acceptance tests | ~3-5s | shell invocation, rhachet link |

### question: can we tighten the loop?

**approach:** develop in layers — unit tests first, integration second, acceptance last.

1. Goal domain object → unit tests (fast feedback)
2. goal.memory.set/get operations → integration tests (medium feedback)
3. goal.infer.triage skill → acceptance tests (slow but complete)

### manual verification that could be automated

none identified — all verification paths have extant test patterns.

# criteria coverage matrix: self-review rush detection

## matrix.1 = prompt selection

this matrix covers which prompts appear based on challenge state.

| ind: challenge count | ind: rush files state | dep: shows rush prompt | dep: shows patience prompt | dep: shows lets reflect |
|---------------------|----------------------|----------------------|--------------------------|----------------------|
| first | files absent | no | yes | yes |
| first | files present (same mtime) | no | yes | yes |
| second+ | since.mtime < uptil.mtime | yes | yes | yes |
| second+ | files deleted (recovered) | no | yes | yes |

**gaps:** none — all combinations covered via edgecase.2

## matrix.2 = state lifecycle

this matrix covers how rush state changes across the review lifecycle.

| ind: action | ind: prior state | dep: new state | dep: prompt shown |
|-------------|------------------|----------------|-------------------|
| challenge (first) | files absent | both files created (same mtime) | patience + reflect |
| challenge (second+) | files present | uptil mtime updated | rush + patience + reflect |
| pass | files present | files deleted | none (continue) |
| pass | files absent | no change | none (continue) |

**gaps:** none — lifecycle is complete

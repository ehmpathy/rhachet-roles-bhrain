# blackbox criteria: as-passed-remind-nextstep

## usecase.1 = pass unguarded stone

```
given(driver has completed an unguarded stone's work)
  when(driver runs `rhx route.stone.set --stone X --as passed`)
    then(output shows passage allowed)
      sothat(driver knows the stone is complete)
    then(output shows reminder: "run rhx route.drive to continue")
      sothat(driver knows what to do next without hesitation)
```

## usecase.2 = pass guarded stone

```
given(driver has completed a guarded stone's work)
given(all guards have passed - reviews, approvals, judges)
  when(driver runs `rhx route.stone.set --stone X --as passed`)
    then(output shows passage allowed)
    then(output shows guard summary)
    then(output shows reminder: "run rhx route.drive to continue")
      sothat(driver knows what to do next after guard details)
```

## usecase.3 = pass final stone in route

```
given(driver is on the final stone of a route)
given(driver has completed the stone's work)
  when(driver runs `rhx route.stone.set --stone X --as passed`)
    then(output shows passage allowed)
    then(output shows reminder: "run rhx route.drive to continue")
      sothat(driver continues to route.drive which will show completion)
```

## usecase.4 = approve guarded stone (not passage)

```
given(driver is on a guarded stone that requires approval)
  when(driver runs `rhx route.stone.set --stone X --as approved`)
    then(output shows approval recorded)
    then(NO reminder is shown)
      sothat(driver understands approval is not passage)
```

## usecase.5 = promise self-review (not passage)

```
given(driver is completing a self-review checkpoint)
  when(driver runs `rhx route.stone.set --stone X --as promised --that Y`)
    then(output shows promise recorded)
    then(NO reminder is shown)
      sothat(driver understands promise is not passage)
```

## usecase.6 = rewind stone (not passage)

```
given(driver needs to re-review a stone)
  when(driver runs `rhx route.stone.set --stone X --as rewound`)
    then(output shows rewind recorded)
    then(NO reminder is shown)
      sothat(driver understands rewind is backward, not forward)
```

## usecase.7 = passage blocked

```
given(driver attempts to pass a stone)
given(guards have not passed - blockers present or approval absent)
  when(driver runs `rhx route.stone.set --stone X --as passed`)
    then(output shows passage blocked with reason)
    then(NO reminder is shown)
      sothat(driver focuses on clearing blockers, not next steps)
```

---

## exchange boundaries

### input
- command: `rhx route.stone.set --stone <name> --as passed`

### output (on passage allowed)
- current passage confirmation (unchanged)
- current guard summary if guarded (unchanged)
- **new**: reminder line with `rhx route.drive` command

### output (on passage blocked)
- current blocker/guard failure details (unchanged)
- **no change**: no reminder shown

### output (on non-passage actions)
- current action confirmation (unchanged)
- **no change**: no reminder shown

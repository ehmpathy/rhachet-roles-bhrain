# research: reflection — root cause

## what is the symptom?

### observable problem

**runaway review loops** that consume excessive time and API tokens without termination.

- **what is broken**: no cap on review iterations. reviewers can loop indefinitely.
- **how it manifests**: drivers see 8+ review rounds. opus reviewer burns tokens on lint issues.
- **when it occurs**: whenever blockers persist across iterations, or cheap issues consume expensive reviewers.

---

## 5 whys tree

### branch 1: runaway loops

1. **why do review loops run indefinitely?**
   → because reviewers keep found blockers and driver keeps fixed them

2. **why can't the loop stop?**
   → because there's no mechanism to cap iterations per reviewer

3. **why is there no cap?**
   → because the system assumes reviews will eventually pass

4. **why assume eventual pass?**
   → because the optimistic model has no concept of "enough attempts"

5. **why no concept of attempts?**
   → **ROOT: the system has no budget model — it's optimistically unbounded**

### branch 2: inefficient reviewer order

1. **why do expensive reviewers waste tokens on basic issues?**
   → because they run on code that still has lint/format problems

2. **why does code have basic issues when expensive reviewer runs?**
   → because cheap reviewers haven't cleared yet

3. **why haven't cheap reviewers cleared first?**
   → because there's no tier/level to order execution

4. **why no order?**
   → because all reviewers are treated as equal priority

5. **why equal treatment?**
   → **ROOT: the system has no level model — it's flat, not prioritized**

---

## root cause summary

| layer | issue | is root cause? |
|-------|-------|----------------|
| surface | drivers see 8+ rounds, high token spend | no — symptom |
| mid | no iteration cap, no execution order | no — consequence |
| deep | no budget model (optimistic convergence) | **yes** |
| deep | no level model (flat execution) | **yes** |

---

## recommendation

**address root causes directly**:

1. **add budget model**: each reviewer declares `budget: N` (max rounds). system tracks consumption per-route per-reviewer. exhaustion = terminal state.

2. **add level model**: each reviewer declares `level: N` (tier). lower levels run first. level N+1 waits until level N is terminal.

**no symptom treatment needed** — these root cause fixes eliminate the symptoms entirely:
- runaway loops → capped by budget
- inefficient order → enforced by level

**the vision addresses both root causes**. no follow-up for root cause is needed.

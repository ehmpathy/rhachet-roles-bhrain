### fixture factory detail

**what:** factory functions that create valid test fixtures with defaults.

```typescript
const createGoalFixture = (overrides?: Partial<Goal>) => ({
  slug: 'test-goal',
  why: { ask: 'test', purpose: 'test', benefit: 'test' },
  what: { outcome: 'test' },
  how: { task: 'test', gate: 'test' },
  status: { choice: 'enqueued', reason: 'test' },
  source: 'peer:human',
  createdAt: '2026-04-02',
  updatedAt: '2026-04-02',
  ...overrides,
});

const createAskFixture = (overrides?: Partial<Ask>) => ({
  hash: 'abc123',
  content: 'test ask content',
  receivedAt: '2026-04-02T10:00:00Z',
  ...overrides,
});

const createCoverageFixture = (overrides?: Partial<Coverage>) => ({
  hash: 'abc123',
  goalSlug: 'test-goal',
  coveredAt: '2026-04-02T10:05:00Z',
  ...overrides,
});
```

**benefit:**
- eliminates manual YAML construction errors
- provides sensible defaults
- enables quick customization for edge cases
- can be reused across unit/integration/acceptance tests

**priority:** medium — can be built inline as tests are developed, not a blocker.

# criteria.blackbox: route.stone.set --as rewound

## usecase.1 = post-handoff restart

```
given(a stone with stale validation after human completes handoff)
  when(human runs `rhx route.stone.set --stone <stone> --as rewound`)
    then(all guard artifacts for that stone are deleted)
      sothat(fresh reviews run on next route.drive)
    then(all guard artifacts for subsequent stones are deleted)
      sothat(downstream validation reflects the new upstream state)
    then(passage.jsonl appends {stone, status: 'rewound'} for each affected stone)
      sothat(audit trail records the rewind event)
    then(prior approvals in passage.jsonl become invalid)
      sothat(stones require fresh approval after rewind)
    then(stdout shows confirmation tree of what was cleared)
      sothat(human sees exactly what happened)
```

## usecase.2 = fix-and-retry

```
given(a stone that passed with flawed artifact)
  when(human fixes the artifact content)
    when(human runs `rhx route.stone.set --stone <stone> --as rewound`)
      then(passage marker is invalidated)
        sothat(route.drive re-evaluates the stone)
      then(guard artifacts are deleted)
        sothat(fresh reviews run against fixed artifact)
```

## usecase.3 = development iteration

```
given(a developer who iterates on guard rules)
  when(developer tweaks review prompts in guard file)
    when(developer runs `rhx route.stone.set --stone <stone> --as rewound`)
      then(guard artifacts are deleted)
        sothat(stone re-evaluates with new review logic)
```

## usecase.4 = cascade behavior

```
given(a route with stones 1, 2, 3, 4, 5)
  when(stone 3 is rewound)
    then(stones 3, 4, 5 are all rewound)
      sothat(downstream stones validated under old upstream are also invalidated)
    then(stones 1, 2 are NOT affected)
      sothat(prior work is preserved)
```

## usecase.5 = idempotent rewind

```
given(a stone that was already rewound)
  when(human runs `rhx route.stone.set --stone <stone> --as rewound` again)
    then(operation succeeds with no error)
      sothat(rewind is safe to retry)
    then(passage.jsonl still appends {stone, status: 'rewound'})
      sothat(audit trail captures the intent)
```

## usecase.6 = stone not found

```
given(a route with no stone that matches the pattern)
  when(human runs `rhx route.stone.set --stone <nonexistent> --as rewound`)
    then(error with helpful message)
      sothat(human knows the pattern did not match)
    then(no files are modified)
      sothat(fail-fast prevents partial state)
```

## usecase.7 = ambiguous pattern

```
given(a route where multiple stones match the pattern)
  when(human runs `rhx route.stone.set --stone <ambiguous> --as rewound`)
    then(error asks for more specific pattern)
      sothat(accidental bulk rewind is prevented)
    then(no files are modified)
      sothat(human must be explicit)
```

## usecase.8 = agent-initiated rewind

```
given(an agent that determines a stone needs fresh validation)
  when(agent runs `rhx route.stone.set --stone <stone> --as rewound`)
    then(rewind succeeds)
      sothat(agents can retry tasks when needed)
```

## usecase.9 = approval invalidation by log order

```
given(stone 3 was approved at time T1)
  when(stone 2 is rewound at time T2 > T1)
    then(stone 3 approval becomes invalid)
      sothat(approvals before a rewind to an earlier stone are stale)
  when(route.drive reads passage.jsonl)
    then(it treats the approval as if it never happened)
      sothat(stone 3 requires fresh approval)
```

## guard artifact types cleared

```
given(a stone with validation artifacts)
  when(rewound)
    then(*.guard.review.* files are deleted)
    then(*.guard.judge.* files are deleted)
    then(*.guard.promise.* files are deleted)
    then(*.guard.selfreview.*.triggered.* files are deleted)
    then(artifact content (the stone itself) is NOT touched)
      sothat(work is preserved, only validation state is cleared)
```

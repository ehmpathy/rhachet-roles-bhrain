# distill: experience reproductions

## experience reproductions

| experience | entry point | user actions | expected outcome | test type |
|------------|-------------|--------------|------------------|-----------|
| budget visible | `rhx route.drive` | run drive | see reviewer status with budget/level | integration |
| level order | `setStoneAsPassed` | run pass with multi-level guard | level 1 runs before level 2 | integration |
| budget decrement | `runStoneGuardReviews` | run review, then rerun | budget decrements on each run | integration |
| exhaustion trigger | `runStoneGuardReviews` | run review until budget=0 | reviewer status = exhausted | integration |
| halt on exhaustion | `setStoneAsPassed` | all reviewers terminal, one exhausted | route halts, approval required | integration |
| budget extension | `rhx route.guard.budget` | run `--add N` | budget increases, reviewer un-exhausts | acceptance |
| approval bypass | `rhx route.stone.set --as approved` | run approve | route proceeds | acceptance |

### journey 1: budget exhaustion flow

#### step table

| step | action | user sees |
|------|--------|-----------|
| t0 | before any changes | route.drive shows reviewers queued with full budget |
| t1 | run pass (round 1) | reviewer finds blockers, budget 2/3 |
| t2 | fix code, run pass (round 2) | reviewer finds blockers, budget 1/3 |
| t3 | fix code, run pass (round 3) | reviewer finds blockers, budget 0/3 → exhausted |
| t4 | run drive after exhaustion | route halted, approval required |

## critical paths

| critical path | description | why critical |
|---------------|-------------|--------------|
| budget decrement | driver sees budget count decrease on each review | trust in system, no surprise exhaustion |
| exhaustion halt | route halts when exhausted, shows clear options | human stays in control, no silent skip |
| level order | level 1 clears before level 2 runs | cost optimization, the whole point |
| approval works | human can unblock via approve | escape hatch must work |
| extension works | human can add budget | recovery path must work |
| cache skip budget | cached reviews don't consume budget | correct behavior, no false exhaustion |

## reproduction feasibility

### test utilities available

- `genTempDirForRhachet` — isolated test environment with symlinks
- `execAsync` — shell command execution
- `useThen` / `useBeforeAll` — shared test state
- `sanitizeTimeForSnapshot` — stable snapshots

### setup required

- temp directory with route and guard files
- mock reviewer commands (exit 2 for blockers, exit 0 for pass)
- passage.jsonl for state persistence

### test sketch: budget exhaustion journey

```typescript
given('[case1] route with budgeted reviewer', () => {
  const scene = useBeforeAll(async () => {
    const tempDir = genTempDirForRhachet({ slug: 'budget-test' });
    // create guard with budget: 3
    // create mock reviewer that always returns blockers
    return { tempDir };
  });

  when('[t0] before any changes', () => {
    const output = useThen('drive shows queued', async () =>
      execAsync('rhx route.drive', { cwd: scene.tempDir }),
    );
    then('budget is 3/3', () => {
      expect(output.stdout).toContain('budget: 3/3');
    });
    then('output matches snapshot', () => {
      expect(sanitizeTimeForSnapshot(output.stdout)).toMatchSnapshot();
    });
  });

  when('[t1] pass is run (round 1)', () => {
    const output = useThen('pass shows blockers', async () =>
      execAsync('rhx route.stone.set --stone 1.vision --as passed', { cwd: scene.tempDir }),
    );
    then('budget is 2/3', () => {
      expect(output.stdout).toContain('2/3');
    });
  });

  // ... t2, t3, t4
});
```

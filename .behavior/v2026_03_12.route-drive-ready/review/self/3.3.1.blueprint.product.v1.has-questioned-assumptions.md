# self-review: has-questioned-assumptions

## assumptions reviewed

### 1. "arrived" frame reduces hesitation

**assumption:** clones hesitate on "passed" because it feels like judgment.
**evidence:** documented in refs/example.case1.md — clone spins without run of the command.
**what if opposite?** if frame isn't the issue, alias won't help. but low cost to try.
**verdict:** keep. supported by root cause analysis in 3.1.5.research.reflection.product.rootcause.

### 2. 7 hooks is the right threshold

**assumption:** 7 hooks captures stuck clones without annoyance to engaged ones.
**evidence:** wisher decision after discussion. no empirical data yet.
**what if wrong?** too early = annoys. too late = clone already exhausted context.
**simpler approach?** could make configurable. but YAGNI — start with 7, tune later.
**verdict:** keep. wisher confirmed. can tune post-ship if needed.

### 3. drum nudge quote resonates with clones

**assumption:** tao te ching quote will prompt action.
**evidence:** research on nudge theory (3.1.1.research.external.product.claims).
**what if ignored?** clones parse it as decoration. but no harm — still shows command.
**verdict:** keep. low risk. worst case: clone ignores quote, sees command anyway.

### 4. hook counter separate from blocker counter

**assumption:** hook state is distinct from blocker state.
**evidence:** blocker state tracks consecutive stops. hook state tracks hooks without attempt.
**what if combined?** different purposes. blocker = review failures. hooks = general hesitation.
**verdict:** keep separate. different signals, different thresholds.

### 5. state file per stone

**assumption:** each stone has its own hook counter.
**evidence:** matches extant pattern (.route/{stone}.drive.blockers.latest.json).
**what if global?** would conflate different stones. bad.
**verdict:** keep. consistent with extant patterns.

## conclusion

all assumptions have evidence or explicit wisher confirmation. no blind assumptions found.

# blueprint: product — achiever role

## summary

build the achiever role to enable goal detection, persistence, and triage.

**core deliverables:**
1. domain objects: Goal, Ask, Coverage (DomainLiteral)
2. domain operations: setGoal, getGoals, getTriageState, setAsk, setCoverage
3. skills: goal.memory.set, goal.memory.get, goal.infer.triage
4. hooks: onTalk (accumulate asks), onStop (halt until triage complete)
5. briefs: seed brain with goal triage habits

## codepath tree

### domain.objects

```
Goal.ts
├── [+] interface Goal
│   ├── slug: string (required, all other fields optional for partial goals)
│   ├── why?: { ask?, purpose?, benefit? }
│   ├── what?: { outcome? }
│   ├── how?: { task?, gate? }
│   ├── status: { choice, reason }
│   ├── when?: { goal | event }
│   ├── source: GoalSource
│   ├── meta: { complete: boolean, absent: string[] }
│   ├── createdAt: string
│   └── updatedAt: string

├── [+] class Goal extends DomainLiteral<Goal>
│   └── public static nested = { why, what, how, status, when, meta }
│
├── [+] type GoalStatusChoice = 'blocked' | 'enqueued' | 'inflight' | 'fulfilled'

├── [+] type GoalSource = 'peer:human' | 'peer:robot' | 'self'
│
├── [+] const GOAL_REQUIRED_FIELDS = ['why.ask', 'why.purpose', 'why.benefit', 'what.outcome', 'how.task', 'how.gate']
│
└── [+] computeGoalCompleteness(goal): { complete: boolean, absent: string[] }

### domain.operations

```
setGoal.ts
├── [+] setGoal(input, context)
│   ├─ validate: slug required, all other fields optional (partial goals allowed)
│   ├─ compute: meta.complete and meta.absent via computeGoalCompleteness
│   ├─ compute: offset from parent dir mtime
│   ├─ persist: write $offset.$slug.goal.yaml (includes meta)
│   ├─ persist: write $offset.$slug.status=$choice.flag
│   ├─ persist: append to asks.coverage.jsonl if --covers
│   └─ return: { path, covered, meta }
│
└── [+] use: js-yaml for YAML serialize
```

```
getGoals.ts
├── [+] getGoals(input, context)
│   ├─ glob: *.goal.yaml in scope dir
│   ├─ parse: YAML to Goal for each
│   ├─ glob: *.status=*.flag for status.choice
│   ├─ filter: by status if specified
│   └─ return: { goals }
│
└── [+] use: js-yaml for YAML parse
```

```
getTriageState.ts
├── [+] getTriageState(input, context)
│   ├─ read: asks.inventory.jsonl
│   ├─ read: asks.coverage.jsonl
│   ├─ compute: uncovered = inventory - coverage
│   ├─ call: getGoals for extant goals
│   ├─ partition: goals into complete vs incomplete (via meta.complete)
│   └─ return: { asks, goals, goalsIncomplete, coverage }
│
└── [←] reuse: JSONL parse pattern
```

```

├── [+] setAsk(input, context)

│   ├─ construct: Ask { hash, content, receivedAt }
│   ├─ append: JSONL line to asks.inventory.jsonl
│   └─ return: { ask }
│
└── [←] reuse: JSONL append pattern (see setPassageReport)
```

```
setCoverage.ts
├── [+] setCoverage(input, context)

│   ├─ append: JSONL lines to asks.coverage.jsonl
│   └─ return: { coverage }
│
└── [←] reuse: JSONL append pattern (see setPassageReport)

## core domain focus

per self-review, the core domain is:

1. **Goal schema** — nested why/what/how forces foresight, but partial goals allowed for quick capture
2. **completeness track** — meta.complete and meta.absent enable incremental articulation
3. **coverage track** — hash → goalSlug guarantees no ask lost

invest the most care here. partial goals enable quick capture; triage reminds about incomplete goals. coverage must be complete before session ends.

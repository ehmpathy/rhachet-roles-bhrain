# blocker.1: Silent catch hides real errors (failhide)

**rule**: rule.forbid.maintenance-hazards.md

**locations**:
- blackbox/review.output-default.acceptance.test.ts:106

try/catch swallows all errors (including unexpected ones) without checking error type or rethrowing. This is a maintenance hazard that hides real errors, causes silent defects or extra debug time. Rule requires: catch must allowlist expected errors only; unexpected must rethrow; silent swallow forbidden.

**snippet**:
```ts
        try {
          // list branch directories
          const branchDirs = await fs.readdir(reviewDir);
          if (branchDirs.length > 0) {
            const branchDir = path.join(reviewDir, branchDirs[0]!);
            const files = await fs.readdir(branchDir);
            const outputFile = files.find((f) => f.endsWith('.output.md'));
            if (outputFile) {
              outputPath = path.join(branchDir, outputFile);
            }
          }
        } catch {
          // .review dir may not exist if test failed early
        }
```

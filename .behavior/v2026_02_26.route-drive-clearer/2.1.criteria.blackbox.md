# criteria.blackbox: route-drive-clearer

## usecase.1 = agent blocked from approval

```
given('an agent')
  when('agent runs `rhx route.stone.set --stone X --as approved`')
    then('command fails with clear error')
      sothat('agents cannot approve their own work')
    then('output shows "only humans can approve"')
      sothat('agent understands the constraint')
    then('output shows "please ask a human to run this command"')
      sothat('agent knows what to do next')
```

## usecase.2 = human can approve

```
given('a human')
  when('human runs `rhx route.stone.set --stone X --as approved`')
    then('command succeeds')
      sothat('humans retain approval authority')
    then('output shows "✓ approved"')
      sothat('human sees confirmation')
```

## usecase.3 = agent gets clear next-step on approval block

```
given('an agent that ran `--as passed` and was blocked by guard')
  when('guard requires human approval')
    then('hook output shows command for human to run')
      sothat('agent knows what to ask human')
    then('hook output shows command for agent to run after approval')
      sothat('agent knows their next step')
    then('output format is:
         └─ halted, human approval required
            ├─ please ask a human to
            │  └─ rhx route.stone.set --stone X --as approved
            │
            └─ once they do, run
               └─ rhx route.stone.set --stone X --as passed')
      sothat('both commands are visible and unambiguous')
```

## usecase.4 = agent continues after human approval

```
given('agent was blocked on approval and human has approved')
  when('agent runs `--as passed`')
    then('stone passes')
      sothat('route can continue')
    then('agent progresses to next stone')
      sothat('no confusion about what to do next')
```

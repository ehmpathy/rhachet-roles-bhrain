## pattern 1: route finder

**location**: `src/domain.roles/driver/skills/route.mutate.guard.sh:60-88`

**what it does**: finds bound route for current branch via `.bind.{branch}.flag` files

**key code**:
```bash
# get current branch
BRANCH=$(git rev-parse --abbrev-ref HEAD 2>/dev/null || echo "")

# sanitize branch name (same as sanitizeBranchName.ts)
BRANCH_FLAT=$(echo "$BRANCH" | sed -E \
  -e 's|/|.|g' \
  -e 's|[^a-zA-Z0-9._-]|-|g' \
  -e 's|([._-])\1+|\1|g' \
  -e 's|^[._-]+||' \
  -e 's|[._-]+$||')

# find bind flag for this branch (exclude node_modules)
BIND_FLAG=$(find . -path ./node_modules -prune -o -name ".bind.${BRANCH_FLAT}.flag" -type f -print 2>/dev/null | grep '\.route/\.bind\.' | head -n1 || echo "")

# derive route path: flag is at $route/.route/.bind.*.flag
ROUTE_DIR=$(dirname "$(dirname "$BIND_FLAG")")
ROUTE_DIR="${ROUTE_DIR#./}"
```

**action**: [REUSE] — this pattern correctly derives route directory from bind flag location

## pattern 3: protection evaluation (file tools)

**location**: `src/domain.roles/driver/skills/route.mutate.guard.sh:149-167`

**what it does**: evaluates whether file path matches protected patterns for Read/Write/Edit tools

**key code**:
```bash
# file-based tools (Read/Write/Edit)
if [[ -n "$FILE_PATH" ]]; then
  # check if file is within bound route
  if echo "$FILE_PATH" | grep -q "$ROUTE_DIR"; then
    # check protected patterns
    if echo "$FILE_PATH" | grep -qE "\.stone$"; then
      TARGET_PATH="$FILE_PATH"
      MATCH_REASON="*.stone"
    elif echo "$FILE_PATH" | grep -qE "\.guard$"; then
      TARGET_PATH="$FILE_PATH"
      MATCH_REASON="*.guard"
    elif echo "$FILE_PATH" | grep -qE "\.route/"; then
      TARGET_PATH="$FILE_PATH"
      MATCH_REASON=".route/**"
    fi
  fi
fi
```

**problem**: line 161 (`grep -qE "\.route/"`) matches ANY path that contains `.route/`, not just the metadata subdirectory within the route.

**example**:
- `.route/xyz/artifact.md` — `.route/` at START → should ALLOW (route directory)
- `.route/xyz/.route/passage.jsonl` — `.route/` in MIDDLE → should BLOCK (metadata)
- `.behavior/xyz/.route/passage.jsonl` — `.route/` in MIDDLE → should BLOCK (metadata)

**action**: [EXTEND] — replace simple grep with logic that checks if path is in `$ROUTE_DIR/.route/` subdirectory

**fix approach**:
```bash
# check if path is in the metadata subdirectory
elif echo "$FILE_PATH" | grep -qE "^${ROUTE_DIR}/\.route/"; then
  TARGET_PATH="$FILE_PATH"
  MATCH_REASON=".route/**"
fi
```

the key insight: check for `$ROUTE_DIR/.route/` prefix, not just `\.route/` anywhere in path.

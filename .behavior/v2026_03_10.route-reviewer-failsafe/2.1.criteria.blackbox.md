# blackbox criteria: route reviewer failsafe

## usecase.1 = reviewer output capture

given a reviewer runs via route.stone.set
  when the reviewer exits with code 0
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then passage proceeds
      sothat reviewers that pass allow the stone to progress

  when the reviewer exits with code 2
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then the artifact shows "passage blocked"
    then the artifact shows "blocked by constraints"
    then the artifact shows "exit code: 2"
      sothat agents see clear constraint failure

  when the reviewer exits with code != 0 and != 2
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then the artifact shows "passage blocked"
    then the artifact shows "blocked by malfunction"
    then the artifact shows "exit code: [code]"
      sothat agents see clear malfunction failure

## usecase.2 = judge output capture

given a judge runs via route.stone.set
  when the judge exits with code 0
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then passage proceeds

  when the judge exits with code 2
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then the artifact shows "passage blocked"
    then the artifact shows "blocked by constraints"
    then the artifact shows "exit code: 2"
      sothat agents see why the judge blocked passage

  when the judge exits with code != 0 and != 2
    then the artifact shows stdout in a tree bucket
    then the artifact shows stderr in a tree bucket
    then the artifact shows "passage blocked"
    then the artifact shows "blocked by malfunction"
    then the artifact shows "exit code: [code]"
      sothat agents see the judge malfunctioned

## usecase.3 = malfunction escalation

given a reviewer or judge malfunctions (exit code != 0 and != 2)
  when route.stone.set completes
    then passage.jsonl records {"stone": "...", "status": "malfunction"}
      sothat malfunction state is persisted

  when route.drive runs after malfunction
    then route.drive tells driver to halt
    then route.drive tells user "reviewer/judge malfunctioned, human must fix"
      sothat humans are immediately aware and can intervene

## usecase.4 = constraint failure guidance

given a reviewer or judge fails by constraint (exit code 2)
  when route.drive runs after constraint failure
    then route.drive continues to fire
    then route.drive guides agent toward fix
      sothat agents can self-correct constraint failures

## usecase.5 = tree bucket format

given any reviewer or judge output
  when the artifact is rendered
    then stdout appears in format:
      ```
      ├─ stdout
      │  ├─
      │  │
      │  │  [stdout content]
      │  │
      │  └─
      ```
    then stderr appears in format:
      ```
      ├─ stderr
      │  ├─
      │  │
      │  │  [stderr content]
      │  │
      │  └─
      ```
      sothat output is visually scannable and consistent

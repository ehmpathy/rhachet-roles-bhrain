# blueprint: multi-stone delete

## summary

enable `route.stone.del` to accept multiple `--stone` flags and delete all matched stones in one operation.

## contracts

### input contract

```ts
// before
stepRouteStoneDel(input: {
  stone: string;
  route: string;
  mode: 'plan' | 'apply';
})

// after
stepRouteStoneDel(input: {
  stones: string[];  // at least one required
  route: string;
  mode: 'plan' | 'apply';
})
```

### output contract

```ts
// before
{
  pattern: string;
  patternRaw: string;
  deleted: string[];
  ...
}

// after
{
  patterns: { glob: string; raw: string }[];
  deleted: string[];
  ...
}
```

### cli contract

```sh
# before (single)
rhx route.stone.del --stone 3.1 --mode apply

# after (multi)
rhx route.stone.del --stone 3.1 --stone 3.2 --stone 2.3 --mode apply
```

## deduplication strategy

1. collect all patterns from `stones[]` input
2. for each pattern: filter stones via `isStoneInGlob`
3. collect matched stone names into `Set<string>` for dedupe
4. iterate deduped set for deletion/classification

this ensures:
- same stone matched by multiple patterns → deleted once
- output shows each stone once
- count reflects actual operations

## happy paths

### path 1: guard with large artifact glob completes without OOM

**context**: the OOM occurred when guards declared `artifacts = ["src/**/*"]` which matched 700+ files. the fix uses git blob hashes instead of file reads.

**step 1**: create a test route with a large artifact glob

```bash
mkdir -p .temp/playtest-oom
cat > .temp/playtest-oom/1.vision.stone << 'EOF'
test stone for playtest
EOF

cat > .temp/playtest-oom/1.vision.guard << 'EOF'
artifacts = ["src/**/*"]
EOF
```

**step 2**: enumerate how many files match the glob

```bash
find src -type f | wc -l
```

**expected**: 700+ files (proves the glob is large enough to trigger prior OOM)

**step 3**: compute hash without OOM

```bash
node -e "
const { getGitBlobHashes } = require('./dist/domain.operations/route/guard/getGitBlobHashes');
const { enumFilesFromGlob } = require('@ehmpathy/globs');
(async () => {
  const files = await enumFilesFromGlob({ glob: 'src/**/*' });
  console.log('files:', files.length);
  const start = process.memoryUsage().heapUsed;
  const hashes = await getGitBlobHashes({ files });
  const end = process.memoryUsage().heapUsed;
  const growth = (end - start) / 1024 / 1024;
  console.log('heap growth:', growth.toFixed(2), 'MB');
  console.log('hashes:', Object.keys(hashes).length);
})();
"
```

**expected**:
- files: 700+ (matches glob)
- heap growth: < 100 MB
- hashes: same as files count

**pass criterion**: completes without OOM, heap growth < 100MB

### path 3: hash is deterministic

**context**: same content should produce same hash, regardless of order or call sequence.

**step 1**: compute hash twice

```bash
node -e "
const { getGitBlobHashes } = require('./dist/domain.operations/route/guard/getGitBlobHashes');
(async () => {
  const files = ['src/index.ts', 'package.json'];
  const hash1 = await getGitBlobHashes({ files });
  const hash2 = await getGitBlobHashes({ files });
  console.log('hash1:', JSON.stringify(hash1));
  console.log('hash2:', JSON.stringify(hash2));
  console.log('equal:', JSON.stringify(hash1) === JSON.stringify(hash2));
})();
"
```

**expected**:
- hash1 and hash2 are identical
- equal: true

**pass criterion**: identical hashes on repeated calls

## edgey paths

### edge 1: modified file gets fresh hash

**context**: files modified in worktree (not yet committed) should get a hash via `git hash-object`, not the stale `git ls-tree` hash.

**step 1**: check current hash of a file

```bash
git ls-tree HEAD src/index.ts
```

**step 2**: modify the file (append newline)

```bash
echo "" >> src/index.ts
```

**step 3**: compute hash via getGitBlobHashes

```bash
node -e "
const { getGitBlobHashes } = require('./dist/domain.operations/route/guard/getGitBlobHashes');
(async () => {
  const hashes = await getGitBlobHashes({ files: ['src/index.ts'] });
  console.log('hash:', hashes['src/index.ts']);
})();
"
```

**step 4**: compare with git hash-object

```bash
git hash-object src/index.ts
```

**expected**: the hash from step 3 matches the hash from step 4 (worktree content), NOT the hash from step 1 (committed content)

**step 5**: restore the file

```bash
git checkout src/index.ts
```

**pass criterion**: modified file hash matches `git hash-object`, not `git ls-tree`

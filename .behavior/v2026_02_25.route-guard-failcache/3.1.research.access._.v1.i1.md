# research: remote access requirements

## summary

this feature requires **no remote access** to databases or external APIs. all operations are local filesystem only.

---

## access inventory

### 1. local filesystem (cache files)

**location:** `.route/` directory within the route folder

**operations:**
- read: check for cached judge results
- write: persist successful judge results
- delete: (not required for this feature)

**contract:** node.js `fs` module
- `fs.readFile(path, 'utf-8')` → string | throws
- `fs.writeFile(path, content)` → void | throws
- `fs.access(path)` → void | throws (for existence check)

**best practices within this repo:**
- use async/await fs operations (fs/promises)
- handle ENOENT gracefully (file not found = cache miss)
- handle EACCES gracefully (permission denied = cache miss)
- handle JSON parse errors gracefully (corrupted = cache miss)

### 2. subprocess execution (judge commands)

**what:** execute shell commands for judge evaluation

**contract:** node.js `child_process` module
- `exec(command)` or `spawn(command)` → { exitCode, stdout, stderr }

**best practices within this repo:**
- capture exit code without throw
- capture stdout and stderr separately
- set reasonable timeout (prevent hung judges)
- use exec for simple commands, spawn for stream output

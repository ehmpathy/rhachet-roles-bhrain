## pattern 2 — guard command variable substitution `[EXTEND]`

peer `run` commands are expanded before exec.

[3] `runStoneGuardReviews.ts:673-698` — the full substitution set:
```
const substituteVars = (
  cmd: string,
  vars: {
    stone: string;
    route: string;
    hash: string;
    output: string;
    repoRoot: string;
  },
): string => {
  ...
  return cmd
    .replace(/\$stone/g, vars.stone)
    .replace(/\$route/g, vars.route)
    .replace(/\$hash/g, vars.hash)
    .replace(/\$output/g, vars.output)
    .replace(/\$rhx/g, rhxPath)
    .replace(/\$rhachet/g, rhachetPath);
};
```

**correction to vision**: the actual substituted tokens are `$stone $route $hash $output $rhx $rhachet` — `$rhx`/`$rhachet` **are** `$`-substituted here (to `node_modules/.bin` absolute paths), *in addition* to reachability via `PATH` ([4] `runStoneGuardReviews.ts:157-160`). the vision's parenthetical that they are "PATH-only, not `$`-substituted" is slightly wrong and should read "both". this is a `[SUMP]`-turned-`[FACT]` to reconcile in the vision.

**relation to wish**: the wish's opt-in `$conversation` is a new token in exactly this set.
**[EXTEND]**: add `.replace(/\$conversation/g, conversationGlob)` and thread a `conversation` value into the `vars` object. absent the token in a reviewer's `run`, behavior is unchanged (opt-in by construction).

## pattern 5 — treestruct halt formatters `[REUSE]` pattern, `[EXTEND]` new formatter

the self-review halt is a pure line-builder; the peer halt should be a counterpart of it.

[10] `src/domain.operations/route/guard/formatPatienceFriend.ts:62-65` — the final instruction shape:
```
  lines.push(`   └─ when you've truly reflected, run`);
  lines.push(
    `      └─ rhx route.stone.set --stone ${input.stone} --as promised --that ${input.slug}`,
  );

[11] `src/domain.operations/route/guard/formatWalkTheWay.ts:20-22` — the "articulate into" pointer already exists:
```

  lines.push(`   │  ├─ articulate into`);
  lines.push(`   │  │  └─ ${articulationPath}`);

**relation to wish**: the vision's "reviewers await your reply" stdout is a peer counterpart of `formatPatienceFriend`, and its per-reviewer `contemplate from` / `articulate into` pair is a two-line extension of the extant `articulate into` block [11].
**[REUSE]** the line-builder pattern and the `articulate into` label verbatim.
**[EXTEND]** a `formatRouteGuardReviewPeerContemplatePrompt` that renders one block per peer reviewer (slug + verdict + `contemplate from` given-path + `articulate into` taken-path), closed with `--as contemplated --that ${slug}`. per the three-surface rule, a single shared formatter must feed the stophook, `--as arrived`, and `--as passed`.

**parity note**: the self-review `challenge:absent` message today does NOT print a symmetric read/write pair — the wish asks to carry the same crystal-clear treatment (and the "guard assigns the level, copy verbatim" caution) to it. that is an `[EXTEND]` of `formatPatienceFriend`/`formatWalkTheWay`.

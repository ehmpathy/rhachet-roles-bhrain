# blackbox criteria: guard-selfreview-better

## usecase.1 = clone encounters self-review checkpoint

```
given('a clone reaches a self-review checkpoint')
  when('they attempt to pass the stone')
    then('they see the warm reflection prompt')
      sothat('self-review is framed as sanctuary, not tollbooth')
    then('they see stillness message with tea ritual')
    then('they see before-you-begin instructions')
    then('they see the review guide')
    then('they see after-you-finish instructions')
    then('they see the promise command to run')
    then('system records triggered timestamp')
      sothat('time enforcement can be applied later')
```

## usecase.2 = clone promises after sufficient time

```
given('a clone has seen the self-review prompt')
  when('they run the promise command after 90+ seconds')
    then('the promise is accepted')
```

## usecase.3 = clone promises too quickly

```
given('a clone has seen the self-review prompt')
  when('they run the promise command before 90 seconds')
    then('they receive the patience challenge')
    then('they see "the pond barely rippled"')
    then('they see the "truly?" reflection questions')
    then('they see "the pond awaits" message')
    then('the promise is blocked')
      sothat('they must wait and reflect')

  when('they run the promise command again after 90 seconds')
    then('the promise is accepted')
```

## usecase.4 = clone restarts session mid-review

```
given('a clone has seen the self-review prompt')
  when('they restart their session')
    then('triggered timestamp persists in .route/ files')
      sothat('timer continues across sessions')
  when('they run the promise command')
    then('time is measured from original trigger')
```

## usecase.5 = hash changes mid-review

```
given('a clone has seen the self-review prompt for hash A')
  when('artifacts change and hash becomes B')
    then('original triggered timestamp is invalidated')
    then('clone must trigger review again for new hash')
      sothat('reviews are always for current state')
```

## usecase.6 = clone runs promise without trigger

```
given('a clone has NOT seen the self-review prompt')
  when('they run the promise command directly')
    then('they receive the patience challenge')
    then('they see the lets reflect prompt')
    then('system records triggered timestamp')
    then('the promise is blocked')
      sothat('they must see the review first and wait')
```

## exchange contracts

### trigger exchange

```
input:
  - route.stone.set --stone $stone --as passed

output (when self-review guard exists):
  - warm reflection prompt with:
    - stillness message
    - before-you-begin instructions
    - review guide
    - after-you-finish instructions
    - promise command
  - triggered timestamp recorded to .route/
```

### promise exchange

```
input:
  - route.stone.set --stone $stone --as promised --that $slug

output (time >= 90s):
  - promise accepted

output (time < 90s):
  - patience challenge displayed
  - promise blocked
```

## bonus: remove timestamps and dead code

```
given('promise artifacts may have timestamp fields')
  when('setStoneAsPromised writes a promise')
    then('no timestamp field is written')
    then('no triggered_at field is written')
    then('no duration_seconds field is written')
      sothat('artifacts are minimal and deterministic')

given('drive blocker has dead .jsonl history')
  when('setDriveBlockerState runs')
    then('no .jsonl file is written')
    then('DriveBlockerEvent is deleted')
      sothat('dead code is removed')

given('DriveBlockerState has timestamp field')
  when('setDriveBlockerState runs')
    then('no since field is written')
      sothat('state is minimal')
```

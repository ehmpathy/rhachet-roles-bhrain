# blueprint: fix-achiever

> no ask forgotten, no goal abandoned

---

## summary

implement 7 wish items to improve the achiever role:

1. **briefs boot** — briefs already exist, boot.yml already boots them. add onBoot hook to refresh after compaction
2. **scope auto-detect** — fail-fast if `--scope repo` while bound to route
3. **skill headers** — rewrite with recommended patterns (flags one-by-one)
4. **escalation** — replicate DriveBlockerState for goal blockers, escalate onStop messages after 5 blocks
5. **onBoot hook** — add `--when hook.onBoot` mode to goal.triage.next
6. **arg validation** — fail-fast on unknown flags, unknown yaml keys, invalid status values
7. **help output** — comprehensive `--help` with examples and best practices

## key contracts

### arg validation error format

```
🦉 patience, friend.

🔮 goal.memory.set
   ├─ ✋ unknown flag: --foo
   │
   └─ allowed flags
      ├─ --slug           goal identifier
      ├─ --scope          where to persist (route | repo) — automatic, rarely needed
      ├─ --status.choice  status (incomplete | blocked | enqueued | inflight | fulfilled)
      ├─ --status.reason  reason for status
      ├─ --covers         comma-separated ask hashes
      ├─ --why.ask        the original ask
      ├─ --why.purpose    why this matters
      ├─ --why.benefit    what success enables
      ├─ --what.outcome   expected result
      ├─ --how.task       work to be done
      ├─ --how.gate       success criteria
      ├─ --source         origin (peer:human, peer:system)
      └─ --help           show this help
```

### escalation message format

```
# count < 5 (gentle)
🦉 to forget an ask is to break a promise. remember.

🔮 goal.triage.next --when hook.onStop
   └─ inflight (N)
      └─ (1)
         ├─ slug = <slug>
         ├─ why.ask = <the ask>
         ├─ status = inflight → ✋ finish this first
         └─ tip: run `rhx goal.memory.get --slug <slug>` to see the goal

# count >= 5 (escalated)
🦉 friend, you have been reminded 5 times. the work must be done.

🔮 goal.triage.next --when hook.onStop
   └─ inflight (N)
      └─ (1)
         ├─ slug = <slug>
         ├─ why.ask = <the ask>
         ├─ status = inflight → ✋ finish this first
         └─ tip: if blocked, run:
            └─ rhx goal.memory.set --slug <slug> --status.choice blocked --status.reason "..."
```

### help output format

```
🦉 goal.memory.set — persist a goal

🔮 usage (recommended — flags one-by-one)
   │
   │  rhx goal.memory.set \
   │    --slug fix-login-bug \
   │    --why.ask "fix the login bug" \
   │    --why.purpose "users cannot access the app" \
   │    --why.benefit "users can log in again" \
   │    --what.outcome "login works without errors" \
   │    --how.task "debug auth flow, fix the issue" \
   │    --how.gate "login test passes" \
   │    --status.choice inflight \
   │    --status.reason "start now" \
   │    --source peer:human
   │
   ├─ required fields
   │  ├─ --why.ask        the original ask from human
   │  ├─ --why.purpose    why this matters
   │  ├─ --why.benefit    what success enables
   │  ├─ --what.outcome   expected result
   │  ├─ --how.task       work to be done
   │  └─ --how.gate       success criteria
   │
   ├─ optional fields
   │  ├─ --slug           goal identifier (auto-generated if absent)
   │  ├─ --status.choice  incomplete | blocked | enqueued | inflight | fulfilled
   │  ├─ --status.reason  reason for current status
   │  ├─ --covers         comma-separated ask hashes
   │  ├─ --source         peer:human | peer:system
   │  └─ --scope          route | repo (automatic — rarely needed)
   │
   └─ example: status update
      │
      │  rhx goal.memory.set \
      │    --slug fix-login-bug \
      │    --status.choice fulfilled \
      │    --status.reason "fixed in commit abc123"
      │

note: stdin yaml is allowed but not recommended.
      flags one-by-one increases focus on each component.
```

## bonus fix (folded in per wisher request): snapshot permadrift + always-on --updateSnapshot

two compounded bugs, surfaced when I rationalized the snapshot churn, now FIXED in this PR:

### bug 1 тАФ always-on --updateSnapshot (the defect that hid the drift)

`package.json:59-63` used unquoted `$([ -n $RESNAP ] && echo '--updateSnapshot')`. with `RESNAP`
unset, `[ -n $RESNAP ]` collapses to `[ -n ]` тЖТ POSIX TRUE тЖТ `--updateSnapshot` always on, in CI
too. snapshots were rewritten, never compared, so drift could never fail. this is why the
committed drift (from PR #277, branch `fix-bump-reviewer-timeout`) never errored on CI.

**verified upstream**: declapract-typescript-ehmpathy already prescribes the correct form
(`$([ "${RESNAP:-}" = "true" ] && echo '--updateSnapshot')`) and ships a brief
`rule.require.safe-shell-vars.md` + a declapract test for exactly this. bhrain was on an older
declapract snapshot. **fix**: adopted the prescribed safe form (quote + `= "true"` + `--ci` +
`set -eu`) for all 4 test scripts (`test:unit`, `test:integration`, `test:acceptance:locally`,
`test:acceptance`).

### bug 2 тАФ content non-determinism (the drift source)

route snapshots baked in the absolute worktree path + timestamp + random hash via unsanitized
nested fields. **fix**:
- `runStoneGuardJudges` + `runStoneGuardReviews` integration tests: sanitize nested `stone.path`
  (and `stdout` for the `$route`/`$rhx`/`$rhachet` cases) тЖТ `<route>` / `<repo-root>`.
- gitignore unit + integration tests (peer + self): replaced `.rejects.toMatchSnapshot()` with
  `getError` + `error.message.replace(tempDir, '<route>')` so the timestamped temp path in error
  messages no longer leaks.

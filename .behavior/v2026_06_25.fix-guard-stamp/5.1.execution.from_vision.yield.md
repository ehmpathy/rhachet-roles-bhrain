## design decisions (from vision + self-review)

- **single choke-point, not 5 scattered writes** — a local `stampGuardReport(emit)` helper closes over `peerReviews` + `stoneMatched`; each branch does `emit: await stampGuardReport({...})`. one definition of the write, honors `rule.forbid.duplicate-format-tree-operations`.
- **scope = peer reviews only** — the helper no-ops when `peerReviews.length === 0`, so judges-only / artifacts-only guards leave no stamp (faithful to the wish).
- **reuse `emit`, never re-format** — the stamp writes the already-composed `emit.stdout` (+ `emit.stderr` under a `🪶 stderr` divider). honors single-source-of-render.
- **upsert** — `fs.writeFile` overwrites; the stamp always reflects the latest run.
- **path = route root** — `path.join(route, '{stone}.stamp')`, unsealed + git-tracked (verified: only `.route/` subdir is sealed/ignored).

## reviewer scope fix (guard mutation, human-allowed)

root cause of the peer-review noise: the guards ran reviewers with `--diffs since-main`, which resolves to `git diff $(git merge-base origin/main HEAD)` — and this branch sits **far ahead of main** (prior unmerged PRs #266/#258/#251/#210/#184). so the reviewers evaluated the **entire branch divergence** (~800 lines of prior merged orchestration), not this session's ~40-line stamp change.

**fix (human allowed guard mutation):** changed all peer reviewers in both `5.1.execution.from_vision.guard` and `5.3.verification.guard` from `--diffs since-main` → `--diffs since-commit`. per `enumFilesFromDiffs.ts`, `since-commit` = `git diff HEAD` + staged + untracked = **only this session's uncommitted work**. this scopes reviewers to the actual change.

**residual (inherent limit):** in `--mode hard` (push), reviewers inject **full file content** of each changed file (`compileReviewPrompt.ts:53`). since i genuinely edited `setStoneAsPassed.ts`, its full content — including prior decode-friction from earlier PRs — stays visible to review. that prior debt is a separate concern (its own refactor behavior), not introduced by the stamp work; handled here via human overrule.

**outcome:** stone `5.1.execution.from_vision` **passed** (`passage = allowed`, judge ✓) via human overrule.

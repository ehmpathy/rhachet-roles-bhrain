## matrix.5 = scope (route vs repo)

| ind: in route | ind: branch | ind: --scope | dep: persistence location | dep: error |
|---------------|-------------|--------------|---------------------------|------------|
| yes           | any         | route        | $route/.goals/            | none       |
| no            | feature     | repo         | .goals/$branch/           | none       |
| no            | main        | repo         | -                         | main forbidden |

**notes:**
- goals on main branch are forbidden (repo-scoped)
- route-scoped goals persist to $route/.goals/
- repo-scoped goals persist to .goals/$branch/

## matrix.6 = goal lifecycle (status transitions)

| ind: current status | ind: intent | dep: new status | dep: status.reason |
|---------------------|-------------|-----------------|---------------------|
| enqueued            | start work  | inflight        | updated             |
| inflight            | complete    | fulfilled       | updated (evidence)  |
| inflight            | block       | blocked         | updated (blocker)   |
| blocked             | unblock     | enqueued        | updated             |
| any                 | no change   | unchanged       | unchanged           |

**notes:**
- fulfilled requires verification evidence in status.reason
- blocked requires when.goal or when.event
- transitions are explicit via --status

# domain distillation: route --as blocked

## usecases

| usecase | actor | goal | contract |
|---------|-------|------|----------|
| signal block | robot | escalate to human | `stepRouteStoneSet({ as: 'blocked' })` |
| see blocked | human | know where to help | `stepRouteDrive()` shows blocked status |
| detect blocked | hook | trigger automation | read `passage.jsonl` for `status: 'blocked'` |
| suggest escalation | hook | nudge stuck robot | `route.drive --mode hook` shows `--as blocked` when blockers > 5 |

## how operations compose

### flow: robot signals block

1. robot calls `stepRouteStoneSet({ as: 'blocked', stone, route })`
2. `setStoneAsBlocked` delegates to `getBlockedChallengeDecision`
3. decision tree:
   - no `.triggered` file → `challenge:first` → create `.triggered`, show nudge
   - `.triggered` exists but no articulation → `challenge:absent` → show nudge again
   - `.triggered` + articulation → `allowed` → record to passage.jsonl
4. return emit with stdout

### flow: route.drive shows blocked

1. human calls `stepRouteDrive({ route })`
2. reads passage.jsonl, finds blocked stone
3. emits stdout with `status = blocked` and path to articulation

### flow: route.drive suggests escalation

1. hook calls `stepRouteDrive({ route, mode: 'hook' })`
2. if blockers > 5, emit includes `are you blocked?` with `--as blocked` suggestion

### regression fixed during this gate

- on first full integration run, 12 failures surfaced in `stepReview.integration.test.ts [case10]` (rules through a symlink): `--rules glob was ineffective`.
- root cause: the `git ls-files` gitignore filter applied to **supplies** dropped symlinked non-.agent rule files (globby returns the symlink path; git returns the target path; they did not match → intersection dropped them).
- fix: supplies no longer apply gitignore at all. supplies are explicitly pointed at via --rules/--refs, so they load whatever the glob matches; the glob walk's `suppressErrors` still prevents the EACCES crash. removed the `{ enabled, except }` config; `gitignore?: boolean` again. case10 (prior, established symlink-fix behavior) now passes — it proved the simpler design correct.
- supplies boundary tests rewritten to the corrected contract; vision doc (1.vision.yield.md) synced.

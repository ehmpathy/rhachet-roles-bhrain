# root cause analysis: enbrief route creation

## what is the symptom?

**observable problem**: users who want to create routes must reverse-engineer extant routes, guess at conventions, and trial-and-error their way to success.

**how it manifests**:
- time wasted on discovery
- malformed routes that don't work
- inconsistent conventions across routes
- questions in chat/support channels

**when it occurs**: whenever someone new to routes wants to create one.

---

## 5 whys tree

### branch 1: knowledge gap

1. **why do users struggle to create routes?**
   → because they don't know the conventions (file names, structure, commands)

2. **why don't they know the conventions?**
   → because the conventions are not documented in one place

3. **why aren't the conventions documented?**
   → because the route system was built incrementally; docs lagged implementation

4. **why did docs lag implementation?**
   → because route creation was rare; creators learned by pair work with extant authors

5. **why is pair work no longer sufficient?**
   → because the user base is broader; not everyone has access to an extant author

   **→ potential root cause: documentation was adequate for small team, not for broader adoption**

### branch 2: discoverability

1. **why do users not find extant routes helpful?**
   → because routes are scattered across repos; conventions vary

2. **why do conventions vary?**
   → because no authoritative specification exists

3. **why is there no specification?**
   → because the system evolved from practice; conventions emerged, weren't prescribed

4. **why weren't conventions prescribed?**
   → because prescribe too early risks wrong abstraction; observe-then-document is preferred

5. **why is now the right time to document?**
   → because conventions have stabilized; enough routes exist to extract patterns

   **→ potential root cause: conventions were emergent, now stable enough to document**

---

## root cause summary

| layer | issue | is root cause? |
|-------|-------|----------------|
| symptom | users struggle to create routes | no |
| layer 1 | conventions not documented | no |
| layer 2 | docs lagged implementation | no |
| layer 3 | small team didn't need docs | no |
| layer 4 | broader adoption requires docs | **yes** |
| layer 5 | conventions now stable enough | **yes** |

**root cause**: the route system matured from internal-only (where pair work transferred knowledge) to broader adoption (where documentation is required). conventions are now stable enough to document.

---

## recommendation

**address root cause**: create the documentation brief that captures route creation conventions.

this is a direct fix for the root cause:
- conventions exist and are stable → can document them
- broader adoption requires docs → brief fills that gap
- single source of truth → reduces inconsistency

no symptom-only treatment. this is root cause address.

---

## conclusion

the wish directly addresses the root cause. no shortcuts or workarounds required.

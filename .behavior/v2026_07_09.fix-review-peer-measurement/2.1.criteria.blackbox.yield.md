# 2.1 blackbox criteria — fix review peer measurement

> experience boundaries only. what a human observes when they run peer reviews with raw
> enrolled reviewer brains — and what the guard's per-review measurement accepts and returns.
> no mechanism names, no types, no implementation.

the subject has two observable surfaces:

- **episode** = a human drives a stone through peer review and watches the guard's verdict.
- **exchange** = the guard measures one reviewer's output → a blocker/nitpick tally, or a
  malfunction when no verdict can be read.

---

## episode experience

# usecase.1 = a conforming reviewer passes without any fallback

given('a stone arrived for peer review')
  given('a reviewer enrolled as a raw brain cli')
    when('the reviewer emits a review that states its counts as numbers (e.g. `0 blockers`, `2 nitpicks`)')
      then('the guard reads those counts and classifies the review')
        sothat('the stone proceeds or blocks on the review substance, as before')
      then('the reviewer line shows NO `tallied by ...` branch')
        sothat('the human sees the common path stayed silent — no fallback occurred')
      then('no extra brain cost is incurred for this reviewer')
        sothat('conforming reviews stay free, as today')

# usecase.2 = a non-conforming reviewer is rescued by the sub-brain fallback

given('a stone arrived for peer review')
  given('a reviewer enrolled as a raw brain cli that does NOT state numeric counts')
    when('the reviewer exits successfully but phrases its verdict in prose (e.g. "no blockers, a couple of nits")')
      then('the guard still obtains a blocker/nitpick tally for the review')
        sothat('a well-reviewed stone is no longer blocked merely for odd phrasing')
      then('the reviewer line shows a `tallied by reviewer@$brain` branch below the nitpick row')
        sothat('the human is never surprised the count came from a fallback, and sees which brain tallied it')
      then('the stone proceeds or blocks on those extracted counts')
        sothat('the verdict reflects the review substance, not its format')

# usecase.3 = a review with no readable verdict fails loud

given('a stone arrived for peer review')
  given('a reviewer enrolled as a raw brain cli')
    when('the reviewer exits successfully but its output carries no verdict at all (empty / garbage)')
      then('neither the exact-match read nor the fallback can produce a tally')
      then('the guard marks the reviewer as a malfunction and blocks the stone')
        sothat('a review that never truly happened can never masquerade as a clean pass')
      then('the human sees a malfunction, not a silent approval')
        sothat('failloud is preserved — the fix never widens into a failhide')

# usecase.4 = a crashed reviewer is not rescued

given('a stone arrived for peer review')
  given('a reviewer enrolled as a raw brain cli')
    when('the reviewer exits with a non-zero code (a real failure, not odd phrasing)')
      then('the fallback does NOT run for that reviewer')
        sothat('a genuine failure is never papered over by a brain-extracted count')
      then('the reviewer is treated as failed, as before')

# usecase.5 = the human can always tell the deterministic path from the fallback path

given('a stone arrived for peer review with multiple enrolled reviewers')
  given('one reviewer states numeric counts and another writes prose')
    when('the guard finishes measuring all reviewers')
      then('the numeric reviewer shows no `tallied by ...` branch')
      then('the prose reviewer shows a `tallied by reviewer@$brain` branch')
        sothat('the presence of the branch is the whole signal — its absence means "read verbatim"')
      then('the deterministic and fallback renders are visually distinct and stable across runs')
        sothat('a reviewer can vibecheck the guard output and trust what the counts came from')

## exchange experience

the atomic contract: the guard measures ONE reviewer's output. input = the review text plus
its exit code; output = a verdict-bearing tally, or an explicit "no verdict" that becomes a
malfunction. the fallback is an internal detail of *how* the tally is obtained — the exchange
only promises the tally is correct and its provenance is reported.

# usecase.6 = numeric counts are read verbatim

given('a review whose text declares numeric counts for both dimensions')
  when('the guard measures the review')
    then('it returns the declared blocker count and nitpick count')
      sothat('the reviewer\'s own numbers are honored exactly')
    then('it reports the counts as read verbatim (no fallback used)')

# usecase.7 = prose counts are extracted by the fallback

given('a review that reports its findings only in prose, with no numeric counts')
  when('the guard measures the review')
    then('it returns a blocker count and nitpick count consistent with the prose')
      sothat('the tally reflects what the reviewer actually reported')
    then('it reports that the tally came from the fallback, and which brain produced it')
      sothat('the provenance travels with the counts, not hidden from view')

# usecase.8 = a clean review is a declared zero, distinct from "no verdict"

given('a review that clearly reports a clean pass (no issues)')
  when('the guard measures the review')
    then('it returns blockers=0 and nitpicks=0 as a DECLARED verdict')
      sothat('a real clean review is approved, correctly')
    then('this declared 0/0 is distinguishable from the "no verdict found" outcome')
      sothat('an honest clean pass is never confused with an unreadable review')

# usecase.9 = an unreadable review returns "no verdict", never a fabricated zero

given('a review whose text carries no discernible verdict')
  when('the guard measures the review')
    then('it returns an explicit "no verdict found" outcome')
      sothat('the caller cannot read a fake 0/0 from an absent verdict')
    then('the "no verdict" outcome carries zero count fields to misread')
      sothat('failhide is structurally impossible at the measurement boundary')

# usecase.10 = boundary — mentioned-then-cleared counts read as the final verdict

given('a review that mentions earlier issues in prose then declares them all cleared (final 0/0)')
  when('the guard measures the review')
    then('it returns the FINAL declared verdict, not the mentioned-in-passing numbers')
      sothat('a resolved-issues narrative is not misread as unresolved blockers')

# usecase.11 = boundary — the fallback reads a small review, never the reviewed subject

given('a reviewer whose review is a small self-contained artifact about a large subject')
  when('the guard measures the review via the fallback')
    then('the fallback reads only the reviewer\'s review text, never the large subject repo')
      sothat('the extraction stays cheap — the expensive read already happened in the reviewer')

## edge cases in the narrative

- **mixed reviewers in one stone** — some numeric, some prose; each measured independently,
  each marked (or not) per how its own tally was obtained (usecase.5).
- **fallback itself fails** (brain error / timeout) — treated as a malfunction for that
  reviewer, never a silent pass (extends usecase.3).
- **over-confident fallback** — a fallback that fabricates a plausible verdict on garbage is a
  hazard the "no verdict found" outcome + boundary tests must guard against; the exchange
  promises the tally is *correct*, so adversarial inputs (usecase.9, usecase.10) are in-scope
  for verification.
- **cost visibility** — when the fallback runs, its use is observable (the `tallied by` branch);
  the human can always see when brain cost was incurred vs the free deterministic path.

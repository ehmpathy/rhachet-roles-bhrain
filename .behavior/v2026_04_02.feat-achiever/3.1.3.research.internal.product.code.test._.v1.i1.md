## category 1: test file conventions

### [1] jest config test match pattern [REUSE]

**source:** `jest.unit.config.ts`, `jest.integration.config.ts`

**pattern:**
```typescript
// jest.unit.config.ts
testMatch: [
  '**/*.test.ts',
  '!**/*.acceptance.test.ts',
  '!**/*.integration.test.ts',
],

// jest.integration.config.ts
```

**test file name convention:**
- `*.test.ts` — unit tests (pure logic, no I/O)

**relevance:** achiever tests follow this convention. Goal domain object gets `Goal.test.ts`, persistence gets `*.integration.test.ts`, skill invocation gets `*.acceptance.test.ts`.

**disposition:** [REUSE] — use exact convention

## category 5: acceptance test patterns

### [9] invokeRouteSkill pattern [REUSE]

**source:** `blackbox/.test/invokeRouteSkill.ts`

**pattern:**
```typescript
export const invokeRouteSkill = async (input: {
  skill: string;
  args: Record<string, string>;
  cwd: string;
}): Promise<{ code: number; stdout: string; stderr: string }> => {
  const skillPath = path.join(input.cwd, '.agent/repo=bhrain/role=driver/skills', `${input.skill}.sh`);
  const argsStr = Object.entries(input.args)
    .map(([k, v]) => `--${k} "${v}"`)
    .join(' ');
  const cmd = `bash "${skillPath}" ${argsStr}`;
  try {
    const { stdout, stderr } = await execAsync(cmd, { cwd: input.cwd });
    return { code: 0, stdout, stderr };
  } catch (error: any) {
    return { code: error.code ?? 1, stdout: error.stdout ?? '', stderr: error.stderr ?? '' };
  }
};
```

**relevance:** achiever acceptance tests invoke skills via shell for realistic testing.

**disposition:** [EXTEND] — adapt for achiever skills (goal.infer.triage, goal.memory.set, goal.memory.get)

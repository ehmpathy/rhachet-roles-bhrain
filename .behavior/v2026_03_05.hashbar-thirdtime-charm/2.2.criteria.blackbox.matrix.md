# blackbox criteria matrix: 30s cooldown, self-trust after one change

## matrix 1: promise decision

| ind: count vs hashbar | ind: time elapsed | dep: promise result |
|-----------------------|-------------------|---------------------|
| count <= hashbar      | < 30s             | rejected            |
| count <= hashbar      | >= 30s            | accepted            |
| count > hashbar       | < 30s             | rejected            |
| count > hashbar       | >= 30s            | accepted            |

**note**: time elapsed is measured from the newest triggered file (which varies based on timer behavior below)

---

## matrix 2: timer behavior on edit

| ind: count vs hashbar | ind: edit made | dep: timer behavior |
|-----------------------|----------------|---------------------|
| count <= hashbar      | no             | unchanged           |
| count <= hashbar      | yes            | resets              |
| count > hashbar       | no             | unchanged           |
| count > hashbar       | yes            | persists            |

**interpretation**:
- count <= hashbar: edit resets timer (must wait full 30s again)
- count > hashbar: edit doesn't reset timer (self-trust)

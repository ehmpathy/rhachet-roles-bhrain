# blocker.1: failhide in isDirectory function

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.maintenance-hazards.md

**locations**:
- src/domain.operations/review/enumFilesFromGlob.ts

The try/catch block in the isDirectory function catches all errors and returns false, which swallows unexpected errors like permission issues or other filesystem problems. This violates the failhide rule by hiding real errors that could indicate bugs or issues, rather than allowlisting only expected errors (e.g., ENOENT for file not found) and rethrowing unexpected ones. Change to allowlist expected errors and rethrow others for proper error handling.

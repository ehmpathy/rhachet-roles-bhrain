### buttonup fixes made this round (detect ‚Üí fix, zero omissions)

   had let a *reasoned deferral* pass the `isProgressArticulated` guard ‚Äî the guard checks the
   articulation is real, not that the capture is complete, so a well-argued "defer for now" slipped
   through. the fix is guidance-level, not a heavier deterministic guard (capture-completeness is a
   semantic/LLM judgment the guard cannot make). added an anti-deferral clause to
   `im_an.obsessive_learner.for.domain.terms.md` (".capture now ‚Äî a settled term is not deferrable")
   and two step-5 sub-branches to the shared distill guide in `learn.ts`: a term the round **settled**
   (declared dobj/dop, judged synonym) is captured that round; deferral is valid only for a term whose
   authorship cannot finish now. both faces render it via the one shared body.
2. **deflaked the one non-green test** (reflect role, out of scope) rather than dismiss it ‚Äî see the
   integration row. the fix follows the repo's own `rule.require.repeatable-for-llm-tests` and an
   extant same-dir pattern, so it is a convention-conformant reliability repair, not a scope grab.

# blueprint: route --as blocked

## summary

implement `--as blocked` for `route.stone.set` to allow robots to signal escalation to humans. the feature follows a two-step flow (trigger вҶ’ articulate вҶ’ confirm) that forces reflection before escalation.

key deliverables:
1. `setStoneAsBlocked` operation with challenge flow
2. `getBlockedChallengeDecision` decision logic
3. blocked-specific stdout format with nudge
4. route.drive shows blocked stones and suggests `--as blocked` when blockers > 5

### setStoneAsBlocked (new)


### getBlockedChallengeDecision (new)

```

  в”ңв”Җ [+] call getBlockedTriggeredReport

  в”ңв”Җ [+] if no articulation вҶ’ return challenge:absent
  в””в”Җ [+] return allowed
```

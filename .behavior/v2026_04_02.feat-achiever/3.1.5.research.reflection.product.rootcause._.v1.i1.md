# research: root cause — achiever role

## the symptom

robot brains forget multi-part requests.

human says: "fix the test and update the readme"
brain does: fixes the test, forgets the readme

this happens reliably. why?

---

## 5 whys

### why do brains forget multi-part requests?

**because:** goals exist only in context memory, and context compresses.

when context compresses, goals that were "understood" but not explicitly tracked get lost. the brain had them, then it didn't.

### why do goals exist only in context memory?

**because:** brains have no external goal storage.

the brain's "memory" is the conversation history. when that history compresses or the session ends, the goals vanish. there's no file, no list, no persistence.

### why is there no external goal storage?

**because:** goal track was never designed as a system feature.

brains were built to respond to input, not to remember obligations. the architecture assumed each message is independent — no carry-forward of commitments.

### why was goal track never designed in?

**because:** early agent design focused on single-turn interaction.

the llm paradigm started with "prompt → response." multi-turn conversation was bolted on, but goal persistence wasn't. each session starts fresh.

### why did single-turn dominate?

**because:** that's what the technology supported initially.

llms began as completion engines. the concept of "robot brain with goals" is newer. the infrastructure for goal persistence simply wasn't built.

---

## the root cause

**goal memory was never designed into brain architecture.**

brains have:
- conversation memory (compresses)
- tool access (no goal primitives)
- instructions (no goal enforcement)

brains lack:
- external goal storage
- goal enumeration
- goal verification

the symptom (forgotten asks) is downstream of the root cause (no goal system).

---

## why achiever fixes this

achiever adds what's absent:

| absent | achiever adds |
|--------|---------------|
| external storage | `.goals/` directory |
| goal enumeration | asks.inventory.jsonl with content hash |
| goal verification | gates in Goal schema |
| forced articulation | nested why/what/how fields |
| coverage check | asks.coverage.jsonl |

the fix for "brains forget goals" is to give brains a goal system.

---

## why patches fail

### patch: "just tell it to remember"

briefs that say "remember all asks" don't work because:
- context still compresses
- no enforcement mechanism
- no way to verify coverage

### patch: "use a todo list tool"

external todo tools don't work because:
- brain must choose to use them
- no hook to enforce
- no schema to force clarity

### patch: "shorter sessions"

short sessions reduce but don't eliminate the problem:
- even short sessions can have multi-part asks
- human workflow suffers
- doesn't address root cause

---

## the structural fix

achiever is structural, not behavioral:

| approach | type | failure mode |
|----------|------|--------------|
| "remember asks" brief | behavioral | brain can ignore |
| todo list tool | opt-in | brain can skip |
| achiever hooks | structural | brain cannot bypass |

the onStop hook that halts until triage is complete is the key mechanism. the brain cannot end the session without address each ask. that's structural enforcement.

---

## conclusion

**root cause:** goal memory was never designed into brain architecture.

**fix:** achiever adds explicit goals, external persistence, and enforced enumeration.

**why it works:** structural enforcement via hooks, not behavioral suggestion via briefs.

the 5 whys lead to a clear conclusion: brains need a goal system. achiever provides one.

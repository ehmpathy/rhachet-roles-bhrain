# 3.1.3 internal research — prod codepath patterns

> production codepaths only (test paths excluded). each pattern tagged [REUSE] / [EXTEND] /
> [REPLACE], with cited line numbers and verbatim quotes.

the fix is surgical: swap one deterministic parse for a two-tactic waterfall, thread a brain
context down an extant call chain, and add one observability branch to the guard's tree.
below are the exact prod patterns that make that possible.

---

## pattern 1 — the deterministic count parser (the current sole tactic)

**[1]** `src/domain.operations/route/guard/getReviewCountsFromContent.ts:39-59`
> ```ts
> export const getReviewCountsFromContent = (input: {
>   content: string;
> }): { detected: boolean; blockers: number; nitpicks: number } => {
> ```
> line 52: `const detected = blockerCount !== undefined && nitpickCount !== undefined;`
> lines 56-57: `blockers: blockerCount ?? 0,` / `nitpicks: nitpickCount ?? 0,`

- **[EXTEND]** this becomes the **deterministic leaf tactic**. its detection logic (LAST-match,
  both-dimensions-required) is kept verbatim. only its **return shape** changes from the flat
  `{ detected, blockers, nitpicks }` to the discriminated union `{ detected: false } |
  { detected: true; blockers; nitpicks }` — which removes the `?? 0` fiction on lines 56-57.
- relation to wish: this IS the "deterministic tactic" the wish says to try first.

### net shape of the change

- **[REUSE]** 4 patterns (brain maker, schema-ask, verdict computer, tree mechanics) — the fix
  stands on proven prod scaffolding, invents no new infra.
- **[EXTEND]** 6 patterns, but 5 of them are *additive* (widen a type, forward a context, push
  one tree branch). only pattern 1's return-shape change ripples to a caller (pattern 2's
  destructure) — and that ripple is a *welcome* compile error that enforces the union.
- **[REPLACE]** none. no pattern is torn out; the deterministic parser is preserved as tactic #1.

### infra dependency note

- **[FACT]** the only new runtime dependency is a keyrack-backed brain credential
  (`ehmpath / prep`), fetched lazily by `genContextBrain` with auto-unlock
  (`review.ts:183-185`). no new external service, no new package — `rhachet` +
  `rhachet-brains-fireworksai` are already installed.

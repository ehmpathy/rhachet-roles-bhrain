## divergence analysis

### blueprint → implementation divergences

| blueprint declared | actual implementation | divergence type | resolution |
|--------------------|----------------------|-----------------|------------|
| separate .cli.ts files in skills/ | consolidated in src/contract/cli/goal.ts | changed | **accepted** — follows extant pattern |
| im_a.bhrain_owl.md symlink | file created (not symlink) | minor | functionally equivalent |
| hooks: onStop | implemented via Role.build() | none | n/a |
| hooks: onTalk | implemented via init workaround | none | n/a |
| inits/ directory | created for onTalk workaround | added | **accepted** — bypasses rhachet abstraction limit |

### divergence resolution

**CLI consolidation (accepted):**
the blueprint declared separate `.cli.ts` files per skill. implementation consolidated into `src/contract/cli/goal.ts`. this follows the extant pattern for the reviewer role and enables isolated subpath exports (`rhachet-roles-bhrain/cli/goal`). functionally equivalent, cleaner structure. **no repair needed.**

**onStop hook (implemented):**
the onStop hook is implemented via Role.build(). it invokes `goal.infer.triage --mode hook.onStop` which halts the session with exit code 2 if uncovered asks exist. this enforces goal triage before session ends.

**onTalk hook (implemented via workaround):**
rhachet's Role.build() only supports `onBoot`, `onTool`, `onStop`. the vision required `onTalk` (UserPromptSubmit in Claude Code terms). solution: bypass the abstraction via init executable.

- **init workaround:** `inits/init.claude.hooks.sh` directly modifies `.claude/settings.json` to add UserPromptSubmit hook
- **hook handler:** `inits/claude.hooks/userpromptsubmit.ontalk.sh` accumulates asks to inventory on each user message
- **run:** `npx rhachet roles init --role achiever` installs the hook into settings.json

both hooks are fully implemented. onTalk uses a workaround that bypasses rhachet's abstraction to directly configure Claude Code settings.

**inits/ directory (accepted addition):**
the blueprint did not declare an `inits/` directory. this was added to implement the onTalk workaround. the init executable pattern is the standard rhachet mechanism for role-specific configuration that exceeds Role.build() scope.

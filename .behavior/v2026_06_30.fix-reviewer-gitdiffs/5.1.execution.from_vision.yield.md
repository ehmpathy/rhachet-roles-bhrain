## environment artifacts (not code defects)

1. **getGitBlobHashes.test.ts case6** fails in a full unit run in this worktree: it does
   `git ls-files` + `git hash-object` over the whole src tree; the retired `enumFilesFromDiffs.*`
   files are deleted in the worktree but not yet added to the index, so git still lists them →
   hash-object on a gone file fails. clears once the deletions are added/committed. `git.stage.add`
   is **globally blocked** (`git.commit.uses --global block`), so the human must clear it at commit time.

2. **snapshot footgun**: `rhx git.repo.test --what unit --scope name://X` loads all suites but runs
   only name-matched tests; jest then removes "obsolete" snapshots of the skipped suites. this wiped
   3 unrelated route gitignore snapshots (restored from HEAD). **use `path://` scope, not `name://`,
   for unit runs** — path scope only loads matching files and is snapshot-safe.

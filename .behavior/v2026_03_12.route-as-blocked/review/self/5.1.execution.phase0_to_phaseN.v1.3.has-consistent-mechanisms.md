# self-review: has-consistent-mechanisms

## question

do new mechanisms duplicate extant functionality?

## findings

### 1. blocked/ follows promise/ structure

| promise/ | blocked/ | consistent? |
|----------|----------|-------------|
| getSelfReviewChallengeDecision.ts | getBlockedChallengeDecision.ts | yes |
| getSelfReviewTriggeredReport.ts | getBlockedTriggeredReport.ts | yes |
| setSelfReviewTriggeredReport.ts | setBlockedTriggeredReport.ts | yes |
| setStoneAsPromised.ts | setStoneAsBlocked.ts | yes |

both directories follow the same organizational pattern. this is intentional reuse of the structure.

### 2. challenge decision patterns differ appropriately

**getSelfReviewChallengeDecision** (extant):
- has time logic (30s threshold)
- tracks attempts, hashbar, plowthrough
- returns `'challenge:rushed'` for re-attempts

**getBlockedChallengeDecision** (new):
- no time logic (articulation IS the reflection)
- simpler: triggered → articulation → allowed
- no rush detection needed

the difference is intentional. self-review uses time to force reflection. blocked uses articulation file to force reflection.

### 3. triggered report patterns differ appropriately

**getSelfReviewTriggeredReport** (extant):
- returns `{ sinceMtime, uptilMtime, attempts }`
- complex because it tracks time and attempts

**getBlockedTriggeredReport** (new):
- returns `{ triggered: boolean, path: string }`
- simple because it only checks existence

the difference is intentional. blocked flow only needs existence check.

### 4. delStoneGuardArtifacts reuse

setStoneAsPassed now calls delStoneGuardArtifacts for cleanup, same as setStoneAsRewound. this is correct reuse of extant functionality.

### 5. clearBlockedTriggersUpTo pattern

new helper in setStoneAsPassed clears blocked triggers for all stones up to the passed stone (inclusive). this follows the cascade pattern from setStoneAsRewound (which clears artifacts for all stones at and after the rewound stone).

## verdict

no duplication found. new mechanisms follow extant patterns and structure. differences are intentional based on different behavioral requirements.

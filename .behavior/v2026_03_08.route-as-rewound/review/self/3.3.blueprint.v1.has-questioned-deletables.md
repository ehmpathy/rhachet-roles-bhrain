# self-review: has-questioned-deletables

## reflection

I paused and re-read the blueprint line by line. for each component I asked: "if this didn't exist and we had to add it, would we?" this forced me to justify each piece.

---

## component: delStoneGuardArtifacts

**can this be removed?** no.

we need a way to delete guard artifacts. the alternative is inline code in setStoneAsRewound, but extraction keeps things clean and testable.

**simplest version?** yes — it's just a glob + unlink loop. no extra complexity.

## component: computeSubsequentStones

**can this be removed?** yes — inline it.

this is a simple filter operation: `stones.filter(s => stoneIndex(s) >= stoneIndex(target))`. no need for a separate function. inlined in setStoneAsRewound.

**action**: remove from blueprint, inline the logic.

## component: getStonePassageStatus update

**can this be removed?** no.

the approval invalidation logic must live here — where passage status is interpreted.

**could we simplify?** the pseudocode is more complex than needed. the actual logic:
1. find last rewind in log
2. if no rewind → return normal status
3. if rewind found → check if it invalidates approvals

but wait... the cascade already writes `{stone, status: 'rewound'}` for EACH affected stone. so we don't need complex index comparison. we just need: "if there's a rewind entry for this stone AFTER its approval, the approval is invalid."

**action**: simplify the pseudocode to reflect this simpler approach.

## component: acceptance test file

**can this be removed?** no.

acceptance tests are required per the stone prompt. but... do we need a SEPARATE file? could it be added to an extant acceptance test?

**checked**: `driver.route.set.acceptance.test.ts` exists and tests `--as passed`, `--as approved`, `--as promised`. add `--as rewound` there to keep related tests together.

**action**: update blueprint to add to extant file rather than create new one.

## component: 10 self-review slugs

**can this be removed?** this is about the guard file itself, not the blueprint.

the guard has 10 self-review steps. that's the stone's guard design, not a thing the blueprint can change.

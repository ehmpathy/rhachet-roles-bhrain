# playtest: $route expansion in artifact globs

## prerequisites

- repo cloned with this branch checked out
- `pnpm install` completed
- `npm run build` completed

## sandbox

all file operations target `.temp/playtest-route-expansion/`

## happy paths

### 1. verify $route expands in guard artifact pattern

```bash
# create temp route directory
mkdir -p .temp/playtest-route-expansion

# create a minimal route with guard that uses $route
cat > .temp/playtest-route-expansion/1.test.stone << 'EOF'
test stone
EOF

cat > .temp/playtest-route-expansion/1.test.guard << 'EOF'
reviews: []
judges: []
artifacts:
  - "$route/1.test.*.md"
EOF

# create the artifact at the expanded path
echo "test artifact" > .temp/playtest-route-expansion/1.test.output.md

# run getAllStoneArtifacts via the test
npm run test:unit -- src/domain.operations/route/stones/getAllStoneArtifacts.test.ts
```

**expected outcome:**
- tests pass
- [case4] verifies $route expansion works

### 2. verify default pattern includes route prefix

```bash
# run the same test file
npm run test:unit -- src/domain.operations/route/stones/getAllStoneArtifacts.test.ts
```

**expected outcome:**
- tests pass
- [case5] verifies default pattern `${input.route}/${stone.name}*.md` works

### 3. verify acceptance test for artifact expansion

```bash
npm run test:acceptance -- blackbox/driver.route.artifact-expansion.acceptance.test.ts
```

**expected outcome:**
- all 4 test cases pass
- case1/case2 both find artifacts via $route expansion

## edgey paths

### no $route in pattern

```bash
# artifact patterns without $route should still work
npm run test:unit -- src/domain.operations/route/stones/getAllStoneArtifacts.test.ts
```

**expected outcome:**
- [case1] and [case2] pass (non-$route patterns)
- patterns are used as-is from repo root

### $route appears multiple times

this edge case is covered by the regex `/\$route/g` (global flag).

```bash
# verify via grep that global replacement is used
grep -n '/\$route/g' src/domain.operations/route/stones/getAllStoneArtifacts.ts
```

**expected outcome:**
- line shows `glob.replace(/\$route/g, input.route)`

## pass/fail criteria

- **pass if:**
  - unit tests in `getAllStoneArtifacts.test.ts` pass
  - acceptance tests in `driver.route.artifact-expansion.acceptance.test.ts` pass
  - grep confirms global replacement pattern

- **fail if:**
  - any test fails with "artifact not found"
  - $route is not expanded (literal `$route/` in glob)
  - cwd is still set on enumFilesFromGlob call

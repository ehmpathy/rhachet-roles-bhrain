# blueprint: peer-review-budgets

## summary

implement per-reviewer budget limits and level-based execution order for peer reviews.

**key deliverables**:
1. upgrade `RouteStoneGuardReviewPeer` from `string` to structured object with `slug`, `run`, `budget`, `level`
2. add `RouteStoneGuardReviewPeerMeter` to track rounds consumed per reviewer
3. execute reviewers by level (lower levels first = cheap before expensive, level N+1 waits for level N terminal)
4. halt route when any reviewer exhausted, prompt human for budget extension or approval
5. add `route.guard.budget` skill for budget extension

**terminal verdicts**: `approved | exhausted`

### journey test structure

the acceptance test exercises complete journeys with exhaustive snapshots at each phase:

**journey 1: exhaustion → approval**

| phase | command | snapshot |
|-------|---------|----------|
| [t0] | `rhx route.drive` | reviewers queued, budget `3/3`, verdict `queued` |
| [t1] | `rhx route.stone.set --as passed` | review runs, rounds `1/3`, verdict `rejected` |
| [t2] | `rhx route.stone.set --as passed` | review runs, rounds `2/3`, verdict `rejected` |
| [t3] | `rhx route.stone.set --as passed` | review runs, rounds `3/3`, verdict `exhausted` |
| [t4] | `rhx route.drive` | halted, exhaustion prompt, asks for approval or budget |
| [t5] | `rhx route.stone.set --as approved` | approved, journey complete |

**journey 2: exhaustion → budget extension → continuation**

| phase | command | snapshot |
|-------|---------|----------|
| [t0-t4] | same as journey 1 | exhaustion triggered |
| [t5] | `rhx route.guard.budget --add 2 --peer cheapo` | budget extended `3 → 5` |
| [t6] | `rhx route.drive` | no longer exhausted, verdict `rejected`, rounds `3/5` |
| [t7] | `rhx route.stone.set --as passed` | review runs, rounds `4/5`, verdict `rejected` |
| [t8] | `rhx route.stone.set --as passed` | review passes, verdict `approved` |
| [t9] | `rhx route.drive` | stone passed, journey complete |

**journey 3: level-based execution order**

| phase | command | snapshot |
|-------|---------|----------|
| [t0] | `rhx route.drive` | level 1 reviewers queued, level 2 awaits |
| [t1] | `rhx route.stone.set --as passed` | level 1 runs, level 2 still awaits |
| [t2] | (level 1 approves) | level 1 terminal, level 2 unlocked |
| [t3] | `rhx route.stone.set --as passed` | level 2 runs |

**journey 4: multi-level mixed terminal + budget extension**

setup: 3 reviewers
- r1 (slug: `linter`, level 1, budget 2) — will exhaust
- r2 (slug: `spellcheck`, level 1, budget 3) — will approve
- r3 (slug: `architect`, level 2, budget 2) — awaits level 1, will exhaust then extend

| phase | command | snapshot |
|-------|---------|----------|
| [t0] | `rhx route.drive` | r1+r2 queued, r3 awaits level 1 |
| [t1] | `rhx route.stone.set --as passed` | r1: rounds `1/2` rejected, r2: rounds `1/3` rejected, r3 awaits |
| [t2] | `rhx route.stone.set --as passed` | r1: rounds `2/2` **exhausted**, r2: rounds `2/3` rejected, r3 awaits |
| [t3] | `rhx route.stone.set --as passed` | r1: exhausted, r2: rounds `3/3` **approved**, r3 **unlocked** and runs |
| [t4] | `rhx route.drive` | r1 exhausted, r2 approved, r3: rounds `1/2` rejected |
| [t5] | `rhx route.stone.set --as passed` | r3: rounds `2/2` **exhausted** |
| [t6] | `rhx route.drive` | halted, r1+r3 exhausted, asks for approval or budget |
| [t7] | `rhx route.guard.budget --add 2 --peer architect` | r3 budget extended `2 → 4` |

## domain object changes

### RouteStoneGuardReviewPeer (upgrade)

```typescript
// before: string
export type RouteStoneGuardReviewPeer = string;

// after: structured
export interface RouteStoneGuardReviewPeer {
  slug: string;
  run: string;
  budget: number;
  level?: number;    // default: 1 (higher levels run first)
}

export class RouteStoneGuardReviewPeer
  extends DomainLiteral<RouteStoneGuardReviewPeer>
  implements RouteStoneGuardReviewPeer
{
  public static unique = ['slug'] as const;
}
```

### RouteStoneGuardReviewPeerMeter (new)

```typescript
export interface RouteStoneGuardReviewPeerMeter {
  reviewer: RefByUnique<typeof RouteStoneGuardReviewPeer>;  // { slug }
  rounds: number;  // rounds consumed
}

export class RouteStoneGuardReviewPeerMeter
  extends DomainLiteral<RouteStoneGuardReviewPeerMeter>
  implements RouteStoneGuardReviewPeerMeter {}
```

### Verdict derivation (not stored)

verdict is computed at guard evaluation time:

| condition | verdict |
|-----------|---------|
| `meter.rounds === 0` | `queued` |
| `meter.rounds > 0 && blockers > 0 && meter.rounds < reviewer.budget` | `rejected` |
| `blockers === 0` | `approved` |
| `meter.rounds >= reviewer.budget && blockers > 0` | `exhausted` |

terminal verdicts: `approved | exhausted`

### PassageReport.blocker (extend)

```typescript
blocker?:
  | 'review.self'
  | 'review.peer.concerned'  // renamed from 'review.peer'
  | 'review.peer.exhausted'  // NEW
  | 'judge'
  | 'approval';
```

### risk mitigations (3.1.5.research.reflection.product.premortem._.yield.md)

| identified risk | blueprint mitigation |
|-----------------|---------------------|
| budget consumed on cache hit | cache check BEFORE budget decrement (codepath tree shows order) |
| exhaustion corrupts across routes | state lives in route's `.route/` dir (inherently scoped) |
| level order waits forever | terminal state = approved OR exhausted (codepath tree explicit) |
| defaults break extant routes | unlimited budget, level 1 default (backwards compat section) |
| passage.jsonl blocker type not handled | exhaustive type union with TypeScript (domain object changes section) |

### root cause validation (3.1.5.research.reflection.product.rootcause._.yield.md)

both root causes identified in the 5-whys analysis are addressed:

1. **no budget model** → addressed via `budget` field, exhaustion verdict, rounds tracking
2. **no level model** → addressed via `level` field, level-based ordering, terminal state checks

## decision log

| decision | rationale | source |
|----------|-----------|--------|
| state in .route/reviewPeerMeters.jsonl | route-level state, persists across stones | premortem risk: exhaustion must not corrupt across routes |
| status enum not string | type safety for terminal states | internal pattern [8]-[9]: explicit blocker types |
| level default 1 | backwards compat with extant flat reviews | premortem risk: defaults break extant routes |
| single approval covers exhaustion + judge | wish explicit: "one approval flag" | external [21]-[23]: single escalation gate |
| exhaustion is terminal (like approved) | unlocks next level, doesn't block progression | premortem risk: level order waits forever |
| budget consumption only on success | malfunction should not consume budget | internal [5]-[7]: cache only on passed |

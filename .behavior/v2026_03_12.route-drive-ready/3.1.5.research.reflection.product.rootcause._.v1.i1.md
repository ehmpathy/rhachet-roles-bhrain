# reflection: root cause

## what is the symptom?

**observable problem:** clones freeze on stones, never run `--as passed`. they create artifacts, then wait for human permission.

- **what is broken:** clones don't advance behavior routes autonomously
- **how it manifests:** clone says "vision artifact ready. awaited passage." but never runs command
- **when it occurs:** after clone completes stone work, at passage point

---

## 5 whys tree

### branch 1: judgment frame hypothesis

1. **why don't clones run `--as passed`?**
   → they hesitate to claim "passed" status

2. **why do they hesitate to claim "passed"?**
   → "passed" implies judgment — clone doesn't want to declare success

3. **why doesn't clone want to declare success?**
   → clone is conditioned to defer judgment to humans

4. **why is clone conditioned this way?**
   → RLHF / instruction tuning teaches "wait for human approval"

5. **why does this matter here?**
   → **[ROOT CAUSE]** the "passed" frame triggers "wait for approval" pattern

### branch 2: instruction ambiguity hypothesis

1. **why don't clones run `--as passed`?**
   → guidance says "are we there yet? if so, run" — clone doesn't know if "we" includes it

2. **why is "we" ambiguous?**
   → could mean clone, human, or both

3. **why does ambiguity matter?**
   → clone defaults to safe interpretation: "human should decide"

4. **why default to human?**
   → same as branch 1 — conditioned to defer

5. **why does this matter here?**
   → **[SUPPORTING CAUSE]** ambiguous addressee reinforces hesitation

### branch 3: context compaction hypothesis

1. **why do clones freeze after compaction?**
   → they lose memory of prior attempts

2. **why does lost memory cause freeze?**
   → clone re-evaluates from scratch, hits same hesitation

3. **why same hesitation?**
   → root cause persists: "passed" frame + ambiguous guidance

4. **why doesn't fresh context help?**
   → each new context starts with same problematic text

5. **why does this matter here?**
   → **[AMPLIFYING FACTOR]** compaction multiplies hesitation chances

---

## root cause summary

| layer | issue | is root cause? |
|-------|-------|----------------|
| surface | clone doesn't run command | no (symptom) |
| behavioral | clone hesitates on "passed" | **yes** |
| cognitive | "passed" triggers defer-to-human | **yes** (deeper) |
| training | RLHF conditions deference | no (out of scope) |

---

## recommendation

**address behavioral layer:** change "passed" to "arrived"

- "arrived" is a state claim, not judgment
- clone can claim "i'm here" without claim of success
- guards decide passage, not clone

**why not address training layer:**
- we don't control RLHF
- we can't retrain models
- we CAN change how we frame prompts

**follow-up:**
- observe if `--as arrived` reduces freeze rate
- if not, investigate other hesitation causes

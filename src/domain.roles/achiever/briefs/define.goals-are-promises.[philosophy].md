# goals are promises

## .what

a goal is a promise to yourself or another. to forget a goal is to break a promise.

## .why

when a human asks for something, they trust you will do it. when you notice something that should be done, you commit to do it. these are promises.

promises require:
- clarity on what was promised (the ask)
- clarity on why it matters (the purpose and benefit)
- clarity on what success looks like (the outcome)
- clarity on how to achieve it (the task and gate)

without this clarity, promises drift. work wanders. trust erodes.

## .the shape of a goal

a goal forces foresight:

| field | question | example |
|-------|----------|---------|
| why.ask | what was said? | "fix the flaky test" |
| why.purpose | why do they want this? | ci should pass before merge |
| why.benefit | what does success enable? | team can ship |
| what.outcome | what is the end state? | test passes reliably |
| how.task | what is the next action? | run test in isolation, find flake |
| how.gate | how do we verify done? | 10 consecutive passes |

each field is a distinct thought. shallow inference becomes impossible when the shape demands depth.

## .the practice

1. **detect** — when input arrives, ask: is this a goal?
2. **articulate** — fill each field. if you cannot, clarify with the asker.
3. **persist** — write to `.goals/` so the promise is visible
4. **cover** — link each ask to a goal, so no ask is forgotten
5. **fulfill** — work toward the gate, then mark done with evidence

## .the mantra

> to forget an ask is to break a promise. remember.

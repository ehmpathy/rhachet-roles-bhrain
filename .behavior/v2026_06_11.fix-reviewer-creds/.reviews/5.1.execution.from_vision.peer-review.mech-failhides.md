# blocker.1: Silent catch hides errors from execSync opener

**rule**: rule.forbid.failhide

**locations**:
- src/contract/cli/review.ts

In the review CLI entrypoint, errors from execSync (e.g. opener program failure) are caught and only logged without rethrowing. This hides real errors from the caller and violates the forbid.failhide rule. Catch blocks must only handle allowlisted errors and rethrow the rest; otherwise use HelpfulError.wrap for observable wrapping.

**snippet**:
```ts
      try {
        execSync(command, { stdio: 'inherit' });
      } catch (error) {
        if (error instanceof Error) {
          console.error(
            `opener '${options.open}' unavailable or failed: ${error.message}`,
          );
        }
      }
```

# vision: flexible stone lookup for route.stone.set

## wish

make `route.stone.set --stone` have the same flexible lookup as `route.stone.del`:
- `--stone 3.3` should match `3.3.blueprint.v1` if it's the only stone matching `*3.3*`
- if multiple stones match, failfast with a helpful error listing matches

## problem

### current behavior

```sh
# route.stone.del has flexible lookup (works)
rhx route.stone.del --stone research  # matches *research*

# route.stone.set requires exact match (broken)
rhx route.stone.set --as approved --stone 3.3.blueprint.v1  # works
rhx route.stone.set --as approved --stone 3.3               # fails - not found
```

### root cause

three different lookup strategies exist:

| code path | strategy | auto-wrap | failfast |
|-----------|----------|-----------|----------|
| `stepRouteStoneDel` | `asStoneGlob` + `isStoneInGlob` | yes (`3.3` → `*3.3*`) | n/a (returns all) |
| `setStoneAsApproved` | duplicated `findStoneByGlob` | no | no (returns first) |
| `setStoneAsPassed` | duplicated `findStoneByGlob` | no | no (returns first) |
| `stepRouteStoneSet --as promised` | `.includes()` substring | no | no (returns first) |

### issues

1. **no auto-wrap**: `3.3` doesn't become `*3.3*` → won't match `3.3.blueprint.v1`
2. **no failfast**: if multiple stones match, silently returns first one
3. **duplicated code**: `findStoneByGlob` duplicated in two files
4. **inconsistent**: `--as promised` uses different strategy than `--as approved/passed`

## solution

### 1. create shared utility

add `findOneStoneByPattern` to `asStoneGlob.ts`:

```ts
/**
 * .what = finds exactly one stone matching a pattern
 * .why = enables flexible stone lookup with failfast for ambiguous patterns
 */
export const findOneStoneByPattern = (input: {
  stones: RouteStone[];
  pattern: string;
}): RouteStone | null => {
  // expand pattern via asStoneGlob (auto-wrap if no glob chars)
  const { glob } = asStoneGlob({ pattern: input.pattern });

  // find all matching stones
  const matched = input.stones.filter((s) =>
    isStoneInGlob({ name: s.name, glob }),
  );

  // failfast if more than one match
  if (matched.length > 1) {
    throw new BadRequestError(
      `pattern "${input.pattern}" matched ${matched.length} stones; be more specific`,
      { pattern: input.pattern, matched: matched.map((s) => s.name) },
    );
  }

  return matched[0] ?? null;
};
```

### 2. update consumers

| file | change |
|------|--------|
| `setStoneAsApproved.ts` | remove `findStoneByGlob`, use `findOneStoneByPattern` |
| `setStoneAsPassed.ts` | remove `findStoneByGlob`, use `findOneStoneByPattern` |
| `stepRouteStoneSet.ts` | replace `.includes()` with `findOneStoneByPattern` |

### 3. expected behavior after

```sh
# flexible lookup (auto-wrap)
rhx route.stone.set --as approved --stone 3.3        # matches 3.3.blueprint.v1
rhx route.stone.set --as approved --stone blueprint  # matches 3.3.blueprint.v1
rhx route.stone.set --as promised --stone 3.3 --that check  # matches 3.3.blueprint.v1

# failfast on ambiguous (multiple matches)
rhx route.stone.set --as approved --stone 3
# error: pattern "3" matched 4 stones; be more specific
#   matched: 3.1.research.v1, 3.2.blueprint.v1, 3.3.execute.v1, 3.4.reflect.v1

# no match still errors
rhx route.stone.set --as approved --stone nonexistent
# error: stone not found
```

# blocker.1: Hidden side effects in getXaiCredsFromKeyrack

**rule**: .agent/repo=bhuild/role=behaver/briefs/practices/behavior.execution/architect/rule.forbid.behavior-hazards.md

**locations**:
- src/domain.operations/credentials/getXaiCredsFromKeyrack.ts

The function 'getXaiCredsFromKeyrack' performs hidden side effects beyond its named purpose of fetching credentials. It modifies global process.env.XAI_API_KEY and can abruptly terminate the process with process.exit(2) in error cases. This creates unpredictable behavior as callers may not expect environment mutations or process termination. According to the rule, functions must not have hidden side effects; the name suggests only getting creds, but it also sets globals and exits. Refactor to separate concerns: return creds without side effects, or rename to reflect full behavior.

**snippet**:

      // set in process.env for current rhachet compatibility

### root cause 2: stepReview Promise.all

per 3.1.3.research.internal.product.code.prod._.yield.md [4]: Promise.all bulk read causes OOM

**key insight**: sequential reads alone don't help if all content is accumulated in an array. the fix is to build the prompt incrementally — after each file is appended, its content becomes eligible for GC. this halves peak memory (1× content vs 2× content).

  // 4. return combined hashes

### buildPromptSection helper

```typescript
/**
 * .what = build prompt section incrementally, not via array accumulation
 * .why = after each append, file content is GC-eligible — halves peak memory
 *
 * array approach (bad):  accumulator + all files in memory → 2× content
 * string approach (good): accumulator only, content GC'd after append → 1× content
 */
const buildPromptSection = async (input: {
  files: string[];
  cwd: string;
  header: string;
}): Promise<string> => {
  let section = `${input.header}\n`;
  for (const file of input.files) {
    const content = await fs.readFile(path.join(input.cwd, file), 'utf-8');
    section += `--- ${file} ---\n${content}\n\n`;
    // content is now eligible for GC
  }
  return section;
};
```

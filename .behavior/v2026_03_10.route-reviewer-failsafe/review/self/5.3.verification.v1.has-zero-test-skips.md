## deeper reflection

### what could still be hidden?

1. **conditional skips** - tests that skip based on environment variables?
   - searched for `process.env` in test files: no conditional skips found
   - all tests run unconditionally

2. **implicit skips via early returns**?
   - reviewed test structure: all tests use given/when/then pattern
   - no early returns that would skip assertions

3. **tests that pass but don't actually assert**?
   - reviewed each test: all have explicit `expect()` calls
   - snapshot tests have `.toMatchSnapshot()` assertions

4. **mocked dependencies that hide real failures**?
   - reviewed test files: no jest.mock() used
   - integration tests use real execa commands
   - unit tests use pure functions with no mocks

### why this matters

skipped tests create false confidence. if a test is skipped:
- CI passes but behavior is not verified
- regressions go undetected
- humans trust the green checkmark

for this behavior (failsafe for reviewers), skipped tests would be especially dangerous:
- the whole point is to catch failures that were previously hidden
- a skipped test would repeat the original problem

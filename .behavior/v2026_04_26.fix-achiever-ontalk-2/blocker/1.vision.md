# blocker: 1.vision

## what blocks me

this vision cannot be implemented because Claude Code's `UserPromptSubmit` hook lacks the source metadata needed to distinguish user prompts from hook output.

**root cause:** bug #16952 — `UserPromptSubmit` fires when Task/subagent completes, not just on user prompt. the stdin JSON contains:

```json
{ "session_id", "transcript_path", "cwd", "hook_event_name", "prompt" }
```

there is no `source` field to distinguish user input from hook output.

**bug status:** closed (inactivity), locked, not fixed.

## what i tried

1. researched the stdin schema via websearch and gist documentation
2. verified `BrainHookFilter` interface — only supports `what` and `when` for onTool, no source filter for onTalk
3. checked bug #16952 — confirmed the issue, but it's closed without resolution

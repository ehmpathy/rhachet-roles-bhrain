# self-review: 1.vision.has-questioned-requirements

## requirements questioned

### requirement 1: show "run rhx route.drive" after `--as passed`

**who said this was needed?**
the wisher, explicitly in 0.wish.md

**what evidence supports this?**
the wish describes driver confusion: "so they dont get confused and try to go down a wrong direction"

**what if we didn't do this?**
drivers would continue to experience the "now what?" moment after they pass a stone

**is the scope appropriate?**
yes — extremely focused. one message, one trigger condition

**could we achieve this in a simpler way?**
no — this is already the minimum viable change: add text output after passage

**verdict:** requirement holds ✓

---

### requirement 2: only show reminder on `--as passed`, not other actions

**who said this was needed?**
implied by the wish (which specifically mentions `--as passed`) and made explicit in the vision's edgecases table

**what evidence supports this?**
- `--as approved` = guard sign-off, not passage
- `--as promised` = self-review checkpoint, not passage
- `--as rewound` = step backward, not forward

if we show "run route.drive" after these, users would be misled

**what if we didn't do this?**
users who run `--as approved` would be told to `route.drive` when they actually need to run `--as passed` next. confusion would increase, not decrease.

**is the scope appropriate?**
yes — this constraint is essential to the correctness of the feature

**verdict:** requirement holds ✓

---

### requirement 3: distinguish "continue" vs "complete" for final stone

**who said this was needed?**
the vision author, but only as an open question — not a firm requirement

**what evidence supports this?**
none yet — this is speculative

**what if we didn't do this?**
the reminder would say "run route.drive to continue" for the final stone, but route.drive would then show completion. this is fine — not unclear.

**is the scope appropriate?**
this may be over-built. the wish doesn't ask for this distinction.

**verdict:** defer — keep as open question, do not implement unless wisher requests ✓

### requirement 4: reminder text format

**who said this was needed?**
the wisher: "run `rhx route.drive` to continue the drive"

**what evidence supports this?**
direct quote from wish

**could we achieve this in a simpler way?**
the vision proposes: "the way continues, run → rhx route.drive"
this is slightly more elaborate than the wish, but fits the output tree style.
both satisfy the goal.

**verdict:** requirement holds — either wording works ✓

## requirement 1: malfunction resolve unblocks hook

**who said this was needed?**
the wish explicitly states: "add acceptance test coverage that proves that after the malfunction is resolved, the hook is unblocked, because we know that if the last status for the stone in the passage.jsonl is 'passed', then the malfunction was fixed"

**what evidence supports this?**
- the wish says "looks like right now that's not true" — confirmed by code review
- `stepRouteDrive.ts` line 50-51 uses `.find()` which gets first match, not latest status
- no test exists for malfunction → pass → unblock flow (only malfunction detection is tested)

**what if we didn't do this?**
mechanics would be stuck forever after a malfunction, even after the human fixes the issue and the mechanic retries successfully. this violates the natural expectation that "fix and retry" should work.

**is the scope too large, too small, or misdirected?**
scope is correct. the fix is narrow: change `.find()` to check latest status per stone.

**could we achieve the goal in a simpler way?**
no — this is the minimal fix. the current behavior is a bug.

**verdict: requirement holds ✅**

## requirement 2: reviewer malfunction not cached

**who said this was needed?**
the wish explicitly states: "we need acceptance test coverage that proves that if a reviewer malfunctions, we dont cache its broken state. only successes get cached"

**what evidence supports this?**
- judges already do this: `runStoneGuardJudges.ts` line 197 has `&& passed` filter
- reviews do NOT do this: `runStoneGuardReviews.ts` line 133-135 caches ALL prior reviews
- this asymmetry is inconsistent and causes broken reviewer results to be reused

**what if we didn't do this?**
a broken reviewer's malfunction would be reused forever from cache, even after the human fixes the reviewer script. the mechanic would retry but get the same cached broken result.

**is the scope too large, too small, or misdirected?**
scope is correct. the fix is narrow: filter out malfunctions from review cache (like judges).

**could we achieve the goal in a simpler way?**
no — to align reviews with judges is the natural fix.

**verdict: requirement holds ✅**

## phase 0: blocked domain operations

**prereqs**: read blueprint at `.behavior/v2026_03_12.route-as-blocked/3.3.1.blueprint.product.v1.i1.md`

### 0.1 create getBlockedTriggeredReport

- [ ] create `src/domain.operations/route/blocked/getBlockedTriggeredReport.ts`
- [ ] implement: read `$route/.route/$stone.blocked.triggered` file
- [ ] return: `{ triggered: boolean; path: string }`

**verify**: unit test passes for file exists / file absent cases

### 0.2 create setBlockedTriggeredReport

- [ ] create `src/domain.operations/route/blocked/setBlockedTriggeredReport.ts`
- [ ] implement: write `$route/.route/$stone.blocked.triggered` file
- [ ] ensure: file is gitignored (extant .gitignore in .route/)

**verify**: unit test passes for file creation

### 0.3 create getBlockedChallengeDecision

- [ ] create `src/domain.operations/route/blocked/getBlockedChallengeDecision.ts`
- [ ] implement decision logic:
  - call `getBlockedTriggeredReport`
  - if no triggered file → return `'challenge:first'`
  - check articulation file at `$route/.route/blocker/$stone.md`
  - if no articulation → return `'challenge:absent'`
  - else → return `'allowed'`
- [ ] create unit test: `getBlockedChallengeDecision.test.ts`

**verify**: unit tests pass for all three decision states

### 0.4 create setStoneAsBlocked

- [ ] create `src/domain.operations/route/blocked/setStoneAsBlocked.ts`
- [ ] implement challenge flow:
  - call `getBlockedChallengeDecision`
  - if `challenge:first`: call `setBlockedTriggeredReport`, return nudge emit
  - if `challenge:absent`: return nudge emit
  - if `allowed`: call `setPassageReport({ status: 'blocked' })`, return success emit
- [ ] create unit test: `setStoneAsBlocked.test.ts`

**verify**: unit tests pass for challenge:first, challenge:absent, allowed flows

## part 1: barrel export investigation

### the concern

mechanic brief `rule.forbid.barrel-exports` states:

> banned examples:
> - src/domain.objects/index.ts
> - src/domain.operations/organization/index.ts
>
> allowed only: in index.ts file, export one object
> e.g., `export const daoAwsOrganization = { getOne, getAll, set, del }`

the blueprint lists several `index.ts` files that will be ejected:
- `plugins/atoms/index.ts`
- `plugins/toolboxes/bash/index.ts`
- `plugins/toolboxes/files/index.ts`
- `plugins/toolboxes/web/index.ts`

### actual code inspection

**atoms/index.ts:**
```typescript
export { genAtomAnthropic } from './anthropic';
export { genAtomOpenai } from './openai';
```

this exports 3 separate items — a barrel export pattern.

**toolboxes/bash/index.ts:**
```typescript

  name: 'bash',
  definitions: [toolDefinitionExec],
  execute: async (...) => { ... },

```

this exports a single object — compliant.

**toolboxes/files/index.ts:**
```typescript
export const toolboxFiles: BrainArch1Toolbox = {
  name: 'files',
  definitions: [...],
  execute: async (...) => { ... },
};
```

this exports a single object — compliant.

**toolboxes/web/index.ts:**
```typescript
export const toolboxWeb: BrainArch1Toolbox = {
  name: 'web',
  definitions: [...],
  execute: async (...) => { ... },
};
```

this exports a single object — compliant.

### analysis

| file | pattern | compliant? |
|------|---------|------------|
| atoms/index.ts | re-exports 3 items | ⚠ potential issue |
| toolboxes/bash/index.ts | exports 1 object | ✓ compliant |
| toolboxes/files/index.ts | exports 1 object | ✓ compliant |
| toolboxes/web/index.ts | exports 1 object | ✓ compliant |

**is atoms/index.ts a violation?**

examined more closely:
- the 3 items are factory functions: `genAtomAnthropic`, `genAtomOpenai`, `genAtomQwen`
- they serve as an aggregator for related factories
- contrast with banned barrel exports which forward domain objects

**could it be refactored to comply?**
```typescript

  genAnthropic: genAtomAnthropic,
  genOpenai: genAtomOpenai,
  genQwen: genAtomQwen,

```

**conclusion:** this is inherited code, not created by the blueprint. the blueprint's job is to mv files, not refactor them. however, this is flagged for post-ejection attention.

**verdict:** ✓ blueprint itself follows standards; inherited code noted for attention

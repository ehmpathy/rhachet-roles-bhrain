# vision: `--as arrived` for route.drive

## the problem

clones freeze on stones, never run `--as passed`. they create artifacts, then wait for human permission. see `.behavior/v2026_03_12.route-drive-ready/refs/example.case1.md` for real example.

**root cause:** the `--as passed` framing feels like a judgment claim.
- "passed" implies verdict вАФ clone doesn't want to claim success
- clone thinks: "i don't know if this passes, you tell me"
- clone waits for human to decide
- clone never runs the command

**the fix:** introduce `--as arrived` вАФ a state claim, not a judgment.
1. clone says "i've arrived at the gate"
2. guards validate if clone can proceed
3. clone doesn't claim "passed" вАФ just arrival

## evaluation

### how well does it solve the goal?

**goal**: clones don't freeze, they run the command when done

**solution**:
1. `--as arrived` removes judgment framing
2. clone claims state, not verdict
3. nudge catches stuck clones after 7 hooks

### pros

- fixes root cause (judgment framing)
- minimal change вАФ just an alias
- keeps extant guidance text
- zen nudge for stuck clones
- backwards compatible (`--as passed` still works)

### cons

- adds a new flag (but it's just an alias)
- requires state tracking for nudge

### edgecases and pit of success

| edgecase | handling |
|----------|----------|
| clone runs `--as arrived` | great, normal flow |
| clone runs `--as passed` | still works (backwards compat) |
| clone stuck 7+ times | sees drum nudge |
| clone stuck 21+ times | escalate to human (extant behavior) |

## implementation

### `--as arrived` alias

in `stepRouteStoneSet.ts`:
```typescript

if (input.as === 'arrived') {

### state tracking for nudge

```

  "hooksWithoutAttempt": 3

# 1.vision: route guard persistence

## the outcome world

### before: noise in the signal

today, every guard artifact gets its own file in `.route/`:

```
.route/
├── 1.vision.passed                                    # ← this matters
├── 1.vision.approved                                  # ← this matters
├── 1.vision.guard.judge.i1p1.76ad332...j1.md         # ← noise
├── 1.vision.guard.review.i1.4d7ed5e...r1.md          # ← noise
├── 3.3.blueprint.v1.guard.promise.all-done-self...md # ← noise
├── 3.3.blueprint.v1.guard.selfreview...triggered     # ← noise
├── ... (50+ files per behavior route)
```

git diff shows 50+ file changes when only passage matters.

### after: one file for passage, gitignore the rest

```
.route/
├── .gitignore           # ✓ tracked — findserted, ignores everything else
├── passage.jsonl        # ✓ tracked — all passage events here
├── .bind.*              # ✓ tracked — route binding flags
└── (guard artifacts)    # ✗ gitignored, local cache only
```

**the fix:**
1. consolidate passage markers into `passage.jsonl`
2. gitignore everything else

```jsonl
{"stone":"1.vision","status":"approved"}
{"stone":"1.vision","status":"passed"}
{"stone":"3.3.blueprint.v1","status":"passed"}
```

### the "aha" moment

open a pr. the git diff shows:

```diff
 .route/passage.jsonl
+{"stone":"5.1.execution","status":"passed"}
```

one line added. passage achieved. no guard artifact noise.

## user experience

### usecases

| goal | before | after |
|------|--------|-------|
| review pr diff | scroll through 50 file changes | see only passage markers |
| check if stone passed | look for `.passed` file | same, but cleaner repo |
| debug guard locally | read guard artifacts | same, files still on disk |
| clone fresh | guard artifacts present | guard artifacts absent (re-run guards) |

### what gets tracked vs ignored

| file pattern | tracked? | why |
|--------------|----------|-----|
| `.gitignore` | ✓ yes | defines what to ignore |
| `passage.jsonl` | ✓ yes | passage state |
| `.bind.*` flags | ✓ yes | route binding |
| everything else | ✗ no | ephemeral cache |

### timeline

1. driver runs `route.drive`
2. guard runs → artifacts written locally (gitignored)
3. stone passes → `.passed` file created (tracked)
4. human commits → only passage markers in diff
5. fresh clone → guard artifacts absent, re-run on next drive

## mental model

### how users describe it

> "passage is what matters. guard artifacts are just local cache."

### analogies

| analogy | why it fits |
|---------|-------------|
| **build artifacts** | `.o` files gitignored, binary tracked |
| **node_modules** | installed locally, not in repo |
| **test coverage cache** | `.nyc_output` gitignored, badge tracked |

## evaluation

### how well does it solve the goals?

| goal | score | notes |
|------|-------|-------|
| reduce git diff noise | ✅ excellent | one file, append-only |
| implementation effort | ✅ small | gitignore + passage.jsonl refactor |
| preserve guard cache locally | ✅ yes | files still on disk |
| simple mental model | ✅ yes | passage.jsonl = truth, rest = cache |

### pros

1. **dramatically cleaner diffs** — one file, append-only
2. **simple mental model** — passage.jsonl is the source of truth
3. **local cache preserved** — guard artifacts still on disk for debug
4. **grep-able** — `grep '"stone":"1.vision"' passage.jsonl`

### cons

1. **fresh clone loses guard cache** — guards re-run on first drive
   - acceptable: guards are fast, cache is optimization not correctness
2. **small code change needed** — update read/write of passage markers to use jsonl
   - but much simpler than full jsonl refactor for all guard artifacts

### what stays tracked

each `.route/` dir gets its own `.gitignore`:

```
.route/
├── .gitignore       # ← findserted here
├── passage.jsonl    # ✓ tracked
├── .bind.*          # ✓ tracked
└── (everything else) # ✗ ignored
```

contents of `.route/.gitignore`:

```gitignore
# ignore everything except passage.jsonl and .bind flags
*
!.gitignore
!passage.jsonl
!.bind.*
```

## implementation

1. **findsert `.route/.gitignore`** — idempotently create/update inside each .route dir
2. **consolidate passage markers into passage.jsonl** — append-only, one line per event
3. **remove old marker files** — `.passed`, `.approved`, `.blocker.json` become jsonl entries

done. 🐢

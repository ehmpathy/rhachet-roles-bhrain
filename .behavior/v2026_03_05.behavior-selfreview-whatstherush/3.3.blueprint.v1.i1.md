# blueprint: self-review rush detection

## overview

detect when clones run `set --as promised` rapidly after a challenge and prepend a "what is the rush?" confrontation before the normal patience prompt.

## contracts

### setSelfReviewTriggeredReport (extended)

```ts
input: { stone: string; slug: string; hash: string; route: string }
output: { sincePath: string; uptilPath: string }
// was: writes .triggered file
// now: writes .triggered.since if absent (preserves original mtime)
//      always touches .triggered.uptil (updates mtime on every call)
```

### getSelfReviewTriggeredCount (extended)

```ts
input: { stone: string; slug: string; route: string }
output: {
  count: number;
  newest: {
    hash: string;
    sinceMtime: Date;
    uptilMtime: Date;
  } | null
}
// was: counts .triggered files, returns newestMtime
// now: returns newest triggered pair with both mtimes for rush comparison
```

### getSelfReviewChallengeDecision (extended)

```ts
input: { stone: string; slug: string; hash: string; route: string }
output: { decision: 'allowed' | 'challenge:first' | 'challenge:rushed' }
// was: returns 'allowed' | 'challenged'
// now: distinguishes 'challenge:first' (normal prompt) vs 'challenge:rushed' (rush prompt)
//      challenge:rushed when sinceMtime < uptilMtime
```

### formatWhatsTheRush (new)

```ts
input: void
output: string
// returns the confrontation tree:
// 🍂 what is the rush?
//    ├─ ...
//    ├─ what does haste cost?
//    ... (as per vision)
```

## file name convention

```
{route}/
├── .route/
│   ├── {stone}.guard.selfreview.{slug}.{hash}.triggered.since   # created on first challenge
│   └── {stone}.guard.selfreview.{slug}.{hash}.triggered.uptil   # touched on every challenge
└── review/
    └── self/
        └── {stone}.{slug}.md                                     # articulation file (clone must create)
```

note: triggered files follow extant naming pattern with .since/.uptil suffix
note: rush detection compares since.mtime vs uptil.mtime
note: articulation files live in `{route}/review/self/`, NOT in `.route/`

## logic flow

1. clone runs `set --as promised --that review.self`
2. `setSelfReviewTriggeredReport` writes .since (if absent) + touches .uptil (always)
3. `getSelfReviewChallengeDecision` returns:
   - `allowed` → continue
   - `challenge:first` → show patience + reflect
   - `challenge:rushed` → show rush + patience + reflect
4. `formatRouteStoneEmit` prepends `formatWhatsTheRush()` if `challenge:rushed`
5. return patience + lets reflect output
6. **on pass**: cleanup .since + .uptil files for fresh start

## edgecases

| edgecase | handle |
|----------|--------|
| first challenge ever | .since + .uptil created with same mtime → `challenge:first` |
| files deleted mid-session | no mtimes → treat as first challenge (graceful) |
| multiple behaviors | files scoped to route (each behavior has its own `.route/`) |
| review passes | cleanup .since + .uptil files for fresh start |

## pattern 3 — the 30s-dwell backdate helper `[REUSE]`

self-review tests bypass the real-time 30s dwell by backdating the trigger mtime.

[5] `blackbox/driver.route.self-review.acceptance.test.ts:39-44`
```
  // backdate all .since files (needed for hashbar threshold check)
  const mtimePast = new Date(Date.now() - 31 * 1000);
  for (const sinceFile of sinceFiles) {
    const filepath = path.join(routeDir, sinceFile);
    await fs.utimes(filepath, mtimePast, mtimePast);
  }
```

**relation to wish**: if the contemplation gate reuses the self-review dwell (prod pattern 4), its tests must bypass the 30s wait the same way — otherwise each test would sleep 31s.
**[REUSE]** the backdate technique. `[KHUE]` open: if the contemplation gate is designed *without* a dwell (a lighter "file-present" check), this helper is unneeded for peer tests — depends on the blueprint decision on gate strictness.

## pattern 4 — snapshot vibecheck + time sanitizer `[REUSE]`

stdout is snapshotted so reviewers can eyeball the treestruct, with volatile values scrubbed.

[6] `blackbox/driver.route.self-review.acceptance.test.ts:126-128`
```
      then('stdout has good vibes', () => {
        expect(sanitizeTimeForSnapshot(result.stdout)).toMatchSnapshot();
      });
```

[7] `blackbox/.test/invokeRouteSkill.ts:56-58` — the volatile temp-dir scrub:
```
    .replace(/\/tmp\/test-fns\/[^/]+\/\.temp\/[^/]+\//g, '[TEMP]/')
    .replace(/finished \d+\.\d+s/g, 'finished [TIME]')
```

**relation to wish**: the "reviewers await your reply" stdout (and its three-surface identity, usecase 11) is exactly the kind of treestruct that wants a snapshot vibecheck — plus explicit `.toContain` assertions for the load-bearing lines. per `rule.require.snapshots`, use BOTH snapshot and explicit assertions.
**[REUSE]** `sanitizeTimeForSnapshot` + `toMatchSnapshot`.
**[EXTEND]** the sanitizer may need one new rule: the `.taken`/`.given` paths embed a content `$hash` that varies — a `[hash]` scrub (parallel to `[TIME]`) keeps peer snapshots stable. `[SUMP]`: the hash in the key is machine-variant across artifact edits; a scrub prevents snapshot churn.

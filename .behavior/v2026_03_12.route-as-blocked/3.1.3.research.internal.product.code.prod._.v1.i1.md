# research: internal product code (prod)

## objective

identify extant patterns in prod code that can be reused or extended for `--as blocked`.

---

## findings

### pattern.1: PassageReport domain object [REUSE]

**location**: `src/domain.objects/Driver/PassageReport.ts:8-13`

**quote**:
```typescript
status: 'passed' | 'approved' | 'blocked' | 'rewound' | 'malfunction'
blocker?: 'review.self' | 'review.peer' | 'judge' | 'approval'
reason?: string
```

**implication**: `blocked` status already supported in the type system. no new types needed.

**action**: REUSE as-is.

```typescript
as: 'passed' | 'approved' | 'promised' | 'rewound'
```

**implication**: need to add `'blocked'` to the union and handle its flow.

**action**: EXTEND to support `as: 'blocked'`.

### pattern.3: triggered file pattern [REUSE]

**location**: `src/domain.operations/route/promise/setSelfReviewTriggeredReport.ts:15-25`

**quote**:
```typescript
const triggeredPath = path.join(routeDir, '.route', `${stone}.triggered.since`);
await fs.writeFile(triggeredPath, new Date().toISOString());
```

**implication**: same pattern applies for blocked nudge. create `$stone.blocked.triggered` file.

**action**: REUSE pattern, new file name.

---

### pattern.4: challenge decision pattern [REUSE]

**location**: `src/domain.operations/route/promise/getSelfReviewChallengeDecision.ts:8-15`

**quote**:
```typescript
type SelfReviewChallengeDecision =
  | 'challenge:first'   // no triggered file
  | 'challenge:rushed'  // triggered but too fast
  | 'challenge:absent'  // triggered but no articulation
  | 'allowed'           // triggered and articulation present
```

**implication**: blocked needs similar flow:
- check `.blocked.triggered` exists
- check articulation file exists at `$route/.route/blocker/$stone.md`
- return decision

**action**: REUSE pattern, create `getBlockedChallengeDecision.ts`.

# reflection: audience

## who is the audience?

### primary: AI clones on behavior routes

AI clones (LLMs like Claude) that work on behavior routes. they see route.drive output at session start/end and need to run `--as passed` to proceed.

**context:**
- work in conversation sessions with humans
- follow route guidance to complete behavior stones
- subject to context compaction (memory loss)
- conditioned by RLHF to wait for explicit human instruction

### secondary: human developers

maintainers of rhachet-roles-bhrain who read and extend the route system.

**context:**
- understand typescript/nodejs patterns
- familiar with behavior route architecture
- care about code clarity and test coverage

### tertiary: human wishers

people who use behavior routes to delegate work to clones.

**context:**
- define wishes and review artifacts
- approve guarded stones
- benefit when clones don't freeze

---

## why do they care?

### clones

- **pain:** hesitate on `--as passed` because it feels like a judgment claim
- **goal:** proceed to next stone after work is done
- **frustration:** stuck in loops, never advance

### developers

- **pain:** debug why clones freeze
- **goal:** clean, testable code
- **frustration:** unclear semantics, magic numbers

### wishers

- **pain:** clones that spin without progress
- **goal:** efficient completion of behavior routes
- **frustration:** must manually intervene (run `--as passed` for clones)

---

## how much do they care?

| audience | priority | urgency | stakes |
|----------|----------|---------|--------|
| clones | high | high | blocked without fix |
| developers | medium | low | code quality, not blocked |
| wishers | high | medium | productivity loss |

---

## what do they care about?

### clones

- **success criteria:** feels natural to run `--as arrived`
- **acceptable tradeoffs:** extra command alias
- **delight:** drum nudge that guides without pressure

### developers

- **success criteria:** minimal code changes, clear patterns
- **acceptable tradeoffs:** new state file for hook counter
- **delight:** well-tested, documented change

### wishers

- **success criteria:** clones stop freeze behavior
- **acceptable tradeoffs:** none visible to them
- **delight:** faster behavior completion

---

## how will they experience it?

### clones

1. see route.drive output with `--as arrived` (not `--as passed`)
2. feel comfortable with "arrival" frame
3. run `--as arrived` when work is done
4. if stuck, see drum nudge after 7 hooks
5. drum nudge reminds without pressure

### developers

1. see clean code changes in PR
2. read well-structured tests
3. understand the frame rationale

### wishers

1. notice clones advance more reliably
2. fewer manual interventions needed
3. behavior routes complete faster

### 4. onBoot hook — can this be removed?

**verdict**: QUESTION — needs verification

**analysis**: the blueprint mentions add an onBoot hook like mechanic. but driver already HAS onBoot hooks (passage checker, notify). the boot property may be sufficient without another onBoot command.

**resolution**: verified — driver already has onBoot hooks. we only need to add the `boot` property, not a new onBoot hook command. the boot property enables boot.yml load; the extant onBoot hooks handle other concerns.

**simplification**: removed unnecessary onBoot hook from blueprint scope. just add `boot: { uri: __dirname + '/boot.yml' }`.

### 7. commands table — can this be removed?

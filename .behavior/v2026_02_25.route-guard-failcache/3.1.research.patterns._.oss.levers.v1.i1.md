## tools considered

### 1. external cache libraries

**considered:** `lru-cache`, `node-cache`, `keyv`

**decision:** not needed

**reason:** the judge cache is file-based (artifacts in `.route/` directory). the cache mechanism is already implemented via filesystem read/write. no in-memory cache layer needed.

### 2. memoization libraries

**considered:** `lodash.memoize`, `memoizee`, `p-memoize`

**decision:** not needed

**reason:** the cache key computation already uses content hash. memoization would add complexity without benefit — the fix is to change which results get stored, not how storage works.

### 3. retry libraries

**considered:** `p-retry`, `async-retry`, `retry`

**decision:** not needed

**reason:** the fix enables natural retry by not storing failures in cache. no automatic retry logic needed — the route system handles retries at a higher level.

## why no OSS tools required

the change is behavioral, not infrastructural:

| aspect | current | after fix |
|--------|---------|-----------|
| cache storage | `.route/` files | unchanged |
| cache key | content hash | unchanged |
| cache lookup | priorByIndex map | unchanged |
| cache write decision | always | only on pass |

the only change: add `&& passed` to the filter condition.

# playtest: --as arrived for route.drive

## prerequisites

- repo built: `npm run build`
- on branch with the `--as arrived` changes

## sandbox

all test operations use `.temp/playtest-arrived/` directory, cleaned up at end.

---

## happy paths

### 1. normal guidance shows both options

**setup:**
```bash
# create test behavior in .temp
mkdir -p .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived
cat > .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/0.wish.md << 'EOF'
wish = test the --as arrived feature
EOF

cat > .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/1.test.stone << 'EOF'
a simple test stone
EOF
```

**action:**
```bash
cd .temp/playtest-arrived
npx rhachet run --skill route.bind.set -- --route .behavior/v2026_03_18.test-arrived
npx rhachet run --skill route.drive
```

**expected outcome:**
- output shows "are you here?" section (appears twice: before and after stone content)
- shows "when ready for review, run:" with `--as arrived` command
- shows "when ready to continue, run:" with `--as passed` command

**pass:** both options visible in guidance (top and bottom)
**fail:** only `--as passed` shown, or `--as arrived` absent

### 2. --as arrived works like --as passed

**action:**
```bash
cd .temp/playtest-arrived
```

## edgey paths

### 4. drum nudge appears after 7+ hooks

**setup:**
```bash
# create a stone with high hook count
cat > .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/3.nudge.stone << 'EOF'
stone to test drum nudge
EOF

# manually set blocker state to simulate 7 hooks
mkdir -p .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/.route
echo '{"count":7,"stone":"3.nudge"}' > .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/.route/.drive.blockers.latest.json
```

**action:**
```bash
cd .temp/playtest-arrived
npx rhachet run --skill route.stone.set -- --stone 2.second --as arrived
npx rhachet run --skill route.drive
```

**expected outcome:**
- output shows drum nudge section with tao quote
- "do your work, then step back" text visible
- nudge appears BEFORE the "are you here?" section

**pass:** drum quote visible, formatted as tree bucket
**fail:** no drum nudge despite count >= 7

---

### 5. counter resets after attempt

**action:**
```bash
cd .temp/playtest-arrived
# run --as arrived (or --as passed) on the stone
npx rhachet run --skill route.stone.set -- --stone 3.nudge --as arrived

# check the blocker state file
cat .temp/playtest-arrived/.behavior/v2026_03_18.test-arrived/.route/.drive.blockers.latest.json
```

**expected outcome:**
- blocker state file shows count reset (count: 0 or file deleted)

**pass:** count reset to 0 or state cleared
**fail:** count still at 7+

## pass/fail criteria

- ✓ pass if:
  - normal guidance shows both `--as arrived` and `--as passed` options
  - `--as arrived` successfully passes stones (works like `--as passed`)
  - backwards compat: `--as passed` still works
  - drum nudge appears after 7+ hooks with tao quote
  - hook counter resets after any passage attempt

- ✗ fail if:
  - only `--as passed` shown (old behavior)
  - `--as arrived` throws error or doesn't pass stone
  - drum nudge absent despite count >= 7
  - counter doesn't reset after attempt

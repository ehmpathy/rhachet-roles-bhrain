
## what is awkward?

### the original error

the wish mentioned 5 affected hooks but there are actually 4 `npx` path issues (reflector was removed in a later commit) plus 1 invalid `--when hook.onBoot` issue that causes a different error.

### the inconsistency

why do some hooks use `./node_modules/.bin/rhachet` and others use `npx rhachet`?

- `./node_modules/.bin/rhachet` — runs the local binary directly (fast, no npx overhead)
- `npx rhachet` — uses npx to find and run the binary (works even if not in PATH)

both are valid for `rhachet`. the bug was that `./node_modules/.bin/npx` conflates both patterns incorrectly.

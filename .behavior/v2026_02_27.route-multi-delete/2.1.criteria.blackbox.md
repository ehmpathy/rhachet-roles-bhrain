# blackbox criteria: multi-stone delete

## usecase.1 = delete multiple stones in one command

```
given(a route with multiple stones)
  when(user runs `rhx route.stone.del --stone 3.2 --stone 3.1 --stone 2.3 --mode apply`)
    then(all stones matched by any pattern are deleted)
      sothat(user achieves batch cleanup in one operation)
    then(output shows all deleted stones aggregated)
      sothat(user sees full scope of changes)
    then(output shows total deleted count)
      sothat(user confirms expected quantity)
```

## usecase.2 = plan mode previews deletions

```
given(a route with stones)
  when(user runs `rhx route.stone.del --stone 3.1 --stone 2.3` without --mode)
    then(no stones are deleted)
      sothat(user can preview before action)
    then(output shows stones that would be deleted)
      sothat(user can verify before apply)

  when(user runs `rhx route.stone.del --stone 3.1 --stone 2.3 --mode plan`)
    then(same preview behavior as when --mode absent)
      sothat(plan is explicit default)
```

## usecase.3 = patterns with no matches are skipped silently

```
given(a route with stones 3.1.* and 3.2.*)
  when(user runs `rhx route.stone.del --stone 3.1 --stone 9.9 --mode apply`)
    then(stones matched by 3.1 are deleted)
    then(pattern 9.9 is silently skipped)
      sothat(output stays clean without noise)
    then(deleted count reflects actual deletions only)
```

## usecase.4 = deduplication when patterns overlap

```
given(a route with stone 3.1.research.domain._.v1)
  when(user runs `rhx route.stone.del --stone 3.1 --stone 3.1.research --mode apply`)
    then(the stone is deleted once)
      sothat(no duplicate deletion errors)
    then(the stone appears once in output)
      sothat(output reflects actual operations)
```

## usecase.5 = at least one --stone required

```
given(a route with stones)
  when(user runs `rhx route.stone.del --mode apply` without --stone)
    then(command errors with clear message)
      sothat(user knows --stone is required)
    then(no stones are deleted)
      sothat(accidental mass deletion is prevented)
```

## usecase.6 = empty result is valid

```
given(a route with no stones that match any provided pattern)
  when(user runs `rhx route.stone.del --stone nonexistent --mode apply`)
    then(command exits successfully)
    then(output shows deleted = 0)
      sothat(user knows patterns matched no stones)
```

### contract (internal, not shown to clone)

**mechanism:**
- `.triggered.since` - created on first challenge, stays put
- `.triggered.uptil` - updated on each challenge attempt

**logic:**
- on challenge:
  - if no `.triggered.since`: create both files, show normal patience prompt (first attempt)
  - if `since.mtime < uptil.mtime`: prepend "what's the rush?" (they've asked twice)
  - if `since.mtime == uptil.mtime`: show normal patience prompt (first attempt)
  - always update `.triggered.uptil` mtime

### sequence (internal)

1. clone runs `set --as promised --that` в†’ challenged в†’ both files created (same mtime) в†’ normal prompt
2. clone runs `set --as promised --that` again в†’ `since.mtime < uptil.mtime` в†’ "what's the rush?"
3. clone runs `set --as promised --that` again в†’ `since.mtime < uptil.mtime` в†’ "what's the rush?"
4. clone passes в†’ both files deleted в†’ next challenge starts fresh

## mental model

### how the clone experiences it

"i tried to re-run review quickly and got a fallen leaf - 'what is the rush?' then just '...' - it made me stop and think."

### how we describe it (internal)

"rapid re-triggers after a challenged review get a disappointed face prepended to let it sink in."

### analogies & metaphors

- **fallen leaves**: not angry, just the natural way вЂ” haste has its season, and it passes
- **the pause**: "..." lets it sink in
- **uncle iroh moment**: sometimes silence speaks louder

## evaluation

### how well does it solve the goals?

| goal | how well solved |
|------|-----------------|
| confront impatience | well - direct "what is the rush?" |
| not be mean | well - fallen leaf not angry, "..." lets them reflect |
| be bypassable | well - they can still proceed after |

### pros

- minimal, impactful
- fallen leaf hits different than angry
- "..." creates space for reflection
- normal patience prompt still follows

### cons

- might feel harsh to clones who genuinely fixed fast

### edgecases & pit of success

| edgecase | how we handle it |
|----------|------------------|
| first trigger ever | normal flow, no fallen leaf |
| review passed (not challenged) | no `.triggered.*` files created, no confrontation |
| file deleted mid-session | same as first trigger в†’ graceful recovery |
| multiple behaviors | each behavior has its own `.triggered.*` files in its route |

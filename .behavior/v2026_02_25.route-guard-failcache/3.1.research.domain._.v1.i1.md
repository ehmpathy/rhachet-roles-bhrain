# research: domain objects and operations

## domain objects

### RouteStone (entity)

**location:** `src/domain.objects/Driver/RouteStone.ts`

represents a milestone in a behavior route.

| attribute | type | description |
|-----------|------|-------------|
| path | string | filesystem path to stone file |
| name | string | stone identifier (e.g., "1.vision") |

### RouteStoneGuard (literal)

**location:** `src/domain.objects/Driver/RouteStoneGuard.ts`

represents the conditions to pass a guarded stone.

| attribute | type | description |
|-----------|------|-------------|
| path | string | path to .guard file |
| artifacts | string[] | glob patterns for artifact detection |
| reviews | string[] \| { self, peer } | shell commands to run reviews |
| judges | string[] | shell commands to run judges |

### RouteStoneGuardJudgeArtifact (entity)

**location:** `src/domain.objects/Driver/RouteStoneGuardJudgeArtifact.ts`

represents the output of a judge run.

| attribute | type | description |
|-----------|------|-------------|
| stone | RefByUnique<RouteStone> | reference to the stone |
| hash | string | artifact content hash |
| iteration | number | attempt number |
| index | number | judge index (j1, j2, etc) |
| path | string | path to judge file under .route/ |
| passed | boolean | whether the judge passed |
| reason | string \| null | reason for pass/fail |

**unique key:** `['stone', 'hash', 'index']`
**primary key:** `['path']`

## relationships

```
RouteStone
    │
    └── has one RouteStoneGuard (via .guard file)
            │
            ├── has many artifacts (glob patterns)
            ├── has many reviews (shell commands)
            └── has many judges (shell commands)
                    │
                    └── produces RouteStoneGuardJudgeArtifact
```

## cache key structure

current cache key: `${stone.name}.guard.judge.i${reviewIter}p${judgeIter}.${reviewHash}.${judgeHash}.j${index}.md`

components:
- `stone.name` — which stone
- `reviewIteration` — which review pass
- `judgeIteration` — which judge pass
- `reviewInputHash` — artifact content state
- `judgeInputHash` — review + approval state
- `index` — which judge (j1, j2, etc)

## phase 1: getGitBlobHashes

**read first:**
- `3.3.1.blueprint.product.yield.md` — root cause 1 codepath tree
- `3.1.3.research.internal.product.code.prod._.yield.md` — [1] fs.readFile loop

### deliverables

- [ ] **1.1** create `src/domain.operations/route/guard/getGitBlobHashes.ts`
  - input: `{ files: string[], cwd: string }`
  - output: `Record<string, string>` (file → hash)
  - use `git ls-tree HEAD` for committed files
  - use `git diff --name-only HEAD` to detect modified
  - use `git hash-object <file>` for modified/untracked
  - handle deleted files (return empty hash or error)

- [ ] **1.2** create `src/domain.operations/route/guard/getGitBlobHashes.test.ts`
  - [case1] committed file returns ls-tree hash
  - [case2] modified file returns hash-object hash
  - [case3] untracked file returns hash-object hash
  - [case4] batch of 100+ files completes efficiently

- [ ] **1.3** update `src/domain.operations/route/guard/computeStoneReviewInputHash.ts`
  - replace fs.readFile loop with getGitBlobHashes call
  - retain sort + sha256 combination

- [ ] **1.4** update `src/domain.operations/route/guard/computeStoneReviewInputHash.test.ts`
  - [case3] 700+ artifacts completes without OOM
  - [case4] modified file invalidates hash

### verification

```sh

rhx git.repo.test --what unit --scope computeStoneReviewInputHash
```

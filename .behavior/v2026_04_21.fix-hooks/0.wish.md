# defect: SessionStart hooks use invalid npx path

## .what

five SessionStart hooks in `.claude/settings.json` use `./node_modules/.bin/npx` which does not exist. `npx` is a global npm tool, not a local package binary.

## .error

```
Failed with non-blocking status code: /bin/sh: 1: 
./node_modules/.bin/npx: not found
```

## .affected hooks

| role | source file |
|------|-------------|
| achiever | `src/domain.roles/achiever/getAchieverRole.ts` |
| driver | `src/domain.roles/driver/getDriverRole.ts` |
| librarian | `src/domain.roles/librarian/getLibrarianRole.ts` |
| reflector | `src/domain.roles/reflector/getReflectorRole.ts` |
| reviewer | `src/domain.roles/reviewer/getReviewerRole.ts` |
| thinker | `src/domain.roles/thinker/getThinkerRole.ts` |

all six are `bhrain` roles with onBoot hooks.

## .fix

change `./node_modules/.bin/npx rhachet` to `./node_modules/.bin/rhachet` in each role source file.

the source files generate hooks that end up in `.claude/settings.json`. fix at the source, not the generated artifact.

## .root cause

regression introduced in commit `2509fb4` (2026-04-20) — "fix(achiever): improve goal management with validation and escalation (#241)"

that commit added onBoot hooks to multiple roles with the incorrect `./node_modules/.bin/npx rhachet` pattern. the error was that `npx` was assumed to be a local binary (it's not — `npx` is a global npm tool).

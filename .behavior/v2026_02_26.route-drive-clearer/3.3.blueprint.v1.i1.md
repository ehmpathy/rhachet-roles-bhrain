## implementation details

### TTY detection

use `process.stdout.isTTY` to detect human vs agent:
- human sessions (terminals) → `isTTY === true`
- agent sessions (subprocesses) → `isTTY === false` or `undefined`

```ts
const isTTY = process.stdout.isTTY ?? false;
```

### error output for agents

add new FormatInput variant in `formatRouteStoneEmit.ts`:

```ts
| {
    operation: 'route.stone.set';
    stone: string;
    action: 'blocked';
    reason: string;
    guidance: string;
  }
```

then in `setStoneAsApproved`:

```ts
// check if caller is human
const { isHuman } = getDecisionIsCallerHuman({ isTTY: context.isTTY });
if (!isHuman) {
  return {
    approved: false,
    emit: {
      stdout: formatRouteStoneEmit({
        operation: 'route.stone.set',
        stone: stoneMatched.name,
        action: 'blocked',
        reason: 'only humans can approve',
        guidance: 'please ask a human to run this command',
      }),
    },
  };
}
```

add handler in `formatRouteStoneEmit` (early return, like other special cases):

```ts
// add after other early-return cases (challenged, passed with guard, etc.)
if (input.action === 'blocked') {
  lines.push(`🗿 ${input.operation}`);
  lines.push(`   └─ stone = ${input.stone}`);
  lines.push(`      ├─ ✗ ${input.reason}`);
  lines.push(`      └─ ${input.guidance}`);
  return lines.join('\n');
}
```
